## 1. НАВІГАЦІЯ ТА ХЕДЕР

### TC-D-1.1 Top Bar
**Передумова:** Будь-яка сторінка, 1280px+
1. Переконатися що зверху є тонкий рядок (text-xs) із:
   - Телефон +38 (000) 123-45-67 (клікабельний, `tel:`)
   - Графік роботи "Пн-Пт: 9:00-18:00, Сб: 10:00-15:00"
   - "Зворотній зв'язок" → `/contacts`
   - "Питання" → `/faq`
2. Переконатися що справа є іконки соцмереж: Telegram, Viber, Instagram, Facebook, TikTok (усі з `target="_blank"`)
3. На xl+ має бути текст "Безкоштовна доставка від 2000 ₴"
4. Проскролити вниз (>80px) — TopBar плавно зникає (max-h-0, opacity-0 transition)
5. Проскролити назад вгору (<20px) — TopBar з'являється
6. **Очікуваний результат:** TopBar функціональний, ховається/з'являється при скролі з гістерезисом

### TC-D-1.2 Логотип та перехід на головну
**Передумова:** Будь-яка сторінка
1. Переконатися що зліва є логотип: бейдж "П" (gradient background, shadow-brand, rounded-xl) + "Поро" + "шок" (градієнтний текст від primary до primary-light)
2. Натиснути на логотип
3. **Очікуваний результат:** Перехід на `/`

### TC-D-1.3 Пошук — inline в хедері
**Передумова:** Будь-яка сторінка, lg+ (на менших екранах — окреме поле під хедером)
1. Переконатися що поле пошуку видиме в хедері (placeholder "Пошук товарів...")
2. Клікнути в поле — з'являється випадаючий список з:
   - Історія пошуку (до 5 записів) з кнопкою "Очистити" (для авторизованих)
   - "Нещодавно переглянуті" (до 4 товарів, горизонтальний скрол)
   - "Популярне" (до 4 товарів)
3. Ввести "порошок" (мінімум 2 символи) — через 300мс з'являються результати: категорії + товари (з мініатюрою, назвою, кодом, ціною)
4. Натиснути Enter → перехід на `/catalog?search=порошок`
5. Перевірити клавіатурну навігацію: Arrow Up/Down по результатах, Enter для вибору, Escape для закриття
6. **Очікуваний результат:** Пошук з автопідказками, клавіатурною навігацією та історією

### TC-D-1.4 Іконки хедера — callback та чат
**Передумова:** Будь-яка сторінка, lg+
1. Переконатися що є іконка телефону (CallbackButton) — тільки десктоп
2. Натиснути → модалка "Замовити дзвінок" з полями:
   - Ім'я (обов'язкове)
   - Телефон (+38 формат, обов'язкове)
   - Бажаний час (dropdown: 9-12, 12-15, 15-18, Будь-який)
3. Заповнити → "Відправити" → повідомлення "Дякуємо! Ми зв'яжемося з вами найближчим часом."
4. Переконатися що є іконка чату (ChatWidget) — тільки десктоп
5. Натиснути → модалка з 5 FAQ-питаннями (клік розгортає відповідь, "Назад до питань")
6. Внизу FAQ — кнопка "Написати менеджеру в Telegram"
7. **Очікуваний результат:** Обидві модалки працюють коректно

### TC-D-1.5 Іконки хедера — сповіщення
**Передумова:** Авторизований з непрочитаними сповіщеннями
1. Переконатися що іконка дзвіночка (Bell) має червоний бейдж з кількістю непрочитаних
2. Кількість оновлюється кожні 60 секунд (polling)
3. Натиснути → перехід на `/account/notifications`
4. **Очікуваний результат:** Бейдж відображає актуальну кількість

### TC-D-1.6 Іконки хедера — обране
**Передумова:** Авторизований, є товари в обраному
1. Переконатися що іконка серця (Heart) має бейдж з кількістю
2. Натиснути → перехід на `/account/wishlist`
3. **Очікуваний результат:** Бейдж синхронізований з кількістю обраних товарів

### TC-D-1.7 Іконки хедера — кошик та міні-кошик
**Передумова:** Є товари в кошику
1. Переконатися що іконка кошика має бейдж з кількістю товарів
2. Переконатися що поруч відображається загальна сума (formatPrice, "XXX ₴")
3. Навести курсор на область кошика
4. **Очікуваний результат:** З'являється MiniCart dropdown:
   - Заголовок кошика
   - Список товарів (скролиться, max-height-64): мініатюра + назва-посилання + кількість (−/+) + ціна + іконка видалення
   - Внизу: "Разом: XXX ₴"
   - Кнопки "Кошик" та "Оформити"
5. Відвести курсор → MiniCart плавно закрився (fade out)
6. Порожній кошик → текст "Кошик порожній"

### TC-D-1.8 Іконки хедера — профіль
**Передумова:** Різні стани авторизації
1. **Авторизований:** іконка User з label "Профіль" → натиснути → `/account`
2. **Неавторизований:** іконка User з label "Увійти" → натиснути → `/auth/login`
3. **Очікуваний результат:** Коректний label та маршрут залежно від стану

### TC-D-1.9 Оптовий бейдж
**Передумова:** Авторизований користувач з роллю wholesaler
1. Біля іконки профілю — золотий бейдж "Оптовий клієнт" (gold gradient)
2. **Очікуваний результат:** Бейдж видимий тільки для оптових

### TC-D-1.10 Skip-to-content (доступність)
**Передумова:** Будь-яка сторінка
1. Натиснути Tab (перший після завантаження)
2. **Очікуваний результат:** З'являється посилання "Перейти до основного вмісту"
3. Enter → фокус переноситься на `<main id="main-content">`

### TC-D-1.11 Категорійна навігація (CategoryNav)
**Передумова:** Будь-яка сторінка, 1024px+ (hidden на мобільній)
1. Переконатися що під хедером є синя панель з градієнтом (primary-dark → primary)
2. Зліва — кнопка "Каталог" (золота, gradient, з іконкою гамбургера)
3. Поруч — кнопка "Акції" (червона/помаранчева, з іконкою вогню) → `/catalog?promo=true`
4. Далі — назви батьківських категорій (білий текст)
5. Навести курсор на категорію з підкатегоріями
6. **Очікуваний результат:** Мега-меню:
   - Grid підкатегорій (2-4 колонки залежно від кількості)
   - Кількість товарів біля кожної підкатегорії
   - "Дивитись все" → `/catalog?category=slug`
   - Якщо є coverImage — зображення справа
7. Натиснути Escape → мега-меню закрилось
8. Перевірити hover delay (меню не блимає при переміщенні курсора)

### TC-D-1.12 Хедер — поведінка при скролі
**Передумова:** Будь-яка сторінка
1. Проскролити вниз (>80px)
2. **Очікуваний результат:** TopBar зникає, HeaderMain стає компактнішим (shrink prop), CategoryNav shrink
3. Проскролити назад вгору (<20px)
4. **Очікуваний результат:** TopBar з'являється, хедер повертається до звичайного розміру

### TC-D-1.13 Немає мобільного таб-бару на десктопі
**Передумова:** 1024px+
1. **Очікуваний результат:** Нижнього таб-бару НЕМАЄ (MobileBottomNav має class hidden lg:)

## 2. ГОЛОВНА СТОРІНКА

### TC-D-2.1 Банер — відображення
**Передумова:** Головна, 1280px+
1. Банер має пропорції 5:2 (aspect-[5/2])
2. Банер має rounded-3xl та shadow-xl
3. Текстовий блок — glass-dark стиль (напівпрозорий), обмежений lg:max-w-[60%]
4. Заголовок великий (lg:text-5xl), числа та "₴" виділені золотим кольором
5. Підзаголовок видимий (hidden на мобільній, block на sm+)
6. CTA-кнопка помітна (з посиланням)
7. Декоративні SVG-елементи на фоні (opacity 0.04-0.06)
8. **Очікуваний результат:** Повноцінний банер з усіма елементами

### TC-D-2.2 Банер — автопрокрутка та hover-пауза
**Передумова:** Головна, банер з кількома слайдами
1. Зачекати 5с → банер автоматично переключився
2. Навести курсор на банер → зачекати 10с → НЕ переключився
3. Відвести курсор → зачекати 5с → переключився
4. **Очікуваний результат:** Hover зупиняє автопрокрутку

### TC-D-2.3 Банер — стрілки навігації
**Передумова:** Головна
1. Натиснути ← (ліву стрілку) → попередній слайд
2. Натиснути → (праву стрілку) → наступний слайд
3. Hover на стрілку → зміна стилю (background)
4. **Очікуваний результат:** Стрілки працюють, анімація плавна

### TC-D-2.4 Банер — прогрес-бар
**Передумова:** Банер з кількома слайдами
1. Індикатори — горизонтальні лінії (НЕ крапки)
2. Активний індикатор має прогрес-анімацію (заповнюється за 5с)
3. При hover — прогрес зупиняється, лінія заповнена на 100%
4. Клік на інший індикатор → перехід на відповідний слайд
5. **Очікуваний результат:** Прогрес-бар синхронізований з автопрокруткою

### TC-D-2.5 Категорії — відображення тільки в CategoryNav
**Передумова:** Головна, десктоп (lg+)
1. Категорії є ТІЛЬКИ у CategoryNav (синя панель в хедері)
2. Блок CategoryGrid на самій сторінці ПРИХОВАНИЙ (hidden lg:)
3. **Очікуваний результат:** Категорії не дублюються

### TC-D-2.6 Каруселі товарів
**Передумова:** Головна, є акційні/нові/популярні товари
1. Переконатися що є до 3 каруселей:
   - "Акційні товари" (посилання "Дивитися все" → `/catalog?promo=true`)
   - "Новинки" → `/catalog?sort=newest`
   - "Хіти продажів" → `/catalog?sort=popular`
2. Стрілки навігації каруселі працюють
3. Навести курсор на картку → картка піднімається (-translate-y-1), з'являється друге фото (hover image swap, fade, duration-500)
4. Натиснути "В кошик" → toast-повідомлення, бейдж кошика +1
5. **Очікуваний результат:** Каруселі з товарними картками, hover-ефекти працюють

### TC-D-2.7 Картка товару — деталі
**Передумова:** Каталог або карусель
1. На картці є:
   - Зображення (lazy loading, blur placeholder)
   - До 2 бейджів (акція, новинка тощо)
   - Іконка серця (обране) — toggle заповнене/порожнє
   - Іконка лупи (швидкий перегляд) — тільки десктоп, при hover
   - Тег категорії (all-caps, дрібний)
   - Назва товару (2-line clamp)
   - Атрибути (об'єм/вага/кількість)
   - Ціна зі знижкою (стара закреслена)
   - Статус наявності ("В наявності" / "Немає в наявності")
   - Кнопка "В кошик" (disabled якщо немає в наявності)
2. Hover: зображення зумиться, картка піднімається
3. **Очікуваний результат:** Картка інформативна та інтерактивна

### TC-D-2.8 Картка товару — hover image swap
**Передумова:** Товар з 2+ фото
1. Навести курсор на картку товару
2. **Очікуваний результат:** Головне фото плавно змінюється на друге (fade, duration-500)
3. Відвести курсор → повернулось перше фото

### TC-D-2.9 Блок брендів — автоскрол
**Передумова:** Головна, "Наші бренди"
1. Заголовок "Наші бренди" із золотим підкресленням
2. Бренди: Frosch, Fairy, Persil, Tide, Domestos, Mr. Proper, Gala, Sarma, Ariel, Vanish, Bref, Somat
3. Бренди автоматично рухаються (translateX animation, 30s, infinite, linear)
4. Навести курсор → анімація зупинилась
5. Відвести курсор → анімація продовжилась
6. Натиснути на бренд → `/catalog?brand=slug`
7. **Очікуваний результат:** Нескінченний горизонтальний скрол брендів

### TC-D-2.10 SEO-текст
**Передумова:** Головна, кінець сторінки
1. Блок має gradient фон (primary-50/60 → white) та тонку рамку
2. Заголовок "Інтернет-магазин побутової хімії Порошок" (lg:text-xl)
3. Текст обмежений max-w-3xl, не розтягнутий на всю ширину
4. **Очікуваний результат:** SEO-текст читабельний та стильно оформлений

### TC-D-2.11 Порожні блоки не відображаються
**Передумова:** Головна
1. Якщо немає акційних товарів — блок "Акційні товари" не рендериться
2. Блоки керуються з БД (getHomepageBlocks), тільки enabled блоки показуються
3. Між блоками немає зайвих порожніх просторів
4. **Очікуваний результат:** Контент щільний, без "дірок"

### TC-D-2.12 Нещодавно переглянуті
**Передумова:** Переглянути 2-3 товари, повернутися на головну
1. Блок "Нещодавно переглянуті" з'являється внизу
2. Карусель з переглянутими товарами (ProductCarousel)
3. Якщо нічого не переглядали — блок не показується (self-hiding)
4. **Очікуваний результат:** Блок відображає реальну історію перегляду

## 3. КАТАЛОГ

### TC-D-3.1 Сітка каталогу
**Передумова:** `/catalog`
1. Товари в сітці: 3 колонки (lg) → 4 колонки (xl) → 6 колонок (2xl)
2. Зліва — бічна панель фільтрів FilterSidebar (sticky top-[140px], ширина 256px)
3. Зверху — CatalogToolbar з кількістю товарів та сортуванням
4. Навести курсор на картку → hover-ефект + друге фото
5. **Очікуваний результат:** Стандартний layout каталогу з фільтрами

### TC-D-3.2 Фільтри — sticky sidebar
**Передумова:** `/catalog` з довгим списком
1. Проскролити сторінку вниз
2. **Очікуваний результат:** Бічна панель фільтрів залишається видимою (sticky)

### TC-D-3.3 Фільтр за категорією
**Передумова:** `/catalog`
1. У FilterSidebar є чекбокси батьківських категорій із кількістю товарів
2. Обрати категорію → натиснути "Застосувати"
3. **Очікуваний результат:** Список оновився, URL `?category=slug`

### TC-D-3.4 Фільтр за ціною
**Передумова:** `/catalog`
1. Двоповзунковий слайдер (0-10000 ₴, крок 50)
2. Або ввести вручну: "від" 100, "до" 500
3. "Застосувати"
4. **Очікуваний результат:** Тільки товари 100-500 грн, URL `?price_min=100&price_max=500`

### TC-D-3.5 Фільтр "В наявності" / "Акції"
**Передумова:** `/catalog`
1. Чекбокс "В наявності" → товарів "Немає в наявності" зникли, URL `?in_stock=true`
2. Чекбокс "Тільки акційні" → тільки акційні товари, URL `?promo=true`
3. **Очікуваний результат:** Фільтри працюють, URL оновлюється

### TC-D-3.6 Скидання фільтрів
**Передумова:** Застосовано кілька фільтрів
1. Натиснути "Скинути" → всі фільтри знято
2. **Очікуваний результат:** Чистий `/catalog`, всі товари показуються

### TC-D-3.7 Сортування
**Передумова:** `/catalog`
1. Dropdown з 5 опціями:
   - "За популярністю" (default)
   - "Новинки"
   - "Від дешевих" → від найдешевших
   - "Від дорогих" → від найдорожчих
   - "За назвою" → за алфавітом
2. Вибір сортування скидає пагінацію на сторінку 1
3. **Очікуваний результат:** Товари відсортовані відповідно

### TC-D-3.8 Пагінація
**Передумова:** 20+ товарів
1. Натиснути "2" → `?page=2`
2. Натиснути "1" → повернення
3. **Очікуваний результат:** Пагінація працює, URL оновлюється

### TC-D-3.9 Пошук у каталозі
**Передумова:** Пошук через хедер
1. Результати: заголовок 'Результати пошуку: "порошок"'
2. Порожні результати: іконка пошуку + "Товарів не знайдено" + посилання скидання
3. **Очікуваний результат:** Пошук працює з відповідними повідомленнями

### TC-D-3.10 Фільтри зберігаються при навігації
1. Обрати категорію → клік на товар → "Назад" у браузері
2. **Очікуваний результат:** Фільтр категорії збережений в URL

## 4. СТОРІНКА ТОВАРУ

### TC-D-4.1 Галерея — вертикальні thumbnails + zoom
**Передумова:** Товар з 3+ фото
1. Зліва — вертикальна колонка мініатюр
2. Справа — велике головне фото (rounded-2xl, blur placeholder)
3. Навести курсор на другу мініатюру (onMouseEnter) → головне фото змінилось
4. Натиснути на третю мініатюру → головне фото = третє, мініатюра має active border
5. Навести курсор на головне фото → курсор zoom-in, зум слідує за мишкою (transform-origin)
6. Натиснути на головне фото → відкрилась lightbox-модалка:
   - Стрілки навігації (вліво/вправо)

### TC-D-12.1 End-to-End: покупка без реєстрації
**Передумова:** Неавторизований, 1440px

4. Контакти (ім'я, телефон, email) → Нова Пошта (місто, адреса) → "Касса при доставці" → Підтвердити
5. **Очікуваний результат:** Замовлення створено, номер відображається

### TC-D-12.2 End-to-End: покупка з реєстрацією
1. `/auth/register` → ПІБ, email, пароль → реєстрація
2. Пошук товару → обране (серце) → `/account/wishlist` → "Додати все в кошик"
3. `/cart` → checkout (автозаповнення контактів) → оформити
4. `/account/orders` → знайти замовлення → деталі
5. **Очікуваний результат:** Повний цикл покупки з обліковим записом

### TC-D-12.3 Респонсивність
1. **1280px** — все на місці, CategoryNav не обрізається
2. **1440px** — контент центрований (max-w-[1440px])
3. **1920px** — контент центрований, великі поля по боках
4. **2560px** — layout не ламається
5. **Очікуваний результат:** Коректне відображення на всіх роздільних здатностях

### TC-D-12.4 Крос-браузерність

# Тестування функціоналу замовлень — ручне тестування

**URL адмінки:** http://localhost:3000/admin/orders
**Обліковий запис:** admin@clean-shop.ua (роль admin)

---

## Підготовка

1. Увійти в адмін-панель як адміністратор
2. Переконатися, що в базі є замовлення (перейти на `/admin/orders`)
3. Якщо замовлень немає — створити тестове замовлення через магазин:
   - Відкрити головну сторінку → додати товар в кошик → оформити замовлення

## 1. Список замовлень (`/admin/orders`)

### 1.1 Завантаження списку
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Відкрити `/admin/orders` | Таблиця замовлень завантажується, спінер зникає | |
| 2 | Перевірити таблицю | 7 колонок: замовлення, клієнт, статус, оплата, доставка, сума, дії | |
| 3 | Перевірити картки статистики | 5 карток: Нових, В обробці, Очікують оплати, Замовлень сьогодні, Виручка сьогодні | |
| 4 | Натиснути картку "Нових" | Фільтрується по статусу `new_order`, URL змінюється | |

### 1.2 Пошук
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Ввести номер замовлення в пошук → "Знайти" | Показує тільки замовлення з цим номером | |
| 2 | Ввести ім'я клієнта → "Знайти" | Фільтрує за іменем | |
| 3 | Ввести телефон → "Знайти" | Фільтрує за телефоном | |
| 4 | Ввести номер ТТН → "Знайти" | Фільтрує за ТТН (якщо є) | |
| 5 | Очистити пошук → "Знайти" | Повертає всі замовлення | |

### 1.3 Сортування
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Вибрати "Найновіші" | Замовлення відсортовані за датою (нові зверху) | |
| 2 | Вибрати "Найстаріші" | Замовлення відсортовані за датою (старі зверху) | |
| 3 | Вибрати "Сума (більше)" | Замовлення з найбільшою сумою зверху | |
| 4 | Вибрати "Сума (менше)" | Замовлення з найменшою сумою зверху | |
| 5 | Вибрати "Статус (A-Z)" | Сортування за статусом | |
| 6 | Вибрати "Номер (новіші)" | Сортування за номером замовлення | |

### 1.4 Фільтри
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Натиснути "Фільтри" | Панель фільтрів розгортається | |
| 2 | Вибрати статус "Нове" | Показує тільки нові замовлення, URL оновлюється | |
| 3 | Вибрати тип клієнта "Опт" | Показує тільки оптові замовлення | |
| 4 | Вибрати спосіб оплати "Накладений платіж" | Фільтрує за способом оплати | |
| 5 | Вибрати доставку "Нова Пошта" | Фільтрує за способом доставки | |
| 6 | Встановити "Дата від" | Показує замовлення після цієї дати | |
| 7 | Встановити "Дата до" | Показує замовлення до цієї дати | |
| 8 | Комбінація: статус + оплата + доставка | Всі фільтри працюють разом | |
| 9 | Натиснути "Скинути фільтри" | Усі фільтри скидаються, показує всі замовлення | |
| 10 | Натиснути "Сховати фільтри" | Панель згортається, синя крапка біля кнопки якщо фільтри активні | |

### 1.5 Швидка зміна статусу (інлайн)
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Знайти замовлення зі статусом "Нове" | В колонці "Дії" є іконка олівця | |
| 2 | Натиснути іконку олівця | З'являється випадаючий список з доступними статусами | |

| 4 | Перевірити що для "Скасоване" / "Повернення" немає олівця | Для кінцевих статусів немає кнопки зміни | |
| 5 | Натиснути × для скасування | Інлайн-редактор закривається без змін | |

### 1.6 Пагінація
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|

| 2 | Натиснути на 2-гу сторінку (якщо є >20 замовлень) | Завантажуються наступні замовлення, URL оновлюється | |

## 2. Деталі замовлення (`/admin/orders/[id]`)

### 2.1 Завантаження деталей
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Натиснути на номер замовлення в списку | Відкривається сторінка деталей | |
| 2 | Перевірити заголовок | Номер замовлення, статус (кольоровий бейдж), дата, мітка ОПТ (якщо опт) | |
| 3 | Перевірити картку "Клієнт" | Ім'я, телефон, email. Якщо зареєстрований — посилання на профіль | |
| 4 | Перевірити картку "Доставка" | Спосіб доставки, місто, адреса/відділення | |
| 5 | Перевірити картку "Оплата" | Спосіб оплати, статус, суми (товари, знижка, доставка, всього) | |
| 6 | Перевірити список товарів | Фото, назва, код, ціна × кількість, підсумок. Мітка "Акція" якщо акційний | |
| 7 | Кнопка "← Замовлення" | Повертає до списку замовлень | |

### 2.2 Зміна статусу (на сторінці деталей)
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Блок "Змінити статус" видимий | Відображається тільки якщо є доступні переходи | |
| 2 | Вибрати новий статус → "Оновити" | Статус змінюється, бейдж оновлюється | |
| 3 | Додати коментар → "Оновити" | Коментар зберігається в історії змін | |
| 4 | Спробувати недопустимий перехід (через API) | Помилка "Неможливо змінити статус..." | |
| 5 | Перевірити "Історія змін" внизу | Показує всі зміни з датою, джерелом, коментарем | |

### 2.3 Повний ланцюг статусів
Пройти весь цикл на одному замовленні:

| # | Перехід | Крок | Очікуваний результат | Статус |
|---|---------|------|---------------------|--------|
| 1 | Нове → В обробці | Вибрати "В обробці" → Оновити | Статус змінився | |
| 2 | В обробці → Підтверджене | Вибрати "Підтверджене" → Оновити | Статус змінився | |
| 3 | Підтверджене → Оплачене | Вибрати "Оплачене" → Оновити | Статус змінився | |
| 4 | Оплачене → Відправлене | Вибрати "Відправлене" → Оновити | Статус змінився | |
| 5 | Відправлене → Виконане | Вибрати "Виконане" → Оновити | Статус змінився, немає більше доступних переходів (крім "Повернення") | |

Альтернативний шлях — скасування:

| # | Перехід | Очікуваний результат | Статус |
|---|---------|---------------------|--------|
| 1 | Нове → Скасоване | Статус змінився, залишки товарів відновлені | |
| 2 | Скасоване → будь-який | Блок зміни статусу не відображається | |

## 3. ТТН (трекінг-номер)

### 3.1 Ручне введення ТТН
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Відкрити замовлення без ТТН | В картці "Доставка" є поле "Додати ТТН" | |
| 2 | Ввести номер ТТН (напр. `20450000000000`) → "OK" | Зелене повідомлення "ТТН збережено", ТТН відображається | |
| 3 | Натиснути "Змінити" біля ТТН | Поле редагування з поточним значенням | |
| 4 | Змінити номер → "OK" | ТТН оновлено | |
| 5 | Натиснути "X" під час редагування | Повернення до попереднього значення без збереження | |
| 6 | Спробувати зберегти порожній ТТН | Кнопка "OK" неактивна (disabled) | |
| 7 | Перевірити в списку замовлень | ТТН відображається в колонці "Доставка" | |

### 3.2 Створення ТТН через API Нової Пошти
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Відкрити замовлення з доставкою "Нова Пошта" без ТТН | Є посилання "Або створити через API Нової Пошти" | |
| 2 | Натиснути на посилання | Відкривається модальне вікно з формою | |
| 3 | Перевірити автозаповнення | Ім'я отримувача, телефон, місто підставлені з замовлення | |
| 4 | Ввести місто відправника (пошук) | Випадаючий список міст з'являється після 2+ символів | |
| 5 | Вибрати місто → перевірити склади | Завантажується список відділень/поштоматів | |
| 6 | Заповнити вагу, кількість місць, опис | Поля валідуються | |
| 7 | Натиснути "Скасувати" | Модальне вікно закривається | |

> **Примітка:** Повне тестування створення ТТН вимагає реальних API-ключів Нової Пошти та налаштування відправника в їхньому кабінеті.

## 9. Сповіщення про нові замовлення

### 9.1 Бейдж нових замовлень
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Відкрити `/admin/orders` | Статистика завантажується | |
| 2 | Створити нове замовлення (з іншого браузера/вкладки) | Протягом 30 секунд з'являється анімований бейдж "+N нових" | |
| 3 | Натиснути на бейдж | Фільтр змінюється на "Нові", бейдж зникає | |

### 9.2 Звуковий сигнал
| # | Крок | Очікуваний результат | Статус |
|---|------|---------------------|--------|
| 1 | Дочекатися нового замовлення (або змоделювати) | Коротий звуковий сигнал (біп) при появі бейджа | |
| 2 | Перевірити що звук тільки при збільшенні кількості | Повторний полінг без нових замовлень — тиша | |

> **Як змоделювати:** Відкрити 2 вкладки. В першій — адмінка `/admin/orders`. В другій — магазин, оформити замовлення. Протягом 30с в першій вкладці має з'явитись бейдж.

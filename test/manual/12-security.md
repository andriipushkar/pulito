# 12 — Безпека

## 12.1 CSRF захист

### TC-SE-1.1 Запит без X-Requested-With
**Кроки:**
1. Відкрити DevTools → Console
2. Виконати: `fetch('/api/v1/auth/login', { method: 'POST', headers: {'Content-Type':'application/json'}, body: '{}' }).then(r=>r.json()).then(console.log)`
3. Перевірити відповідь
**Очікуваний результат:** `CSRF check failed: missing X-Requested-With header`
**Статус:** ⬜

---

## 12.2 Rate Limiting

### TC-SE-2.1 Глобальний rate limit (API)
**Кроки:**
1. Швидко оновити сторінку 120+ разів за хвилину (або через curl loop)
**Очікуваний результат:** 429 Too Many Requests з Retry-After header
**Статус:** ⬜

### TC-SE-2.2 Rate limit на Server Actions
**Кроки:**
1. Спробувати оформити 6+ замовлень за хвилину
**Очікуваний результат:** "Забагато запитів. Спробуйте пізніше." після 5-го
**Статус:** ⬜

### TC-SE-2.3 Rate limit на логін
**Кроки:**
1. 5 невдалих спроб входу підряд
**Очікуваний результат:** Блокування на 15 хвилин
**Статус:** ⬜

---

## 12.3 Завантаження файлів

### TC-SE-3.1 Upload санітизація
**Кроки:**
1. В адмінці завантажити JPEG зображення
2. Перевірити що файл збережений як .webp (перекодований через sharp)
3. Перевірити відсутність EXIF метаданих (через exiftool або онлайн)
**Очікуваний результат:** Файл перекодований, метадані видалені
**Статус:** ⬜

### TC-SE-3.2 Заборонені формати
**Кроки:**
1. Спробувати завантажити .php файл (перейменований в .jpg)
2. Спробувати завантажити .svg
**Очікуваний результат:** Помилка "Вміст файлу не відповідає заявленому формату"
**Статус:** ⬜

### TC-SE-3.3 Ліміт розміру
**Кроки:**
1. Спробувати завантажити файл > 5MB
**Очікуваний результат:** Помилка "Файл занадто великий"
**Статус:** ⬜

## 12.4 Security Headers

### TC-SE-4.1 Перевірка headers
**Кроки:**
1. Відкрити DevTools → Network → будь-який запит
2. Перевірити наявність:
   - `X-Frame-Options: DENY`
   - `X-Content-Type-Options: nosniff`
   - `Strict-Transport-Security`
   - `Content-Security-Policy`
   - `Referrer-Policy`
   - `Permissions-Policy`
**Очікуваний результат:** Всі заголовки присутні
**Статус:** ⬜

## 12.5 Path Traversal

### TC-SE-5.1 Спроба path traversal
**Кроки:**
1. Відкрити `/uploads/../.env`
2. Відкрити `/uploads/../../etc/passwd`
**Очікуваний результат:** 404 Not Found для обох
**Статус:** ⬜

## 12.6 XSS

### TC-SE-6.1 XSS в пошуку
**Кроки:**
1. Ввести в пошук: `<script>alert('xss')</script>`
**Очікуваний результат:** Текст відображається як текст, alert не спрацьовує
**Статус:** ⬜

### TC-SE-6.2 XSS в відгуку
**Кроки:**
1. Залишити відгук з текстом: `<img src=x onerror=alert(1)>`
2. Якщо опублікований — перевірити на сторінці товару
**Очікуваний результат:** HTML екрановано, скрипт не виконується
**Статус:** ⬜

---

## 12.7 Доступ до адмінки

### TC-SE-7.1 Доступ звичайним юзером
**Передумова:** Увійти як звичайний юзер (не admin)
**Кроки:**
1. Перейти на `/admin`
2. Спробувати відкрити `/admin/orders`
**Очікуваний результат:** 403 Forbidden або редирект
**Статус:** ⬜

### TC-SE-7.2 API доступ без токена
**Кроки:**
1. Відкрити DevTools → Console
2. `fetch('/api/v1/admin/orders', {headers:{'X-Requested-With':'XMLHttpRequest'}}).then(r=>r.json()).then(console.log)`
**Очікуваний результат:** 401 "Токен не надано"
**Статус:** ⬜

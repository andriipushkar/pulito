## 8.3 Звіти

### TC-AA-3.1 Генерація PDF звіту
**Передумова:** `/admin/reports`
**Кроки:**
1. Обрати шаблон: "Звіт про продажі"
2. Вказати період: останній місяць
3. Обрати формат: PDF
4. Натиснути "Генерувати"
5. Завантажити файл
6. Відкрити PDF і перевірити:
   - Синій хедер з назвою компанії
   - Заголовок звіту з датою
   - Таблиця з даними (колонки рівномірно)
   - Футер з контактами
**Очікуваний результат:** Гарний PDF з фірмовим стилем
**Статус:** ⬜

### TC-AA-3.2 Генерація Excel звіту
**Кроки:**
1. Обрати шаблон: "Залишки товарів"

3. Генерувати → завантажити → відкрити

## 8.5 SEO та розширена аналітика

### TC-AA-5.1 SEO Аудит
**Передумова:** Увійти як admin, перейти `/admin/seo-audit`
**Кроки:**
1. Перевірити список сторінок з SEO проблемами
2. Перевірити виявлені біті посилання (broken links)
3. Перевірити список сторінок з пропущеними meta (title, description)
4. Натиснути на проблему — перевірити деталі та рекомендації
**Очікуваний результат:** SEO аудит показує біті посилання, пропущені meta теги, рекомендації для виправлення
**Статус:** ⬜

### TC-AA-5.2 SEO шаблони
**Передумова:** Увійти як admin, перейти `/admin/seo-templates`
**Кроки:**
1. Натиснути "Створити шаблон"
2. Обрати тип сторінки (товар, категорія, новина)
3. Задати шаблон meta title: `{product_name} — купити в {shop_name}`
4. Задати шаблон meta description: `{product_name} за ціною {price} грн. {category_name} в інтернет-магазині.`
5. Зберегти
6. Перевірити що автогенерація працює на товарах без ручних meta
**Очікуваний результат:** Правило автогенерації meta створено, шаблон застосовується до сторінок без ручних meta
**Статус:** ⬜

### TC-AA-5.3 Когортний аналіз
**Передумова:** Увійти як admin, перейти `/admin/analytics` → Cohorts
**Кроки:**
1. Перевірити таблицю когорт (по місяцях реєстрації)
2. Перевірити retention rate для кожної когорти
3. Змінити період аналізу
4. Перевірити кольорову шкалу (heat map)
**Очікуваний результат:** Когортна таблиця з retention rate відображається, дані змінюються при зміні періоду
**Статус:** ⬜

### TC-AA-5.4 Churn аналіз
**Передумова:** Увійти як admin, перейти `/admin/analytics` → Churn
**Кроки:**
1. Перевірити метрику churn rate за обраний період
2. Перевірити графік динаміки відтоку
3. Перевірити список клієнтів "At Risk" (під загрозою відтоку)
4. Змінити період — перевірити оновлення даних
**Очікуваний результат:** Churn аналіз показує відсоток відтоку, динаміку та список клієнтів під загрозою
**Статус:** ⬜

### TC-AA-5.5 Performance метрики
**Передумова:** Увійти як admin, перейти `/admin/analytics` → Performance
**Кроки:**
1. Перевірити метрики продуктивності: час завантаження сторінок, час відповіді API
2. Перевірити графіки: TTFB, LCP, CLS, FID
3. Перевірити статистику по найповільніших сторінках
4. Змінити період — перевірити оновлення
**Очікуваний результат:** Performance метрики відображаються з графіками та деталями по сторінках
**Статус:** ⬜

## 8.6 Дашборд — KPI, прогнози, auto-insights

### TC-AA-6.1 KPI картки на дашборді
**Передумова:** `/admin/analytics` → Dashboard
**Кроки:**
1. Перевірити 6 KPI карток: Виручка, Замовлення, Середній чек, Нові користувачі, Очікуючі замовлення, Low Stock
2. Кожна картка має: значення + % зміни (зелений/червоний)
3. Змінити період з 7 на 30 днів → значення оновлюються
**Очікуваний результат:** 6 KPI з порівнянням до попереднього періоду
**Статус:** ⬜

### TC-AA-6.2 Auto-Insights
**Кроки:**
1. На дашборді знайти блок "Insights" або "Автоматичні спостереження"
2. Перевірити наявність alerts: виручка зросла/впала >10%, замовлення впали >15%, pending >5, low stock <5
**Очікуваний результат:** Автоматичні попередження відображаються при аномаліях
**Статус:** ⬜

### TC-AA-6.3 Прогноз (Forecast)
**Кроки:**
1. На дашборді знайти прогноз
2. Перевірити: середній дохід/день, прогноз на 7 днів, прогноз на 30 днів
**Очікуваний результат:** Числові прогнози на основі середнього за останній тиждень
**Статус:** ⬜

**Очікуваний результат:** Графік + таблиця + порівняння + CSV файл

### TC-AA-7.2 Вкладка "Товари" (Products)
**Кроки:**

3. Перевірити: оцінка маржі

### TC-AA-7.3 Вкладка "Клієнти" (Clients)

1. Перевірити: нові юзери, загальна кількість, оптовиків

## 8.8 Алерти аналітики

### TC-AA-8.1 Створення алерту
**Передумова:** `/admin/analytics` → Alerts
**Кроки:**
1. Натиснути "Створити алерт"
2. Обрати метрику: `daily_revenue`
3. Умова: `below` (менше)
4. Поріг: `1000`
5. Канал: `telegram`
6. Зберегти
**Очікуваний результат:** Алерт створено, відображається у списку
**Статус:** ⬜

### TC-AA-8.2 Спрацювання алерту
**Передумова:** Алерт daily_revenue < 1000 активний
**Кроки:**
1. Запустити cron `analytics-alerts`
2. Перевірити Telegram/email — чи прийшло повідомлення
**Очікуваний результат:** Сповіщення при порушенні порогу
**Статус:** ⬜

### TC-AA-8.3 Редагування та видалення алерту
**Кроки:**
1. Натиснути "Редагувати" → змінити поріг → зберегти
2. Натиснути "Видалити" → підтвердити
**Очікуваний результат:** CRUD операції з алертами працюють
**Статус:** ⬜

### TC-AA-8.4 Доступні метрики алертів
**Кроки:**
1. Перевірити список метрик у dropdown:
   - `daily_revenue` (денна виручка)
   - `daily_orders` (денні замовлення)
   - `avg_check` (середній чек)
   - `stock_zero` (товарів без залишку)
   - `new_users` (нових юзерів)
   - `cancelled_orders` (скасованих)
**Очікуваний результат:** Всі 6 метрик доступні
**Статус:** ⬜

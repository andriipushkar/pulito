# 10 — Адмін: налаштування системи

## 10.1 Загальні налаштування (6 вкладок)

### TC-AS-1.1 Вкладка "Загальні"
**Передумова:** Увійти як admin, перейти `/admin/settings`, вкладка "Загальні"
**Кроки:**
1. Перевірити поля: назва сайту, телефон, email, адреса, графік роботи
2. Змінити телефон на `+380 44 123 45 67`
3. Змінити графік роботи на `Пн-Сб 09:00-20:00`
4. Натиснути "Зберегти"
5. Перевірити що телефон оновився у хедері/футері сайту
**Очікуваний результат:** Поля збережені, зміни відображаються на сайті
**Статус:** ⬜

### TC-AS-1.2 Вкладка "Компанія"
**Передумова:** `/admin/settings`, вкладка "Компанія"
**Кроки:**
1. Перевірити поля: юридична назва, ЄДРПОУ, ІПН, IBAN, банк, юридична адреса
2. Змінити юридичну назву на `ТОВ "Тест Компанія"`
3. Ввести ЄДРПОУ: `12345678`
4. Ввести ІПН: `1234567890`
5. Ввести IBAN: `UA213223130000026007233566001`
6. Вказати банк та юридичну адресу
7. Зберегти
**Очікуваний результат:** Усі реквізити компанії збережені
**Статус:** ⬜

### TC-AS-1.3 Вкладка "Соціальні мережі"
**Передумова:** `/admin/settings`, вкладка "Соціальні мережі"
**Кроки:**
1. Перевірити поля: Telegram, Viber, Facebook, Instagram
2. Ввести посилання на Telegram: `https://t.me/testshop`
3. Ввести посилання на Viber: `viber://chat?number=380991234567`
4. Ввести посилання на Facebook: `https://facebook.com/testshop`
5. Ввести посилання на Instagram: `https://instagram.com/testshop`
6. Зберегти
7. Перевірити іконки соцмереж у футері сайту — посилання коректні
**Очікуваний результат:** Посилання на соцмережі збережені, іконки у футері ведуть на правильні URL
**Статус:** ⬜

### TC-AS-1.4 Вкладка "Аналітика"
**Передумова:** `/admin/settings`, вкладка "Аналітика"
**Кроки:**
1. Ввести Google Analytics ID: `G-XXXXXXXXXX`
2. Ввести Facebook Pixel ID: `1234567890`
3. Заповнити SEO за замовчуванням: title, description, keywords
4. Зберегти
5. Перевірити що GA та Pixel скрипти присутні у source коді сайту
**Очікуваний результат:** Аналітичні коди збережені та підключені, SEO meta-теги за замовчуванням на місці
**Статус:** ⬜

### TC-AS-1.5 Збереження загальних налаштувань (всі вкладки)
**Передумова:** `/admin/settings`, внесені зміни на кількох вкладках
**Кроки:**
1. Змінити дані на вкладці "Загальні"
2. Переключитися на "Компанія" — внести зміни
3. Зберегти
4. Оновити сторінку — перевірити що всі зміни збереглися на обох вкладках
**Очікуваний результат:** Дані зберігаються при переключенні між вкладками
**Статус:** ⬜

### TC-AS-1.6 Валідація полів загальних налаштувань
**Передумова:** `/admin/settings`
**Кроки:**
1. Очистити обов'язкове поле "Назва сайту"
2. Натиснути "Зберегти"
3. Перевірити що з'явилась помилка валідації
4. Ввести некоректний email — перевірити помилку формату
**Очікуваний результат:** Валідація не дає зберегти пусті або некоректні обов'язкові поля
**Статус:** ⬜

## 10.2 Налаштування доставки

### TC-AS-2.1 Поріг безкоштовної доставки
**Передумова:** Увійти як admin, перейти `/admin/delivery-settings`
**Кроки:**
1. Знайти поле "Безкоштовна доставка від"
2. Ввести суму: `1500` грн
3. Зберегти
4. На фронті: оформити замовлення на 1600 грн — доставка безкоштовна
5. На фронті: оформити замовлення на 1000 грн — доставка платна
**Очікуваний результат:** Поріг безкоштовної доставки працює коректно
**Статус:** ⬜

### TC-AS-2.2 Картка провайдера: Нова Пошта
**Передумова:** `/admin/delivery-settings`, картка "Нова Пошта"
**Кроки:**
1. Перевірити перемикач увімкнення — увімкнути
2. Ввести API ключ (перевірити що поле password з кнопкою show/hide)
3. Заповнити дані відправника: ПІБ, телефон, місто, відділення
4. Вказати фіксовану вартість доставки: `70` грн
5. Натиснути "Тест підключення" — перевірити результат (OK / помилка)
6. Зберегти
**Очікуваний результат:** Нова Пошта налаштована, тест успішний, дані відправника збережені
**Статус:** ⬜

### TC-AS-2.3 Картка провайдера: УкрПошта
**Передумова:** `/admin/delivery-settings`, картка "УкрПошта"
**Кроки:**
1. Перевірити перемикач увімкнення — увімкнути
2. Ввести API токен (password show/hide)
3. Заповнити дані відправника
4. Вказати фіксовану вартість: `50` грн
5. Натиснути "Тест підключення"
6. Зберегти
**Очікуваний результат:** УкрПошта налаштована, тест пройшов
**Статус:** ⬜

### TC-AS-2.4 Картка провайдера: Meest
**Передумова:** `/admin/delivery-settings`, картка "Meest"
**Кроки:**
1. Увімкнути перемикач
2. Ввести API ключ (password show/hide)
3. Заповнити дані відправника
4. Вказати фіксовану вартість: `65` грн
5. Натиснути "Тест підключення"
6. Зберегти
**Очікуваний результат:** Meest налаштований, тест підключення успішний
**Статус:** ⬜

### TC-AS-2.5 Помилки валідації доставки
**Передумова:** `/admin/delivery-settings`
**Кроки:**
1. Увімкнути провайдера без заповнення API ключа
2. Натиснути "Зберегти"
3. Перевірити що з'явилися помилки валідації біля обов'язкових полів
4. Натиснути "Тест підключення" без ключа — повідомлення про помилку
**Очікуваний результат:** Валідація не дає зберегти без обов'язкових полів, тест показує помилку
**Статус:** ⬜

### TC-AS-2.6 Збереження налаштувань доставки з підтвердженням
**Передумова:** `/admin/delivery-settings`, всі провайдери налаштовані
**Кроки:**
1. Змінити параметри одного провайдера
2. Натиснути "Зберегти"
3. Перевірити діалог підтвердження
4. Підтвердити — зміни збережені
5. Оновити сторінку — дані на місці
**Очікуваний результат:** Збереження вимагає підтвердження, дані зберігаються
**Статус:** ⬜

### TC-AS-2.7 Password show/hide для API ключів доставки
**Передумова:** `/admin/delivery-settings`, будь-яка картка провайдера
**Кроки:**
1. Перевірити що поле API ключа за замовчуванням замасковане (тип password)
2. Натиснути кнопку "показати" (іконка ока) — ключ стає видимим
3. Натиснути повторно — ключ знову замаскований
**Очікуваний результат:** Перемикання видимості API ключа працює коректно
**Статус:** ⬜

## 10.3 Налаштування оплати

### TC-AS-3.1 Онлайн: LiqPay
**Передумова:** Увійти як admin, перейти `/admin/payment-settings`
**Кроки:**
1. Знайти картку LiqPay — перевірити перемикач увімкнення
2. Ввести public key та private key (password show/hide для кожного)
3. Перевірити перемикач sandbox-режиму — увімкнути
4. Перевірити webhook URL — натиснути кнопку копіювання
5. Натиснути "Тест підключення" — перевірити результат
6. Зберегти
**Очікуваний результат:** LiqPay налаштований, ключі збережені, webhook URL скопійовано
**Статус:** ⬜

### TC-AS-3.2 Онлайн: Monobank
**Передумова:** `/admin/payment-settings`, картка "Monobank"
**Кроки:**
1. Увімкнути перемикач
2. Ввести API token (password show/hide)
3. Перевірити sandbox toggle
4. Перевірити webhook URL (copyable)
5. Натиснути "Тест" — перевірити результат
6. Зберегти
**Очікуваний результат:** Monobank підключено, тест успішний
**Статус:** ⬜

### TC-AS-3.3 Онлайн: WayForPay
**Передумова:** `/admin/payment-settings`, картка "WayForPay"
**Кроки:**
1. Увімкнути перемикач
2. Ввести merchant account та secret key (password show/hide)
3. Перевірити sandbox toggle
4. Перевірити webhook URL (copyable)
5. Натиснути "Тест" — перевірити результат
6. Зберегти
**Очікуваний результат:** WayForPay підключено, тест пройшов
**Статус:** ⬜

### TC-AS-3.4 Офлайн-методи оплати
**Передумова:** `/admin/payment-settings`, секція "Офлайн-методи"
**Кроки:**
1. Перевірити 3 офлайн-методи (наприклад: Накладений платіж, Оплата при отриманні, Безготівковий рахунок)
2. Для кожного: перевірити перемикач увімкнення
3. Увімкнути "Накладений платіж", вимкнути "Безготівковий рахунок"
4. Зберегти
**Очікуваний результат:** Стан перемикачів офлайн-методів збережено
**Статус:** ⬜

### TC-AS-3.5 Збереження налаштувань оплати з підтвердженням
**Передумова:** `/admin/payment-settings`, зміни внесені
**Кроки:**
1. Натиснути "Зберегти"
2. Перевірити діалог підтвердження
3. Підтвердити
4. Оновити сторінку — всі налаштування на місці
**Очікуваний результат:** Збереження вимагає підтвердження, дані зберігаються коректно
**Статус:** ⬜

### TC-AS-3.6 Webhook URL копіювання
**Передумова:** `/admin/payment-settings`, будь-який онлайн-провайдер
**Кроки:**
1. Знайти поле "Webhook URL"
2. Натиснути кнопку копіювання
3. Перевірити що URL скопійовано в буфер обміну
4. Вставити — перевірити коректність URL
**Очікуваний результат:** Webhook URL копіюється одним кліком
**Статус:** ⬜

### TC-AS-3.7 Sandbox-режим оплати
**Передумова:** `/admin/payment-settings`, LiqPay налаштований
**Кроки:**
1. Увімкнути sandbox toggle
2. Зберегти
3. На фронті: оформити тестове замовлення з LiqPay — перевірити що використовується тестове середовище
4. Вимкнути sandbox — зберегти
**Очікуваний результат:** Sandbox-режим перемикає на тестове середовище платіжної системи
**Статус:** ⬜

## 10.4 SMTP

### TC-AS-4.1 Налаштування SMTP
**Передумова:** Увійти як admin, перейти `/admin/smtp-settings`
**Кроки:**
1. Ввести хост: `smtp.gmail.com`
2. Ввести порт: `587`
3. Ввести користувача: `shop@example.com`
4. Ввести пароль (перевірити кнопку show/hide)
5. Ввести email відправника: `noreply@example.com`
6. Зберегти
**Очікуваний результат:** SMTP налаштування збережені
**Статус:** ⬜

### TC-AS-4.2 Тестова відправка SMTP
**Передумова:** `/admin/smtp-settings`, налаштування збережені
**Кроки:**
1. Ввести email для тесту: `test@example.com`
2. Натиснути "Надіслати тест"
3. Перевірити повідомлення про успішну відправку
4. Перевірити пошту — тестовий лист отримано
**Очікуваний результат:** Тестовий лист надіслано через налаштований SMTP
**Статус:** ⬜

## 10.6 Купони

### TC-AS-6.1 Створення купона (всі поля)
**Передумова:** Увійти як admin, перейти `/admin/coupons`, натиснути "Додати купон"
**Кроки:**
1. Ввести код: `SPRING25`
2. Обрати тип зі списку: процентна знижка (також доступні: фіксована знижка, безкоштовна доставка)
3. Ввести значення: `25`%
4. Мінімальна сума замовлення: `500` грн
5. Максимальний розмір знижки (cap): `300` грн
6. Ліміт використань (загальний): `100`
7. Ліміт на одного користувача: `1`
8. Дійсний з: сьогодні
9. Дійсний до: через місяць
10. Перевірити перемикач "Активний" — увімкнено
11. Зберегти
**Очікуваний результат:** Купон створено з усіма параметрами, відображається у списку
**Статус:** ⬜

### TC-AS-6.2 Редагування купона
**Передумова:** `/admin/coupons`, є створений купон
**Кроки:**
1. Натиснути "Редагувати" на купоні
2. Змінити значення знижки з 25% на 15%
3. Зберегти
4. Перевірити що зміни відображаються у списку
**Очікуваний результат:** Купон оновлено, нове значення збережене
**Статус:** ⬜

### TC-AS-6.3 Видалення купона
**Передумова:** `/admin/coupons`, є купон
**Кроки:**
1. Натиснути "Видалити" → підтвердити
2. Перевірити що купон зник зі списку
**Очікуваний результат:** Купон видалено
**Статус:** ⬜

### TC-AS-6.4 Застосування купона при checkout
**Передумова:** Кошик з сумою > 500 грн, купон `SPRING25` активний
**Кроки:**
1. На сторінці checkout ввести `SPRING25`
2. Натиснути "Застосувати"
3. Перевірити що сума зменшилась на 25% (але не більше 300 грн cap)
4. Перевірити що знижка відображається у деталях замовлення
**Очікуваний результат:** Знижка застосована з урахуванням максимального обмеження
**Статус:** ⬜

## 10.7 Персональні ціни

### TC-AS-7.1 Додавання персональної ціни
**Передумова:** Увійти як admin, перейти `/admin/personal-prices`
**Кроки:**
1. Натиснути "Додати"
2. У модальному вікні: ввести ID користувача
3. Ввести ID товару або категорії
4. Ввести знижку у %: `15`
5. Або вказати фіксовану ціну: `450` грн
6. Задати період: дійсний з / дійсний до
7. Зберегти
**Очікуваний результат:** Персональна ціна створена, відображається у списку
**Статус:** ⬜

### TC-AS-7.2 Редагування персональної ціни
**Передумова:** `/admin/personal-prices`, є запис
**Кроки:**
1. Натиснути "Редагувати" на записі
2. Змінити знижку з 15% на 20%
3. Зберегти
**Очікуваний результат:** Персональна ціна оновлена
**Статус:** ⬜

### TC-AS-7.3 Видалення персональної ціни з підтвердженням
**Передумова:** `/admin/personal-prices`, є запис
**Кроки:**
1. Натиснути "Видалити"
2. Перевірити діалог підтвердження
3. Підтвердити — запис видалено
**Очікуваний результат:** Видалення вимагає підтвердження, запис видалено
**Статус:** ⬜

### TC-AS-7.4 Пагінація персональних цін
**Передумова:** `/admin/personal-prices`, > 20 записів
**Кроки:**
1. Перевірити що відображається перша сторінка (наприклад, 20 записів)
2. Натиснути на другу сторінку — нові записи завантажились
3. Повернутися на першу — ті самі записи
**Очікуваний результат:** Пагінація працює коректно

## 10.14 2FA для адмінів

### TC-AS-14.1 Увімкнення 2FA
**Передумова:** Увійти як admin, перейти `/admin/setup-2fa`
**Кроки:**
1. Натиснути "Увімкнути 2FA"
2. Перевірити що з'явився QR-код
3. Відсканувати QR-код додатком автентифікації (Google Authenticator / Authy)
4. Ввести 6-значний код для верифікації
5. Підтвердити
**Очікуваний результат:** 2FA увімкнено, верифікація пройшла успішно
**Статус:** ⬜

### TC-AS-14.2 Backup-коди 2FA
**Передумова:** 2FA щойно увімкнено
**Кроки:**
1. Перевірити що відображається список backup-кодів (мінімум 8 кодів)
2. Зберегти коди (скопіювати або завантажити)
3. Перевірити що є попередження про одноразовість кодів
**Очікуваний результат:** Backup-коди відображаються, доступні для збереження
**Статус:** ⬜

### TC-AS-14.3 Вимкнення 2FA
**Передумова:** 2FA увімкнено для адмін-акаунту
**Кроки:**
1. Перейти `/admin/setup-2fa`
2. Натиснути "Вимкнути 2FA"
3. Ввести поточний 6-значний код для підтвердження
4. Підтвердити
5. Вийти та увійти — 2FA код не запитується
**Очікуваний результат:** 2FA вимкнено, вхід без другого фактора
**Статус:** ⬜

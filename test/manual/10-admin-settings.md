# 10 — Адмін: налаштування системи

---

## 10.1 Загальні налаштування

### TC-AS-1.1 Редагування загальних
**Передумова:** `/admin/settings`
**Кроки:**
1. Перевірити поля: назва сайту, опис, телефон, email, адреса
2. Змінити телефон → зберегти
3. Перевірити що телефон оновився в хедері сайту
**Очікуваний результат:** Зміни відображаються на сайті
**Статус:** ⬜

---

## 10.2 Налаштування оплати

### TC-AS-2.1 Перевірка конфігурації
**Передумова:** `/admin/payment-settings`
**Кроки:**
1. Перевірити що є конфіг для: LiqPay, Monobank, WayForPay
2. Перевірити що ключі (частково замасковані) відображаються
3. Натиснути "Тест підключення" (якщо є)
**Очікуваний результат:** Статус підключення кожного провайдера
**Статус:** ⬜

---

## 10.3 Налаштування доставки

2. Змінити тариф → зберегти

**Статус:** ⬜

## 10.6 Купони та знижки

### TC-AS-6.1 Створення купона
**Передумова:** `/admin/coupons`
**Кроки:**
1. "Додати купон"
2. Код: `TEST10`
3. Тип: процентна знижка, 10%
4. Мін. сума: 500 грн
5. Дійсний до: через місяць
6. Зберегти
**Очікуваний результат:** Купон створено
**Статус:** ⬜

### TC-AS-6.2 Застосування купона при checkout
**Передумова:** Кошик з сумою > 500 грн
**Кроки:**
1. На сторінці checkout ввести `TEST10`
2. Натиснути "Застосувати"
3. Перевірити що сума зменшилась на 10%
**Очікуваний результат:** Знижка застосована
**Статус:** ⬜

## 10.10 Лояльність, доставка, 2FA та інтеграції

### TC-AS-10.1 Налаштування лояльності
**Передумова:** Увійти як admin, перейти `/admin/loyalty`
**Кроки:**
1. Перевірити правила нарахування балів (за замовлення, за реєстрацію, за відгук)
2. Перевірити правила списання балів (мін. сума, макс. % оплати балами)
3. Змінити правило нарахування — зберегти
4. Перевірити що зміни впливають на фронт (баланс лояльності в кабінеті)
**Очікуваний результат:** Правила нарахування та списання балів лояльності налаштовуються та працюють
**Статус:** ⬜

### TC-AS-10.2 Налаштування палетної доставки
**Передумова:** Увійти як admin, перейти `/admin/pallet-delivery`
**Кроки:**
1. Перевірити налаштування: максимальна вага палети, зони доставки, тарифи
2. Змінити тариф для зони — зберегти
3. Додати нову зону доставки — зберегти
4. Перевірити що тарифи відображаються при оформленні замовлення з відповідною вагою
**Очікуваний результат:** Налаштування палетної доставки зберігаються та впливають на розрахунок вартості
**Статус:** ⬜

### TC-AS-10.3 2FA для адмінів
**Передумова:** Увійти як admin, перейти `/admin/setup-2fa`
**Кроки:**
1. Натиснути "Увімкнути 2FA"
2. Відсканувати QR-код додатком автентифікації
3. Ввести 6-значний код для підтвердження
4. Зберегти backup-коди
5. Вийти та увійти знову — перевірити що запитується 2FA код
**Очікуваний результат:** 2FA для адмін-акаунту налаштовано, при вході запитується код
**Статус:** ⬜

### TC-AS-10.4 Marketplace інтеграції
**Передумова:** Увійти як admin, перейти `/admin/marketplaces`
**Кроки:**
1. Перевірити список доступних маркетплейсів (Rozetka, Prom, тощо)
2. Натиснути "Підключити" на одному з маркетплейсів
3. Ввести API ключ / облікові дані
4. Зберегти та перевірити статус підключення
5. Натиснути "Синхронізувати ціни" — перевірити результат
**Очікуваний результат:** Маркетплейс підключено, синхронізація цін працює, статус відображається
**Статус:** ⬜

### TC-AS-10.5 Dashboard налаштування
**Передумова:** Увійти як admin, перейти `/admin/dashboard/settings`
**Кроки:**
1. Перевірити список доступних віджетів (KPI, графіки, таблиці)
2. Додати новий віджет на дашборд
3. Змінити розташування віджетів (drag & drop)
4. Видалити віджет
5. Зберегти налаштування
6. Перейти на `/admin` — перевірити що дашборд оновився
**Очікуваний результат:** Віджети додаються, переміщуються та видаляються, розташування зберігається
**Статус:** ⬜

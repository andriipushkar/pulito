# 04 — Особистий кабінет користувача

---

## 4.1 Дашборд акаунту

### TC-UA-1.1 Головна кабінету
**Передумова:** Увійти, перейти `/account`
**Кроки:**
1. Перевірити наявність:
   - Привітання з ім'ям
   - Останні замовлення
   - Частопокупні товари
   - Баланс лояльності
2. Навігація в бічному меню працює
**Очікуваний результат:** Дашборд з основною інформацією
**Статус:** ⬜

---

## 4.2 Замовлення

### TC-UA-2.1 Список замовлень
**Передумова:** `/account/orders`
**Кроки:**
1. Перевірити список замовлень з:
   - Номер замовлення
   - Дата
   - Статус (кольоровий badge)
   - Сума
2. Перевірити пагінацію (якщо > 20)
3. Перевірити фільтр за статусом
**Очікуваний результат:** Замовлення відображаються з фільтрацією
**Статус:** ⬜

### TC-UA-2.2 Деталі замовлення
**Кроки:**
1. Натиснути на замовлення → `/account/orders/[id]`
2. Перевірити:
   - Список товарів з цінами
   - Статус доставки
   - Номер ТТН (якщо є)
   - Спосіб оплати
   - Кнопка "Завантажити рахунок" (якщо є)
**Очікуваний результат:** Повна інформація про замовлення
**Статус:** ⬜

### TC-UA-2.3 Повторне замовлення
**Кроки:**
1. На сторінці замовлення натиснути "Замовити знову"
2. Перевірити що товари додались в кошик
**Очікуваний результат:** Товари з замовлення в кошику
**Статус:** ⬜

### TC-UA-2.4 QR-код замовлення
**Кроки:**
1. На сторінці замовлення знайти QR-код або кнопку для нього
**Очікуваний результат:** QR-код з інформацією про замовлення
**Статус:** ⬜

---

## 4.3 Адреси

### TC-UA-3.1 Додавання адреси
**Передумова:** `/account/addresses`
**Кроки:**
1. Натиснути "Додати адресу"
2. Заповнити:
   - Назва: `Дім`
   - Місто: `Київ`
   - Адреса: `вул. Хрещатик 1`
   - Телефон: `+380991234567`
3. Зберегти
**Очікуваний результат:** Адреса з'явилась у списку
**Статус:** ⬜

### TC-UA-3.2 Редагування адреси
**Кроки:**
1. Натиснути "Редагувати" на збереженій адресі
2. Змінити назву на `Офіс`
3. Зберегти
**Очікуваний результат:** Назва оновлена
**Статус:** ⬜

### TC-UA-3.3 Видалення адреси
**Кроки:**
1. Натиснути "Видалити"
2. Підтвердити
**Очікуваний результат:** Адреса видалена зі списку
**Статус:** ⬜

## 4.4 Налаштування профілю

### TC-UA-4.1 Зміна імені та телефону
**Передумова:** `/account/settings`
**Кроки:**
1. Змінити ім'я на `Тест Оновлений`
2. Змінити телефон на `+380997776655`
3. Зберегти
4. Оновити сторінку
**Очікуваний результат:** Дані збережені та відображаються
**Статус:** ⬜

### TC-UA-4.2 Зміна паролю
**Передумова:** `/account/security`
**Тестові дані:**
- Поточний: `Admin123!`
- Новий: `NewPass456!`
- Підтвердження: `NewPass456!`
**Кроки:**
1. Заповнити поля
2. Зберегти
3. Вийти та увійти з новим паролем
4. **Повернути пароль назад на `Admin123!`**
**Очікуваний результат:** Пароль змінено, вхід з новим паролем працює
**Статус:** ⬜

### TC-UA-4.3 Історія входів
**Передумова:** `/account/security`
**Кроки:**
1. Знайти секцію "Історія входів"
2. Перевірити записи: дата, IP, браузер
**Очікуваний результат:** Список останніх входів
**Статус:** ⬜

## 4.9 Додаткові функції кабінету

### TC-UA-9.1 Експорт персональних даних (GDPR)
**Передумова:** Увійти, перейти `/account`
**Кроки:**
1. Знайти кнопку "Експортувати дані" (GDPR)
2. Натиснути кнопку
3. Дочекатись генерації файлу
4. Завантажити файл
**Очікуваний результат:** Завантажено файл (JSON або ZIP) з персональними даними: профіль, замовлення, адреси, відгуки
**Статус:** ⬜

### TC-UA-9.2 Історія пошуку
**Передумова:** Увійти, перейти `/me/search-history`
**Кроки:**
1. Перевірити що відображається список попередніх пошукових запитів
2. Перевірити дати та тексти запитів
3. Натиснути "Очистити історію"
4. Підтвердити
**Очікуваний результат:** Історія пошуку відображається, після очищення список порожній
**Статус:** ⬜

### TC-UA-9.3 Сторінка фінансів
**Передумова:** Увійти, перейти `/account/finance`
**Кроки:**
1. Перевірити історію платежів: дата, сума, статус, метод оплати
2. Перевірити фільтрацію за періодом
3. Перевірити загальну статистику витрат
**Очікуваний результат:** Історія платежів відображається коректно з фільтрацією
**Статус:** ⬜

### TC-UA-9.4 Сторінка статусу
**Передумова:** Увійти, перейти `/account/status`
**Кроки:**
1. Перевірити поточний рівень користувача
2. Перевірити наявні бейджі
3. Перевірити прогрес до наступного рівня (якщо є)
**Очікуваний результат:** Відображається рівень, бейджі та прогрес
**Статус:** ⬜

### TC-UA-9.5 Менеджер акаунту
**Передумова:** Увійти як оптовий користувач, перейти `/account/manager`
**Кроки:**
1. Перевірити інформацію про персонального менеджера: ім'я, телефон, email
2. Перевірити можливість зв'язатись з менеджером (кнопка/посилання)
**Очікуваний результат:** Інформація про менеджера відображається (для оптовиків)
**Статус:** ⬜

### TC-UA-9.6 Повернення
**Передумова:** Увійти, перейти `/me/returns`
**Кроки:**
1. Натиснути "Створити запит на повернення"
2. Обрати замовлення та товар для повернення
3. Вказати причину повернення
4. Додати фото (якщо потрібно)
5. Відправити запит
**Очікуваний результат:** Запит на повернення створено, відображається у списку з відповідним статусом
**Статус:** ⬜

### TC-UA-9.7 Нещодавно переглянуті
**Передумова:** Увійти, переглянути кілька товарів, перейти `/me/recently-viewed`
**Кроки:**
1. Перевірити що відображається список нещодавно переглянутих товарів
2. Перевірити що товари відсортовані за датою перегляду (найновіші першими)
3. Натиснути на товар — перевірити перехід на сторінку товару
**Очікуваний результат:** Список нещодавно переглянутих товарів відображається коректно
**Статус:** ⬜

## 4.11 Прайс-листи (B2B)

### TC-UA-11.1 Завантаження роздрібного прайсу
**Передумова:** Увійти, перейти `/account/pricelist`
**Кроки:**
1. Перевірити наявність кнопки "Роздрібний прайс-лист"
2. Натиснути → завантажити PDF
3. Відкрити — перевірити: таблиця з кодами, назвами, цінами, наявністю
**Очікуваний результат:** PDF з роздрібними цінами завантажений
**Статус:** ⬜

### TC-UA-11.2 Завантаження оптового прайсу (для оптовиків)
**Передумова:** Увійти як wholesaler, `/account/pricelist`
**Кроки:**
1. Перевірити що є кнопка "Оптовий прайс-лист" (тільки для оптовиків/менеджерів/адмінів)
2. Завантажити PDF
3. Перевірити що ціни оптові (менші за роздрібні)
**Очікуваний результат:** PDF з оптовими цінами, кнопка доступна тільки відповідним ролям
**Статус:** ⬜

### TC-UA-11.3 Прайс недоступний для звичайного юзера
**Передумова:** Увійти як звичайний (retail) юзер
**Кроки:**
1. Перейти на `/account/pricelist`
2. Перевірити що кнопка "Оптовий прайс-лист" НЕ доступна або сторінка обмежена
**Очікуваний результат:** Оптовий прайс недоступний для роздрібних користувачів
**Статус:** ⬜

## 4.12 Замовлення — деталізація

### TC-UA-12.1 Трекінг доставки з кабінету
**Передумова:** Замовлення зі статусом "Відправлено" та ТТН
**Кроки:**
1. `/account/orders/[id]` → натиснути "Відстежити"
2. Перевірити статус відправлення (Нова Пошта / Укрпошта)
**Очікуваний результат:** Поточний статус посилки з етапами
**Статус:** ⬜

### TC-UA-12.2 Завантаження рахунку з кабінету
**Кроки:**
1. `/account/orders/[id]` → натиснути "Завантажити рахунок"
2. Перевірити PDF: branded хедер, таблиця товарів, підсумок
**Очікуваний результат:** PDF рахунок-фактура
**Статус:** ⬜

### TC-UA-12.3 Скасування замовлення клієнтом
**Передумова:** Замовлення зі статусом "Нове" або "В обробці"
**Кроки:**
1. `/account/orders/[id]` → натиснути "Скасувати замовлення"
2. Підтвердити
**Очікуваний результат:** Статус змінено на "Скасовано", залишки повернулись
**Статус:** ⬜

### TC-UA-12.4 Спроба скасування відправленого замовлення
**Передумова:** Замовлення зі статусом "Відправлено"
**Кроки:**
1. Перевірити що кнопка "Скасувати" НЕ доступна
**Очікуваний результат:** Скасування неможливе після відправки
**Статус:** ⬜

### TC-UA-12.5 Частопокупні товари
**Передумова:** Увійти, `/account` (дашборд)
**Кроки:**
1. Перевірити блок "Часто покупаєте"
2. Натиснути "В кошик" на одному з товарів
**Очікуваний результат:** Товари на основі історії, швидке додавання в кошик
**Статус:** ⬜

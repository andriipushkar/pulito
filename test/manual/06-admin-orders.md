# 06 — Адмін: управління замовленнями

---

## 6.1 Список замовлень

### TC-AO-1.1 Перегляд списку
**Передумова:** Увійти як admin, перейти `/admin/orders`
**Кроки:**
1. Перевірити таблицю: №, дата, клієнт, статус, сума, оплата
2. Перевірити кольорові бейджі статусів
3. Перевірити пагінацію
**Очікуваний результат:** Всі замовлення відображаються
**Статус:** ⬜

### TC-AO-1.2 Фільтрація замовлень
**Кроки:**
1. Фільтр за статусом → "Нове замовлення"
2. Фільтр за датою → останній тиждень
3. Пошук за номером замовлення
4. Пошук за телефоном клієнта
**Очікуваний результат:** Кожний фільтр звужує результати
**Статус:** ⬜

### TC-AO-1.3 Експорт замовлень
**Кроки:**
1. Натиснути "Експорт" → Excel
2. Перевірити що файл завантажився
3. Відкрити файл — дані коректні
**Очікуваний результат:** Excel файл з замовленнями
**Статус:** ⬜

---

## 6.2 Деталі замовлення

### TC-AO-2.1 Перегляд деталей
**Кроки:**
1. Клікнути на замовлення → `/admin/orders/[id]`
2. Перевірити: товари, ціни, контакти, доставка, оплата, коментарі
**Очікуваний результат:** Повна інформація
**Статус:** ⬜

### TC-AO-2.2 Зміна статусу
**Кроки:**
1. На деталях замовлення обрати новий статус (наприклад "Підтверджено")
2. Додати коментар: `Підтверджено менеджером`
3. Зберегти
4. Перевірити що статус змінився
5. Перевірити історію статусів
**Очікуваний результат:** Статус оновлений, запис в історії
**Статус:** ⬜

### TC-AO-2.3 Додавання коментаря
**Кроки:**
1. В полі коментарів написати: `Зателефонувати клієнту`
2. Натиснути "Додати"
**Очікуваний результат:** Коментар з датою та ім'ям менеджера
**Статус:** ⬜

### TC-AO-2.4 Генерація рахунку-фактури
**Кроки:**
1. Натиснути "Генерувати рахунок" або "Завантажити PDF"
2. Перевірити що PDF завантажився
3. Відкрити — перевірити: хедер з логотипом, таблиця товарів, суми, footer
**Очікуваний результат:** Красивий PDF з фірмовим стилем (#2563eb синій хедер)
**Статус:** ⬜

### TC-AO-2.5 Генерація видаткової накладної
**Кроки:**
1. Натиснути "Видаткова накладна"
2. Перевірити PDF: постачальник, отримувач, товари, підписи
**Очікуваний результат:** PDF з двома колонками (постачальник / отримувач)
**Статус:** ⬜

### TC-AO-2.6 Введення ТТН (трекінг)
**Кроки:**
1. В полі ТТН ввести: `20450000000001`
2. Зберегти
3. Натиснути "Відстежити" — перевірити статус від Нової Пошти
**Очікуваний результат:** ТТН збережена, трекінг відображає статус
**Статус:** ⬜

## 6.3 Ручне створення замовлення

### TC-AO-3.1 Створити замовлення вручну
**Кроки:**
1. Натиснути "Нове замовлення"
2. Ввести контакти клієнта
3. Додати товари за кодом
4. Обрати доставку та оплату
5. Зберегти
**Тестові дані:**
- Клієнт: `Ручне Замовлення`
- Телефон: `+380501234567`
- Товар: будь-який код з каталогу, кількість 3
**Очікуваний результат:** Замовлення створено з номером
**Статус:** ⬜

## 6.4 Комерційні пропозиції та повернення

### TC-AO-4.1 Генерація комерційної пропозиції
**Передумова:** Увійти як admin, перейти `/admin/commercial-offer`
**Кроки:**
1. Натиснути "Створити комерційну пропозицію"
2. Обрати клієнта або ввести дані вручну
3. Додати товари з каталогу
4. Вказати умови (знижки, терміни дії)
5. Натиснути "Генерувати PDF"
6. Завантажити та перевірити PDF: хедер, таблиця товарів, умови, підпис
**Очікуваний результат:** PDF комерційної пропозиції згенеровано з фірмовим стилем, дані коректні
**Статус:** ⬜

### TC-AO-4.2 Повернення та скасування
**Передумова:** Увійти як admin, перейти `/admin/returns`
**Кроки:**
1. Перевірити список запитів на повернення: номер, клієнт, товар, причина, статус
2. Натиснути на запит — перевірити деталі
3. Обробити повернення: змінити статус на "Підтверджено"
4. Додати коментар менеджера
5. Зберегти
6. Перевірити що статус оновився
**Очікуваний результат:** Список повернень відображається, обробка повернення працює, статус оновлюється
**Статус:** ⬜

## 6.5 Масові операції з замовленнями

### TC-AO-5.1 Bulk зміна статусу
**Передумова:** `/admin/orders/bulk` або список замовлень
**Кроки:**
1. Обрати кілька замовлень (чекбокси)
2. В dropdown масових дій обрати "Змінити статус → Підтверджено"
3. Підтвердити
**Очікуваний результат:** Всі обрані замовлення отримали статус "Підтверджено"
**Статус:** ⬜

### TC-AO-5.2 Bulk друк накладних
**Кроки:**
1. Обрати кілька замовлень
2. Натиснути "Друк накладних" або "Генерувати PDF"
**Очікуваний результат:** PDF з накладними для всіх обраних замовлень
**Статус:** ⬜

### TC-AO-5.3 Bulk створення ТТН
**Кроки:**
1. Обрати кілька замовлень зі способом доставки "Нова Пошта"
2. Натиснути "Створити ТТН для обраних"
**Очікуваний результат:** ТТН створено для кожного замовлення, номери відображаються
**Статус:** ⬜

### TC-AO-5.4 Bulk експорт обраних
**Кроки:**
1. Обрати замовлення
2. Натиснути "Експорт обраних → Excel"
**Очікуваний результат:** Excel тільки з обраними замовленнями
**Статус:** ⬜

## 6.6 Редагування позицій замовлення

### TC-AO-6.1 Додавання товару до замовлення
**Передумова:** `/admin/orders/[id]`
**Кроки:**
1. Натиснути "Додати товар"
2. Пошук за кодом або назвою
3. Обрати товар, вказати кількість
4. Зберегти
**Очікуваний результат:** Товар додано, сума перерахована
**Статус:** ⬜

### TC-AO-6.2 Зміна кількості в замовленні
**Кроки:**
1. Змінити кількість існуючого товару з 2 на 5
2. Зберегти
**Очікуваний результат:** Кількість оновлена, сума перерахована, залишок на складі оновлений
**Статус:** ⬜

### TC-AO-6.3 Видалення товару із замовлення
**Кроки:**
1. Натиснути "×" біля товару
2. Підтвердити
**Очікуваний результат:** Товар видалено, сума зменшилась, залишок повернувся
**Статус:** ⬜

### TC-AO-6.4 Відправка фото товару клієнту
**Кроки:**
1. На деталях замовлення натиснути "Надіслати фото товару"
2. Обрати фото з галереї товару
**Очікуваний результат:** Фото надіслано в Telegram/Viber клієнту (якщо прив'язано)
**Статус:** ⬜

#zси: Мобільна версія

> Пристрій: Chrome DevTools → iPhone 14 (390px) або реальний смартфон
> Додатково перевірити на 320px (iPhone SE) де зазначено

## 1. НАВІГАЦІЯ ТА ХЕДЕР

### TC-M-1.1 Логотип та перехід на головну
**Передумова:** Відкрити будь-яку сторінку, наприклад `/catalog`
1. Подивитися на логотип у верхньому лівому куті
2. Бейдж "П" — gradient background, rounded-xl, shadow-brand
3. "Поро" + "шок" — градієнтний текст (від primary до primary-light)
4. Натиснути на логотип
5. **Очікуваний результат:** Перехід на головну `/`

### TC-M-1.2 Top Bar — посилання та соцмережі
**Передумова:** Головна сторінка
1. У верхній частині є рядок з "Зв'язок" та "Питання"
2. Іконки соцмереж: Telegram, Viber, Instagram, Facebook, TikTok (розмір 14px)
3. Натиснути "Зв'язок" → `/contacts`
4. Натиснути "Питання" → `/faq`
5. Проскролити вниз (>80px) → TopBar зникає
6. **Очікуваний результат:** TopBar функціональний, ховається при скролі

### TC-M-1.3 Сповіщення в хедері
**Передумова:** Авторизований з непрочитаними сповіщеннями
1. У хедері — між логотипом та кнопкою "Зателефонувати"
2. Іконка дзвіночка (Bell) з червоним бейджем кількості
3. Кількість оновлюється кожні 60 секунд
4. Натиснути → `/account/notifications`
5. **Очікуваний результат:** Перехід на сторінку сповіщень

### TC-M-1.4 Кнопка "Зателефонувати"
**Передумова:** Будь-яка сторінка
1. У хедері — кнопка "Зателефонувати" з іконкою телефону (синій фон primary/10)
2. Натиснути → відкрився номеронабирач з номером магазину (+38 (000) 123-45-67)
3. **Очікуваний результат:** tel: посилання працює

### TC-M-1.5 Пошук — поле під хедером
**Передумова:** Будь-яка сторінка
1. Під хедером — поле пошуку (placeholder "Пошук товарів...")
2. Натиснути на поле → фулскрін режим пошуку (< 640px)
3. При фокусі з'являється клавіатура
4. Ввести "порошок" (мін. 2 символи) → через 300мс з'являються результати:
   - Категорії + товари (мініатюра, назва, код, ціна)
5. Enter → `/catalog?search=порошок`
6. **Очікуваний результат:** Пошук з автопідказками у фулскрін-режимі

### TC-M-1.6 Пошук за кодом
**Передумова:** Поле пошуку
1. Ввести артикул товару → Enter
2. **Очікуваний результат:** Знайдено товар за кодом

### TC-M-1.7 Порожній пошук
1. Ввести "фівапролджфівапролдж" → Enter
2. **Очікуваний результат:** Каталог з іконкою пошуку + "Товарів не знайдено" + посилання скидання

### TC-M-1.8 Таб-бар — структура
**Передумова:** Будь-яка сторінка
1. Внизу — фіксований таб-бар з 5 іконками:
   - Головна
   - Каталог (відкриває MobileMenu)
   - Кошик (центр, підняття вгору, gradient background — акцент)
   - Обране
   - Профіль
2. Glass-ефект (border white/30, shadow, rounded-2xl)
3. **Очікуваний результат:** Таб-бар видимий та стилізований

### TC-M-1.9 Таб-бар — перехід на головну
**Передумова:** Відкрити `/catalog`
1. Натиснути "Головна" в таб-барі
2. **Очікуваний результат:** Перехід на `/`, таб підсвічується синім

### TC-M-1.10 Таб-бар — каталог (bottom sheet)
**Передумова:** Будь-яка сторінка
1. Натиснути "Каталог" у таб-барі
2. **Очікуваний результат:** Відкривається bottom sheet (MobileMenu) з drag handle:
   - "Акції та знижки" (червоний банер зверху)
   - "Весь каталог" з іконкою grid (синій текст)
   - Список батьківських категорій з ChevronRight
   - "Мій кабінет" + "Обране"
   - Контакти: телефон (посилання), "Зворотній зв'язок", "Питання та відповіді"
   - Соцмережі внизу: Telegram, Viber, Instagram, Facebook, TikTok
3. Натиснути на категорію → `/catalog?category=slug`, sheet закрився
4. **Очікуваний результат:** Меню каталогу повне та функціональне

### TC-M-1.11 Таб-бар — кошик з бейджем
**Передумова:** У кошику є товари
1. Кнопка "Кошик" у таб-барі — центральна, піднята, з gradient
2. Червоний бейдж із кількістю товарів
3. Натиснути → `/cart`
4. **Очікуваний результат:** Перехід у кошик

### TC-M-1.12 Таб-бар — обране з бейджем
**Передумова:** Авторизований, є товари в обраному
1. "Обране" — синій бейдж із кількістю
2. Натиснути → `/account/wishlist`
3. **Очікуваний результат:** Перехід до обраного

### TC-M-1.13 Таб-бар — профіль зі сповіщеннями
**Передумова:** Авторизований з непрочитаними сповіщеннями
1. "Профіль" — червоний бейдж із кількістю сповіщень
2. Натиснути → `/account`
3. Неавторизований → `/auth/login`
4. **Очікуваний результат:** Правильний маршрут залежно від стану

### TC-M-1.14 Таб-бар — приховування при скролі
**Передумова:** Головна або каталог
1. Швидко скролити вниз
2. **Очікуваний результат:** Таб-бар плавно ховається (translate-y-full) з гістерезисом
3. Скролити вгору
4. **Очікуваний результат:** Таб-бар плавно з'являється

### TC-M-1.15 Bottom sheet — закриття
**Передумова:** Відкритий bottom sheet каталогу
1. Натиснути X (правий верхній кут) → закрився
2. Відкрити знову → натиснути на затемнену область → закрився
3. Відкрити знову → Escape → закрився
4. Body scroll заблокований поки sheet відкритий
5. Focus trap працює (Tab не виходить за межі sheet)
6. **Очікуваний результат:** Всі методи закриття працюють, доступність забезпечена

### TC-M-1.16 Таб-бар на 320px (iPhone SE)
**Передумова:** Chrome DevTools → 320px
1. Всі 5 підписів видимі, не обрізаються
2. Іконки не накладаються одна на одну
3. Кнопка кошика (центральна) не виходить за межі
4. **Очікуваний результат:** Таб-бар коректний на вузькому екрані

## 2. ГОЛОВНА СТОРІНКА

### TC-M-2.1 Банер — відображення
**Передумова:** Головна сторінка
1. Банер має пропорції 5:2 (aspect-[5/2])
2. Зображення заповнює весь контейнер (object-cover)
3. rounded-3xl та shadow
4. Текстовий блок: glass-dark стиль, заголовок + CTA-кнопка
5. Підзаголовок прихований на мобільній (hidden, block на sm+)
6. Прогрес-індикатори внизу — тонкі лінії (НЕ крапки)
7. **Очікуваний результат:** Банер відображається коректно

### TC-M-2.2 Банер — свайп та автопрокрутка
**Передумова:** Головна, банер видимий
1. Зачекати 5с → банер автоматично переключився, прогрес-бар анімувався
2. Свайпнути банер вліво → наступний слайд
3. Свайпнути вправо → попередній слайд
4. **Очікуваний результат:** Touch-свайп працює плавно

### TC-M-2.3 Банер — клік
**Передумова:** Головна
1. Натиснути на банер (на CTA-кнопку або зображення)
2. **Очікуваний результат:** Перехід за посиланням (buttonLink)

### TC-M-2.4 Категорії — сітка (тільки мобільна/планшет)
**Передумова:** Головна, блок категорій (hidden на lg+)
1. **Мобільна:** Горизонтальний скрол (stories-стиль, 72px карточки)
2. **Планшет (sm+):** Горизонтальний скрол, більші карточки (88-96px, 120-130px wide)
3. Кожна карточка: іконка/зображення + назва (2-line clamp) + кількість товарів
4. Натиснути → `/catalog?category=slug`
5. **Очікуваний результат:** Категорії відображаються, посилання працюють

### TC-M-2.5 Каруселі товарів
**Передумова:** Головна, є акційні/нові/популярні товари
1. Блоки: "Акційні товари", "Новинки", "Хіти продажів" (якщо є дані)
2. Заголовок + "Дивитися все" (посилання)
3. Свайп каруселі вліво/вправо
4. Натиснути на картку → `/product/slug`
5. Натиснути "В кошик" на картці → toast, бейдж кошика у таб-барі +1
6. **Очікуваний результат:** Каруселі працюють з touch

### TC-M-2.6 Блок брендів — автоскрол
**Передумова:** Головна, "Наші бренди"
1. Заголовок із золотим підкресленням
2. Бренди прокручуються горизонтально автоматично (30s, infinite)
3. Натиснути на бренд → `/catalog?brand=slug`
4. **Очікуваний результат:** Автоскрол та посилання працюють

### TC-M-2.7 SEO-текст
**Передумова:** Головна, проскролити до кінця
1. Блок "Інтернет-магазин побутової хімії Порошок"
2. Gradient фон (primary-50/60 → white) + тонка рамка
3. Текст max-w-3xl, читабельний
4. **Очікуваний результат:** SEO-блок оформлений та не розтягнутий

### TC-M-2.8 Нещодавно переглянуті
**Передумова:** Переглянути 2-3 товари, повернутися на головну
1. Блок "Нещодавно переглянуті" з каруселлю
2. Якщо нічого не переглядали — блок не показується (self-hiding)
3. **Очікуваний результат:** Відображає реальну історію перегляду

### TC-M-2.9 Порожні блоки не відображаються
**Передумова:** Головна
1. Блоки керуються з БД (enabled/disabled)
2. Якщо немає акційних товарів — блок не рендериться
3. Між видимими блоками немає порожніх просторів
4. **Очікуваний результат:** Тільки блоки з контентом, без "дірок"

## 3. КАТАЛОГ

### TC-M-3.1 Перегляд каталогу
**Передумова:** `/catalog`
1. Товари у сітці 2 колонки (мобільна)
2. Кожна картка: фото (blur placeholder, lazy loading), бейджі, назва (2-line clamp), ціна, статус, "В кошик"
3. CatalogToolbar зверху: "Знайдено: N товарів" + кнопка фільтрів + сортування
4. **Очікуваний результат:** Каталог рендериться коректно

### TC-M-3.2 Фільтри — mobile sheet
**Передумова:** `/catalog`
1. Натиснути кнопку фільтрів у CatalogToolbar (іконка Filter)
2. **Очікуваний результат:** Відкривається MobileFilterSheet з:
   - Чекбокси категорій (з кількістю товарів)
   - Слайдер ціни (0-10000 ₴, крок 50) + ручне введення min/max
   - "Тільки акційні" (чекбокс)
   - "В наявності" (чекбокс)
   - "Застосувати" + "Скинути"
3. Бейдж на кнопці фільтрів показує кількість активних фільтрів
4. **Очікуваний результат:** Фільтри працюють з mobile sheet

### TC-M-3.3 Фільтр за категорією
1. Обрати категорію → "Застосувати"
2. **Очікуваний результат:** Список оновився, URL `?category=slug`

### TC-M-3.4 Фільтр за ціною
1. "від" 50, "до" 200 → "Застосувати"
2. **Очікуваний результат:** Тільки товари 50-200 грн

### TC-M-3.5 Фільтр "В наявності"
1. Увімкнути "В наявності" → "Застосувати"
2. **Очікуваний результат:** Товари "Немає в наявності" зникли

### TC-M-3.6 Фільтр "Акції"
1. Увімкнути "Тільки акційні" → "Застосувати"
2. **Очікуваний результат:** Тільки акційні товари

### TC-M-3.7 Скидання фільтрів
**Передумова:** Застосовано кілька фільтрів
1. "Скинути" → всі фільтри знято
2. **Очікуваний результат:** Чистий каталог

### TC-M-3.8 Сортування
1. Dropdown з 5 опціями:
   - За популярністю (default)
   - Новинки
   - Від дешевих
   - Від дорогих
   - За назвою
2. Вибір скидає пагінацію
3. **Очікуваний результат:** Сортування працює

### TC-M-3.9 Пагінація
**Передумова:** 20+ товарів
1. Проскролити до кінця → натиснути "2"
2. **Очікуваний результат:** Друга сторінка

## 4. СТОРІНКА ТОВАРУ

### TC-M-4.1 Відкриття товару
**Передумова:** Каталог або карусель
1. Натиснути на картку товару
2. **Очікуваний результат:** `/product/slug` з повною інформацією

### TC-M-4.2 Галерея зображень — свайп
**Передумова:** Товар з кількома фото
1. Embla-карусель з pagination dots + індикатор "X / Total"
2. Свайпнути вліво → наступне фото
3. Свайпнути вправо → попереднє фото
4. Натиснути на фото → відкрився lightbox (модалка з навігацією)
5. **Очікуваний результат:** Touch-свайп та lightbox працюють

### TC-M-4.3 Інформація про товар
**Передумова:** Сторінка товару
1. Хлібні крихти: Головна > Каталог > Категорія > Назва
2. Назва, код/артикул, бейджі
3. Ціна в glass-style блоці + статус наявності ("В наявності (N шт.)" / "Немає в наявності")
4. Оптова ціна (для wholesaler)
5. **Очікуваний результат:** Повна інформація, все читабельне

### TC-M-4.4 Додавання в кошик
**Передумова:** Товар в наявності
1. Селектор кількості (−/+)
2. "В кошик" (gradient кнопка з іконкою) → toast, бейдж кошика в таб-барі +1
3. Якщо "Немає в наявності" → кнопка disabled
4. **Очікуваний результат:** Додавання працює з підтвердженням

### TC-M-4.5 Додавання в обране
**Передумова:** Будь-який користувач
1. Натиснути "В обране" (серце) → серце заповнилось
2. Натиснути ще раз → порожнє
3. Авторизовані — API, гості — localStorage
4. **Очікуваний результат:** Toggle обраного працює

### TC-M-4.6 Таби інформації
1. "Опис" → текстовий опис (HTML)
2. "Характеристики" → таблиця (HTML)
3. "Доставка та оплата" → hardcoded список
4. **Очікуваний результат:** Таби перемикаються

### TC-M-4.7 Floating buy bar
**Передумова:** Сторінка товару
1. Проскролити вниз поки кнопка "В кошик" зникне з видимої області
2. **Очікуваний результат:** Sticky панель над таб-баром (glass-nav, rounded-2xl):
   - Назва товару + ціна + "Купити"
3. Натиснути → товар додано
4. Прокрутити назад → панель зникає
5. **Очікуваний результат:** Floating bar не перекриває таб-бар

### TC-M-4.8 "Купують разом"
**Передумова:** Товар з рекомендаціями
1. Горизонтальний скрол карток (150px ширина)
2. До 6 товарів (API `/api/v1/products/{id}/recommendations`)
3. "В кошик" → додано
4. **Очікуваний результат:** Рекомендації з touch-скролом

### TC-M-4.9 Схожі товари та нещодавно переглянуті
1. Якщо є категорія — "Схожі товари" (карусель, свайп)
2. "Нещодавно переглянуті" (self-hiding)
3. **Очікуваний результат:** Додаткові блоки працюють

## 5. КОШИК

### TC-M-5.1 Порожній кошик
1. `/cart` → іконка кошика + "Додайте товари з каталогу" + посилання в каталог
2. **Очікуваний результат:** Порожній стан

### TC-M-5.2 Кошик з товарами
**Передумова:** 2+ товари
1. Заголовок "Кошик (N)" + "Очистити кошик"
2. Мобільний layout (без header row): фото (80x80) + назва-посилання + код + QuantitySelector
3. Загальна сума правильна
4. **Очікуваний результат:** Кошик читабельний на мобільній

### TC-M-5.3 Зміна кількості
1. "+" → кількість +1, сума оновилась
2. "-" → кількість −1 (мінімум 1)
3. **Очікуваний результат:** Суми перераховуються миттєво

### TC-M-5.4 Свайп-видалення (тільки мобільна)
**Передумова:** Кошик з товаром
1. Свайпнути рядок товару вліво
2. **Очікуваний результат:** Відкрилась червона зона з іконкою кошика
3. Продовжити свайп далі вліво (більше 160px)
4. **Очікуваний результат:** Товар видалено з анімацією (500px translate)
5. Альтернативно: свайпнути трохи (80px) → з'явилась кнопка → натиснути
6. **Очікуваний результат:** Товар видалено
7. Свайп менше 80px → повернення на місце (snap back)
8. **Очікуваний результат:** Товар залишився

### TC-M-5.5 Очищення кошика
1. "Очистити кошик" → всі товари видалено

# Імпорт та Експорт Товарів

## Огляд

Система підтримує повний round-trip імпорт/експорт товарів через XLSX файли. Один і той самий файл можна скачати, відредагувати в Excel/Google Sheets і завантажити назад.

## Експорт

### Швидкий експорт (тільки ціни)

**URL:** `GET /api/v1/admin/export?type=catalog`

Експортує тільки базові поля: код, назва, категорія, ціни, залишок, статус.

### Повний експорт (з описами та SEO)

**URL:** `GET /api/v1/admin/export?type=products_full`

Включає ВСІ поля товару:

| Колонка            | Опис                         |
| ------------------ | ---------------------------- |
| Код                | Унікальний артикул (SKU)     |
| Назва              | Назва товару                 |
| Категорія          | Назва категорії              |
| Роздрібна ціна     | Ціна для роздрібних клієнтів |
| Ціна: Дрібний опт  | Гуртова ціна tier 1          |
| Ціна: Середній опт | Гуртова ціна tier 2          |
| Ціна: Великий опт  | Гуртова ціна tier 3          |
| Залишок            | Кількість на складі          |
| Акція              | Так/Ні                       |
| Статус             | Активний/Неактивний          |
| Короткий опис      | Короткий текст для карточки  |
| Опис               | Повний HTML-опис             |
| Характеристики     | Текст характеристик          |
| SEO заголовок      | Meta title                   |
| SEO опис           | Meta description             |
| SEO ключові слова  | Meta keywords                |
| Зображення         | Шляхи до файлів (через `;`)  |
| Бейджі             | Типи бейджів (через `,`)     |

### Параметри

| Параметр | Значення                          | За замовчуванням |
| -------- | --------------------------------- | ---------------- |
| `type`   | `products_full`                   | —                |
| `format` | `xlsx` або `csv`                  | `xlsx`           |
| `ids`    | ID через кому (напр. `42,55,100`) | всі активні      |

### Приклади

```bash
# Експорт всіх товарів (XLSX)
GET /api/v1/admin/export?type=products_full

# Експорт одного товару
GET /api/v1/admin/export?type=products_full&ids=42

# Експорт обраних товарів (CSV)
GET /api/v1/admin/export?type=products_full&ids=1,2,3&format=csv
```

### В адмін-панелі

- **Список товарів** → кнопка **"Експорт повний"** — експортує всі товари
- **Сторінка редагування товару** → кнопка **"Експорт XLSX"** — експортує один товар

## Імпорт

### URL

```
POST /api/v1/admin/import/products
```

### Підтримувані формати

- `.xlsx` (Excel 2007+)
- `.xls` (Excel 97-2003)
- `.csv` (з роздільником `,` або `;`)

Максимальний розмір: **10 MB** / **10 000 рядків**

### Формат файлу

Файл повинен мати заголовки в першому рядку. Підтримуються українські та англійські назви колонок:

**Обов'язкові колонки (стандартний формат):**

- `Код` / `code` / `Артикул` / `SKU`
- `Назва` / `name` / `Найменування`
- `Ціна роздріб` / `price` / `Роздрібна ціна`

**Додаткові колонки (автоматично розпізнаються):**

- `Кількість` / `quantity` / `Залишок`
- `Категорія` / `category`
- `Ціна опт` / `wholesale_price` (та tier 2, tier 3)
- `Зображення` / `image_url` — URL для автоматичного завантаження
- `Акція` / `promo` — `Так`/`true`/`1`

**Колонки контенту (для round-trip):**

- `Короткий опис` / `short_description`
- `Опис` / `description`
- `Характеристики` / `specifications`
- `SEO заголовок` / `seo_title`
- `SEO опис` / `seo_description`
- `SEO ключові слова` / `seo_keywords`

### Логіка імпорту

1. Якщо товар з таким **кодом** вже існує → **оновлення**
2. Якщо товару немає → **створення**
3. Якщо категорія не існує → **автоматичне створення**
4. Якщо є URL зображення → **автоматичне завантаження** (SSRF-захист)
5. При зміні ціни → запис в **PriceHistory**
6. Якщо є контент-колонки → створення/оновлення **ProductContent**

### Постачальницький формат

Якщо в файлі **немає колонки "Код"**, система автоматично перемикається на постачальницький формат:

- Код генерується з назви товару (slug)
- Потрібні тільки: `Назва` + `Ціна`

### Попередній перегляд

```
POST /api/v1/admin/import/preview
```

Перевіряє файл без імпорту. Повертає: кількість рядків, формат (standard/supplier), розпізнані колонки.

### Імпорт зображень (ZIP)

```
POST /api/v1/admin/import/images
```

Завантажує ZIP-архів з зображеннями. Файли мають бути названі за кодом товару:

- `SOAP-001.jpg` → товар з кодом `SOAP-001`
- `GEL-002.png` → товар з кодом `GEL-002`

Максимум: **50 MB**

## Round-trip workflow

1. **Експорт**: Адмінка → Товари → "Експорт повний" → скачується `products_full_2026-03-25.xlsx`
2. **Редагування**: Відкрити в Excel/Google Sheets, змінити ціни/описи/SEO
3. **Імпорт**: Адмінка → Імпорт → завантажити той самий файл → товари оновляться

Всі колонки експорту підтримуються при імпорті. Непотрібні колонки (Зображення, Бейджі, Статус) ігноруються при імпорті.

## API Reference

| Endpoint                                  | Method | Призначення                   |
| ----------------------------------------- | ------ | ----------------------------- |
| `/api/v1/admin/export?type=products_full` | GET    | Повний експорт товарів        |
| `/api/v1/admin/export?type=catalog`       | GET    | Швидкий експорт (тільки ціни) |
| `/api/v1/admin/import/products`           | POST   | Імпорт товарів                |
| `/api/v1/admin/import/preview`            | POST   | Попередній перегляд імпорту   |
| `/api/v1/admin/import/images`             | POST   | Імпорт зображень (ZIP)        |
| `/api/v1/admin/import/logs`               | GET    | Історія імпортів              |
| `/api/v1/admin/import/logs/[id]`          | GET    | Деталі імпорту                |

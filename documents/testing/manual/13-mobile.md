# 13 — Мобільна версія та PWA

---

## 13.1 Адаптивність

### TC-MB-1.1 Головна на мобільному

**Передумова:** Відкрити сайт на телефоні (або DevTools → 375px)
**Кроки:**

1. Перевірити що контент не виходить за межі екрану
2. Банери адаптивні
3. Товари в 1-2 колонки
4. Шрифти читабельні
   **Очікуваний результат:** Все адаптивне, горизонтального скролу немає
   **Статус:** ⬜

### TC-MB-1.2 Мобільне меню

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Натиснути "бургер" іконку
2. Перевірити що відкривається бічне меню
3. Навігація по категоріях
4. Закрити меню
   **Очікуваний результат:** Меню плавно відкривається/закривається
   **Статус:** ⬜

### TC-MB-1.3 Каталог на мобільному

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Перейти на `/catalog`
2. Перевірити фільтри (popup або бічна панель)
3. Перевірити пагінацію
   **Очікуваний результат:** Фільтри доступні через кнопку, товари в 2 колонки
   **Статус:** ⬜

### TC-MB-1.4 Checkout на мобільному

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Додати товар → перейти на `/checkout`
2. Пройти всі 4 кроки
3. Перевірити що форми зручні на тач-екрані
   **Очікуваний результат:** Кнопки достатнього розміру, форми зручні
   **Статус:** ⬜

### TC-MB-1.5 Адмінка на мобільному

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Відкрити `/admin` на мобільному
2. Перевірити навігацію
3. Перевірити таблиці (горизонтальний скрол)
   **Очікуваний результат:** Адмінка функціональна на мобільному
   **Статус:** ⬜

## 13.2 PWA

### TC-MB-2.1 Встановлення як додаток

**Передумова:** Chrome на Android або Safari на iOS
**Кроки:**

1. Відкрити сайт
2. Chrome: меню → "Додати на головний екран"
3. iOS: Share → "На Під. екран"
4. Відкрити як додаток
   **Очікуваний результат:** Сайт відкривається в standalone режимі з іконкою
   **Статус:** ⬜

### TC-MB-2.2 Manifest

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Відкрити `/manifest.webmanifest`
2. Перевірити: name, icons, start_url, theme_color
   **Очікуваний результат:** Валідний маніфест
   **Статус:** ⬜

### TC-MB-2.3 Офлайн сторінка

**Передумова:** Chrome DevTools, ширина 390px, або реальний смартфон
**Кроки:**

1. Відкрити сайт
2. Вимкнути інтернет (DevTools → Network → Offline)
3. Спробувати перейти на іншу сторінку
   **Очікуваний результат:** Показується сторінка "Немає з'єднання" замість помилки браузера
   **Статус:** ⬜

## 13.5 Негативні тест-кейси

### TC-MB-5.1 Меню на дуже маленькому екрані 320px (Негативний)

**Передумова:** Chrome DevTools → viewport 320px (старий iPhone SE або аналог)
**Кроки:**

1. Натиснути "бургер" іконку для відкриття бічного меню
2. Перевірити що меню не виходить за межі екрану 320px
3. Перевірити що всі пункти меню читабельні та клікабельні
4. Спробувати навігацію по підкатегоріях — перевірити що вкладені рівні вміщуються
5. Закрити меню — перевірити що контент повертається у нормальний стан
   **Очікуваний результат:** Меню повністю вміщується в 320px без горизонтального скролу. Всі пункти мають touch target >= 44px. Текст не обрізається критично. Вкладені категорії доступні. Закриття працює коректно.
   **Статус:** ⬜

### TC-MB-5.2 Навігація в офлайн-режимі (Негативний)

**Передумова:** Відкрити сайт на мобільному (або DevTools → 390px). Сайт завантажений.
**Кроки:**

1. Вимкнути інтернет (DevTools → Network → Offline)
2. Спробувати перейти на іншу сторінку через меню (наприклад, `/catalog`)
3. Спробувати натиснути на товар
4. Спробувати відкрити кошик
   **Очікуваний результат:** Показується офлайн-сторінка або кешована версія (якщо PWA). Користувач не бачить стандартну помилку браузера "ERR_INTERNET_DISCONNECTED". При поверненні онлайн — сторінка відновлюється без ручного оновлення.
   **Статус:** ⬜

### TC-MB-5.3 Галерея товару з одним зображенням — свайп (Негативний)

**Передумова:** Сторінка товару на мобільному (390px). Товар має лише 1 зображення.
**Кроки:**

1. Спробувати свайпнути вліво/вправо по зображенню товару
2. Перевірити індикатори (точки/лічильник)
   **Очікуваний результат:** Свайп не спрацьовує або зображення повертається на місце (bounce back). Індикатори пагінації (точки) приховані або показують "1/1". Зображення не зникає при свайпі. Помилок у консолі немає.
   **Статус:** ⬜

### TC-MB-5.4 Додавання товару без залишку на мобільному (Негативний)

**Передумова:** Сторінка товару на мобільному (390px). Товар з залишком = 0 (немає в наявності).
**Кроки:**

1. Перевірити кнопку "В кошик" — стан та візуальне оформлення
2. Спробувати натиснути на кнопку
3. Перевірити FloatingBuyBar (sticky bar) при скролі
   **Очікуваний результат:** Кнопка "В кошик" має стан disabled або замінена на "Немає в наявності". Натискання не додає товар у кошик. FloatingBuyBar або прихований, або кнопка в ньому disabled. Бейдж кошика не змінюється.
   **Статус:** ⬜

### TC-MB-5.5 Checkout з невалідним номером телефону (Негативний)

**Передумова:** Мобільний (390px). Товар додано в кошик, перехід на `/checkout`.
**Кроки:**

1. У полі телефону ввести невалідний номер: `12345`
2. Спробувати перейти до наступного кроку
3. Ввести літери замість цифр: `abcdefgh`
4. Ввести занадто довгий номер: `+380501234567890123`
   **Очікуваний результат:** Валідація підсвічує поле червоним з повідомленням "Невірний формат телефону". Перехід на наступний крок заблокований. Літери не вводяться або відхиляються при валідації. Занадто довгий номер обрізається або відхиляється.
   **Статус:** ⬜

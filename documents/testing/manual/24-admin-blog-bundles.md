## 24.1 Список блог-постів

### TC-BB-1.1 Таблиця блог-постів — відображення та структура
**Передумова:** Авторизований адміністратор. Перейти за URL `/admin/blog`. Існує щонайменше 2 статті.
**Тестові дані:** Стаття 1: title «Як обрати засіб для прання», category «porady» (відображається як slug категорії), status `published`, views 142, createdAt 15.03.2026. Стаття 2: title «Новинки весна 2026», category null, status `draft`, views 0, createdAt 10.03.2026.
**Кроки:**
1. Перевірити заголовок сторінки «Блог» (h2, text-xl font-bold)
2. Перевірити наявність кнопки «Категорії» (Button variant=outline, Link на `/admin/blog/categories`) у правому верхньому куті
3. Перевірити наявність кнопки «+ Нова стаття» (Button, Link на `/admin/blog/new`) правіше від «Категорії»
4. Перевірити заголовки стовпців таблиці: «Заголовок», «Категорія», «Статус» (text-center), «Перегляди» (text-right), «Дата», «Дії» (text-right)
5. У рядку статті 1 перевірити: заголовок «Як обрати засіб для прання» — синє посилання (text-primary, hover:underline), яке веде на `/admin/blog/{id}`
6. Перевірити категорію «porady» (text-secondary)
7. Перевірити бейдж статусу «Опубліковано» (bg-green-100 text-green-700, rounded-full, text-center)
8. Перевірити кількість переглядів «142» (text-right, text-secondary)
9. Перевірити дату «15.03.2026» (uk-UA locale, text-secondary)
10. У стовпці «Дії» перевірити посилання «Редагувати» (text-primary, Link на `/admin/blog/{id}`) та кнопку «Видалити» (text-danger)
11. У рядку статті 2 перевірити: категорія «—» (прочерк), бейдж «Чернетка» (bg-gray-100 text-gray-500)
**Очікуваний результат:**
- Таблиця з 6 стовпцями у блоці з рамкою (border, rounded, bg)
- Рядки заголовків — bg-secondary з font-medium
- Маппінг статусів: `published` → «Опубліковано» (зелений бейдж), `draft` → «Чернетка» (сірий бейдж)
- Якщо category порожня — відображається «—»
- Кнопки «Редагувати» (посилання) та «Видалити» (кнопка) у стовпці дій
**Статус:** ⬜

### TC-BB-1.3 Видалення статті — підтвердження
**Передумова:** Авторизований адміністратор на сторінці `/admin/blog`. Існує стаття.
**Тестові дані:** Стаття «Як обрати засіб для прання».
**Кроки:**
1. У рядку статті «Як обрати засіб для прання» натиснути кнопку «Видалити» (text-xs text-danger hover:underline)
2. Перевірити ConfirmDialog (модальне вікно) з variant=danger
3. Перевірити текст повідомлення: «Видалити статтю?»
4. Перевірити кнопки «Скасувати» (variant=outline) та «Підтвердити» (bg-danger, червона)
5. Натиснути «Підтвердити»
6. Дочекатися API DELETE `/api/v1/admin/blog/{id}`
**Очікуваний результат:**
- Toast «Статтю видалено» (toast.success)
- Модальне вікно закривається
- Таблиця перезавантажується, стаття зникає зі списку
**Статус:** ⬜

### TC-BB-1.4 Видалення статті — скасування
**Передумова:** ConfirmDialog видалення відкритий.
**Тестові дані:** Будь-яка стаття.
**Кроки:**
1. Натиснути «Видалити» біля статті
2. У ConfirmDialog натиснути «Скасувати»
**Очікуваний результат:** Модальне вікно закривається. Стаття залишається. API DELETE не викликається.
**Статус:** ⬜

### TC-BB-1.5 Видалення статті — помилка сервера
**Передумова:** Авторизований адміністратор на сторінці `/admin/blog`. API DELETE повертає помилку.
**Тестові дані:** API повертає `{ success: false, error: "Стаття має пов'язані коментарі" }`.
**Кроки:**
1. Натиснути «Видалити» біля статті
2. Підтвердити у ConfirmDialog
3. API повертає помилку
**Очікуваний результат:** Toast помилки з текстом «Стаття має пов'язані коментарі» (або «Помилка видалення» якщо error не повернено)
**Статус:** ⬜

## 24.2 Створення блог-посту

### TC-BB-2.1 Сторінка створення статті — структура форми
**Передумова:** Авторизований адміністратор. Перейти за URL `/admin/blog/new`.
**Тестові дані:** Існує категорія «Поради» (slug: «porady»).
**Кроки:**
1. Перевірити посилання «← Блог» (text-sm, text-primary, hover:underline, Link на `/admin/blog`)
2. Перевірити заголовок «Нова стаття» (h2, text-xl font-bold)
3. Перевірити кнопку «Зберегти» (Button) у правому верхньому куті
4. Перевірити секцію «Основне» (h3, text-sm font-semibold) з полями в сітці (grid sm:grid-cols-2):
   - «Заголовок *» (Input, label з зірочкою)
   - «Slug *» (Input, label з зірочкою)
   - «Категорія» (select з опцією «Без категорії» за замовчуванням та списком існуючих категорій)
   - «Теги (через кому)» (Input, placeholder «тег1, тег2, тег3»)
5. Перевірити поле «URL обкладинки» (Input, placeholder «https://...») під сіткою
6. Перевірити чекбокс «Опубліковано» (checkbox + label, accent-primary)
7. Перевірити секцію «Короткий опис» (h3) з textarea (rows=3, placeholder «Короткий опис для списку та SEO...»)
8. Перевірити секцію «Вміст» (h3) з textarea (rows=15, placeholder «Вміст статті...»)
9. Перевірити секцію «SEO» (h3) з полями «Meta Title» (Input) та «Meta Description» (textarea, rows=3)
**Очікуваний результат:**
- 4 секції: «Основне», «Короткий опис», «Вміст», «SEO» — кожна у блоці з рамкою (border, p-4)
- Поля «Заголовок *» та «Slug *» позначені як обов'язкові (зірочка в label)
- Випадаючий список «Категорія» містить «Без категорії» (value="") та всі існуючі категорії
- Чекбокс «Опубліковано» знято за замовчуванням (isPublished=false)
**Статус:** ⬜

### TC-BB-2.2 Створення статті — успішне збереження
**Передумова:** Авторизований адміністратор на сторінці `/admin/blog/new`. Існує категорія «Поради» (slug: «porady»).
**Тестові дані:**
- Заголовок: «Як обрати засіб для прання»
- Slug: «yak-obraty-zasib-dlya-prannya»
- Категорія: «Поради»
- Теги: «прання, засоби, поради»
- URL обкладинки: «https://example.com/images/laundry.jpg»
- Чекбокс «Опубліковано»: увімкнути
- Короткий опис: «Детальний гід по вибору найкращих засобів для прання у 2026 році»
- Вміст: «Обрати правильний засіб для прання — непроста задача...» (довільний текст, 200+ символів)

- Meta Description: «Детальний гід по вибору найкращих засобів для прання. Поради від експертів Clean Shop.»
**Кроки:**
1. У поле «Заголовок *» ввести «Як обрати засіб для прання»

3. У випадаючому списку «Категорія» обрати «Поради» (value=«porady»)
4. У поле «Теги (через кому)» ввести «прання, засоби, поради»
5. У поле «URL обкладинки» ввести «https://example.com/images/laundry.jpg»
6. Увімкнути чекбокс «Опубліковано»
7. У textarea «Короткий опис» ввести текст опису
8. У textarea «Вміст» ввести текст статті
9. У поле «Meta Title» ввести «Як обрати засіб для прання — Clean Shop»
10. У textarea «Meta Description» ввести мета-опис
11. Натиснути кнопку «Зберегти»
12. Перевірити що на кнопці з'являється Spinner (isLoading/isSaving)
13. Дочекатися API POST `/api/v1/admin/blog` з payload, де tags — масив ["прання", "засоби", "поради"]
**Очікуваний результат:**
- Toast «Статтю створено» (toast.success)
- Автоматичний redirect на `/admin/blog` (router.push)
- У таблиці з'являється нова стаття «Як обрати засіб для прання» зі статусом «Опубліковано»
**Статус:** ⬜

## 24.3 Редагування блог-посту

### TC-BB-3.1 Сторінка редагування — завантаження даних
**Передумова:** Авторизований адміністратор. Перейти за URL `/admin/blog/{id}`, де {id} — ID існуючої статті.
**Тестові дані:** Стаття: title «Як обрати засіб для прання», slug «yak-obraty-zasib-dlya-prannya», category «porady», tags ["прання", "засоби"], coverImage «https://example.com/img.jpg», isPublished=true, excerpt «Короткий опис», content «Повний текст статті...», metaTitle «SEO Title», metaDescription «SEO Description».
**Кроки:**
1. Перейти на `/admin/blog/{id}`
2. Перевірити що під час завантаження відображається Spinner (size=md, centered)
3. Після завантаження перевірити посилання «← Блог» (Link на `/admin/blog`)
4. Перевірити заголовок сторінки — назва статті «Як обрати засіб для прання» (h2, не «Нова стаття»)
5. Перевірити що поле «Заголовок *» заповнене значенням «Як обрати засіб для прання»
6. Перевірити що поле «Slug *» заповнене значенням «yak-obraty-zasib-dlya-prannya»
7. Перевірити що у «Категорія» обрано «Поради» (value=«porady»)
8. Перевірити що «Теги (через кому)» містить «прання, засоби»
9. Перевірити що «URL обкладинки» містить «https://example.com/img.jpg»
10. Перевірити що чекбокс «Опубліковано» увімкнено
11. Перевірити що textarea «Короткий опис» містить «Короткий опис»
12. Перевірити що textarea «Вміст» містить «Повний текст статті...»
13. Перевірити що «Meta Title» містить «SEO Title»
14. Перевірити що «Meta Description» містить «SEO Description»
**Очікуваний результат:** Всі поля форми заповнені даними існуючої статті. Заголовок сторінки — назва статті, а не «Нова стаття».
**Статус:** ⬜

### TC-BB-3.2 Збереження змін існуючої статті
**Передумова:** Авторизований адміністратор на сторінці `/admin/blog/{id}`.
**Тестові дані:** Змінити заголовок на «Як обрати засіб для прання — оновлена версія», додати тег «оновлення».
**Кроки:**
1. Змінити значення поля «Заголовок *» на «Як обрати засіб для прання — оновлена версія»
2. У поле «Теги (через кому)» дописати «, оновлення» (результат: «прання, засоби, оновлення»)
3. Натиснути кнопку «Зберегти»
4. Дочекатися API PATCH `/api/v1/admin/blog/{id}`
**Очікуваний результат:**
- Toast «Збережено» (toast.success)
- Під заголовком з'являється зелений блок повідомлення «Збережено!» (bg-green-50 text-green-700)
- Redirect НЕ відбувається (залишаємося на сторінці редагування)
- Заголовок сторінки оновлюється відповідно до зміненого значення форми
**Статус:** ⬜

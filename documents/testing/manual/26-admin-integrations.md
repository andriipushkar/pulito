# 26 — Адмін: інтеграція 1С, feature flags, підписки

---

## 26.1 Інтеграція 1С — підключення

### TC-INT-1.1 Статус підключення 1С
**Передумова:** `/admin/integrations/1c`
**Кроки:**
1. Перевірити індикатор статусу: підключено / не підключено
2. Перевірити час останньої синхронізації
**Очікуваний результат:** Статус та час останнього синку відображаються коректно
**Статус:** ⬜

### TC-INT-1.2 Синхронізація товарів з 1С
**Кроки:**
1. Натиснути "Синхронізувати товари"
2. Дочекатися завершення
3. Перевірити лог синхронізації: кількість створених, оновлених, помилок
**Очікуваний результат:** Товари синхронізовані, лог відображає результат
**Статус:** ⬜

### TC-INT-1.3 Лог синхронізації товарів — помилки
**Передумова:** Є товари з некоректними даними в 1С
**Кроки:**
1. Запустити синхронізацію
2. Перевірити помилки в лозі: SKU, опис помилки
**Очікуваний результат:** Помилки логуються з деталями, решта товарів синхронізовано
**Статус:** ⬜

## 26.2 Інтеграція 1С — замовлення та ціни

### TC-INT-2.1 Експорт замовлень в 1С
**Кроки:**
1. Натиснути "Експорт замовлень"
2. Обрати період або конкретні замовлення
3. Підтвердити
**Очікуваний результат:** Замовлення відправлені в 1С, статус "Експортовано"
**Статус:** ⬜

### TC-INT-2.2 Імпорт статусів замовлень з 1С
**Кроки:**
1. Запустити синхронізацію статусів
2. Перевірити що статуси замовлень оновились
**Очікуваний результат:** Статуси з 1С відображаються в адмінці
**Статус:** ⬜

### TC-INT-2.3 Імпорт цін з 1С
**Кроки:**
1. Натиснути "Синхронізувати ціни"
2. Перевірити оновлені ціни товарів
3. Перевірити вирішення конфліктів (ціна в системі ≠ ціна в 1С)
**Очікуваний результат:** Ціни оновлені, конфлікти показані для ручного вирішення
**Статус:** ⬜

### TC-INT-2.4 Синхронізація залишків з 1С
**Кроки:**
1. Натиснути "Синхронізувати залишки"
2. Перевірити маппінг складів 1С → склади системи
3. Перевірити оновлені залишки
**Очікуваний результат:** Залишки оновлені з урахуванням маппінгу складів
**Статус:** ⬜

## 26.3 Feature Flags

### TC-INT-3.1 Список feature flags
**Передумова:** `/admin/feature-flags`
**Кроки:**
1. Перевірити таблицю: ключ, опис, статус (увімкнено/вимкнено)
**Очікуваний результат:** Всі флаги відображаються з актуальними статусами
**Статус:** ⬜

### TC-INT-3.2 Перемикання feature flag
**Кроки:**
1. Натиснути toggle біля флага
2. Перевірити що статус змінився
3. Перевірити що функція увімкнулась/вимкнулась на сайті
**Очікуваний результат:** Флаг перемикається миттєво, функція реагує
**Статус:** ⬜

### TC-INT-3.3 Створення нового feature flag
**Кроки:**
1. Натиснути "Створити флаг"
2. Ввести ключ (латинськими, snake_case) та опис
3. Зберегти
**Очікуваний результат:** Флаг створено, доступний для використання в коді
**Статус:** ⬜

### TC-INT-3.4 Редагування та видалення feature flag
**Кроки:**
1. Натиснути "Редагувати" — змінити опис
2. Зберегти
3. Натиснути "Видалити" на іншому флазі
4. Підтвердити
**Очікуваний результат:** Опис оновлено, флаг видалено
**Статус:** ⬜

### TC-INT-3.5 Feature flag по ключу — управління середовищами
**Передумова:** `/admin/feature-flags/{key}`
**Кроки:**
1. Перевірити статус флага для кожного середовища: dev, staging, production
2. Увімкнути для staging, вимкнути для production
**Очікуваний результат:** Флаг можна контролювати окремо для кожного середовища
**Статус:** ⬜

## 26.4 Адміністрування підписок

### TC-INT-4.1 Список підписок
**Передумова:** `/admin/subscriptions`
**Кроки:**
1. Перевірити таблицю: клієнт, план, статус (активна/призупинена/скасована), дата
**Очікуваний результат:** Всі підписки відображаються з актуальними даними
**Статус:** ⬜

### TC-INT-4.2 Фільтрація підписок за статусом
**Кроки:**
1. Обрати фільтр "Активні"
2. Перевірити результати
3. Обрати "Призупинені", потім "Скасовані"
**Очікуваний результат:** Фільтрація працює для кожного статусу
**Статус:** ⬜

### TC-INT-4.3 Перегляд деталей підписки
**Кроки:**
1. Натиснути на підписку
2. Перевірити: план, дата початку, наступне списання, історія платежів
**Очікуваний результат:** Повна інформація про підписку на окремій сторінці
**Статус:** ⬜

# 17 — Платіжні системи: LiqPay, Monobank, WayForPay

> **Тестові ключі:** налаштовуються в `/admin/payment-settings`
> **Передумова:** для кожного тесту — створити замовлення з онлайн-оплатою

---

## 17.1 LiqPay

### TC-PM-1.1 Створення платежу LiqPay
**Передумова:** Замовлення з оплатою онлайн, провайдер LiqPay
**Кроки:**
1. Оформити замовлення → обрати "Оплата онлайн" → LiqPay
2. Підтвердити замовлення
3. Перевірити redirect на checkout.liqpay.ua
4. Перевірити що на сторінці LiqPay: сума в UAH, опис "Замовлення #XXX"
**Очікуваний результат:** Redirect на LiqPay з правильною сумою
**Статус:** ⬜

### TC-PM-1.2 Успішна оплата LiqPay (sandbox)
**Тестові дані:**
- Картка: `4242424242424242`
- Термін: будь-який у майбутньому
- CVV: `123`
**Кроки:**
1. Ввести тестову картку
2. Підтвердити оплату
3. Перевірити redirect на сайт
4. Перевірити статус замовлення: "Оплачено"
5. Перевірити що webhook доставлено (в `/admin/webhook-logs`)
**Очікуваний результат:** Оплата пройшла, статус "Оплачено", конверсія відправлена в аналітику
**Статус:** ⬜

### TC-PM-1.3 Невдала оплата LiqPay
**Кроки:**
1. На сторінці LiqPay натиснути "Скасувати"
2. Перевірити redirect на сайт
3. Перевірити статус замовлення: "Очікує оплати"
**Очікуваний результат:** Замовлення залишається неоплаченим
**Статус:** ⬜

### TC-PM-1.4 Повторна оплата LiqPay
**Передумова:** Замовлення зі статусом "Очікує оплати"
**Кроки:**
1. В кабінеті перейти до замовлення → "Оплатити"
2. Пройти оплату
**Очікуваний результат:** Оплата успішна, статус оновлено
**Статус:** ⬜

## 17.2 Monobank

### TC-PM-2.1 Створення платежу Monobank
**Передумова:** Замовлення з онлайн-оплатою, провайдер Monobank
**Кроки:**
1. Оформити замовлення → Monobank
2. Перевірити redirect на pay.mbnk.biz або подібне
3. Перевірити суму (в копійках: 100.50 грн = 10050)
**Очікуваний результат:** Redirect на Monobank з правильною сумою
**Статус:** ⬜

### TC-PM-2.2 Успішна оплата Monobank
**Кроки:**
1. Оплатити через Monobank (тестовий режим)
2. Перевірити redirect назад
3. Перевірити статус замовлення
4. Перевірити webhook з EC підписом в `/admin/webhook-logs`
**Очікуваний результат:** Оплата підтверджена, підпис верифіковано
**Статус:** ⬜

### TC-PM-2.3 Webhook replay protection
**Кроки:**
1. В webhook-logs знайти successful webhook
2. Перевірити що повторний webhook з тим самим transactionId ігнорується (лог "Duplicate webhook ignored")
**Очікуваний результат:** Дублікат ігнорується, замовлення не змінюється
**Статус:** ⬜

## 17.3 WayForPay

### TC-PM-3.1 Створення платежу WayForPay
**Передумова:** Замовлення з онлайн-оплатою, провайдер WayForPay
**Кроки:**
1. Оформити замовлення → WayForPay
2. Перевірити redirect на secure.wayforpay.com
3. Перевірити суму та опис
**Очікуваний результат:** Redirect з правильними параметрами
**Статус:** ⬜

### TC-PM-3.2 Успішна оплата WayForPay
**Кроки:**
1. Оплатити (тестовий режим)
2. Перевірити redirect
3. Перевірити статус замовлення: "Оплачено"
4. Перевірити що WayForPay отримав відповідь `{ status: "accept" }`
**Очікуваний результат:** Оплата підтверджена
**Статус:** ⬜

### TC-PM-3.3 Перевірка підпису WayForPay
**Кроки:**
1. Перевірити webhook-logs — HMAC-MD5 підпис валідний
2. Перевірити формат orderReference: `order_<id>_<timestamp>`
**Очікуваний результат:** Підпис валідний, orderId коректно витягнутий
**Статус:** ⬜

## 17.4 Загальні перевірки оплати

### TC-PM-4.1 Server-side tracking при оплаті
**Передумова:** Успішна оплата будь-яким провайдером
**Кроки:**
1. Перевірити логи сервера — `trackPurchase` викликано
2. Якщо FB CAPI налаштовано: перевірити Events Manager
3. Якщо GA4 налаштовано: перевірити Real-time Reports
**Очікуваний результат:** Конверсія відправлена в Facebook CAPI та GA4 Measurement Protocol
**Статус:** ⬜

### TC-PM-4.2 Перевірка суми (amount validation)
**Кроки:**
1. Створити замовлення на 500 грн
2. Симулювати webhook з amount=300 (невідповідність)
**Очікуваний результат:** Лог "Payment amount mismatch", статус залишається pending
**Статус:** ⬜

### TC-PM-4.3 Вже оплачене замовлення
**Кроки:**
1. Повторити webhook для вже оплаченого замовлення
**Очікуваний результат:** Лог "Order already paid, ignoring callback"
**Статус:** ⬜

### TC-PM-4.4 Налаштування в адмінці
**Передумова:** `/admin/payment-settings`
**Кроки:**
1. Перевірити конфігурацію кожного провайдера (ключі замасковані)
2. Натиснути "Тест підключення" для кожного
**Очікуваний результат:** Статус "Підключено" або помилка конкретного провайдера
**Статус:** ⬜

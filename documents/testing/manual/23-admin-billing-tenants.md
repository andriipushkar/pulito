# 23 — Адмін: біллінг, плани, тенанти, домени

---

## 23.1 Сторінка біллінгу

### TC-BT-1.1 Огляд біллінгу
**Передумова:** `/admin/billing`
**Кроки:**
1. Відкрити сторінку біллінгу
2. Перевірити наявність секцій: поточний план, використання, історія платежів
**Очікуваний результат:** Всі секції відображаються коректно
**Статус:** ⬜

### TC-BT-1.2 Відображення поточного плану
**Передумова:** `/admin/billing`
**Кроки:**
1. Перевірити назву поточного плану
2. Перевірити дату наступного списання
3. Перевірити суму підписки
**Очікуваний результат:** Назва плану, дата та сума відповідають дійсності
**Статус:** ⬜

### TC-BT-1.3 Статистика використання ресурсів
**Передумова:** `/admin/billing`
**Кроки:**
1. Перевірити прогрес-бари: товари, замовлення, користувачі, сховище
2. Порівняти поточне використання з лімітами плану
**Очікуваний результат:** Прогрес-бари відображають реальне використання відносно лімітів
**Статус:** ⬜

### TC-BT-1.4 Історія платежів
**Передумова:** `/admin/billing`
**Кроки:**
1. Перевірити таблицю платежів: дата, сума, статус, метод оплати
2. Перевірити пагінацію при великій кількості записів
**Очікуваний результат:** Таблиця з усіма платежами, коректна пагінація
**Статус:** ⬜

## 23.2 Плани та тарифи

### TC-BT-2.1 Список планів
**Передумова:** `/admin/billing/plans`
**Кроки:**
1. Перевірити відображення всіх доступних планів
2. Перевірити що поточний план виділений
**Очікуваний результат:** Всі плани видимі, поточний виділено візуально
**Статус:** ⬜

### TC-BT-2.2 Порівняння функцій планів
**Кроки:**
1. Перевірити таблицю порівняння: ліміти товарів, замовлень, користувачів
2. Перевірити позначки включених/виключених функцій
**Очікуваний результат:** Чітка таблиця порівняння з усіма відмінностями
**Статус:** ⬜

### TC-BT-2.3 Апгрейд плану
**Передумова:** Поточний план — не максимальний
**Кроки:**
1. Натиснути "Оновити" на вищому плані
2. Перевірити модальне вікно підтвердження з різницею в ціні
3. Підтвердити зміну
**Очікуваний результат:** Модал з деталями, план оновлено після підтвердження
**Статус:** ⬜

### TC-BT-2.4 Даунгрейд плану
**Передумова:** Поточний план — не мінімальний
**Кроки:**
1. Натиснути "Змінити" на нижчому плані
2. Перевірити попередження про втрату функцій
3. Підтвердити зміну
**Очікуваний результат:** Попередження відображається, план змінюється з наступного періоду
**Статус:** ⬜

### TC-BT-2.5 Підтвердження зміни плану — метод оплати
**Передумова:** Модальне вікно підтвердження зміни плану
**Кроки:**
1. Перевірити відображення поточного методу оплати
2. Спробувати змінити картку
3. Підтвердити оплату
**Очікуваний результат:** Метод оплати відображається, можна змінити перед підтвердженням
**Статус:** ⬜

## 23.3 Інвойси

### TC-BT-3.1 Список інвойсів
**Передумова:** `/admin/billing/invoices`
**Кроки:**
1. Перевірити таблицю: номер, дата, сума, статус (оплачено/очікує)
2. Перевірити сортування за датою (за замовчуванням — найновіші)
**Очікуваний результат:** Таблиця з усіма інвойсами, коректне сортування
**Статус:** ⬜

### TC-BT-3.2 Фільтрація інвойсів за датою
**Кроки:**
1. Обрати діапазон дат: початок та кінець
2. Натиснути "Фільтрувати"
**Очікуваний результат:** Відображаються тільки інвойси за обраний період
**Статус:** ⬜

### TC-BT-3.3 Завантаження PDF інвойсу
**Кроки:**
1. Натиснути іконку завантаження біля інвойсу
2. Перевірити що PDF завантажився
3. Відкрити PDF: реквізити, таблиця послуг, сума
**Очікуваний результат:** Коректний PDF з усіма реквізитами
**Статус:** ⬜

### TC-BT-3.4 Деталі інвойсу
**Кроки:**
1. Натиснути на номер інвойсу
2. Перевірити: реквізити компанії, перелік послуг, ціни, загальна сума, статус оплати
**Очікуваний результат:** Повна інформація про інвойс на окремій сторінці
**Статус:** ⬜

## 23.4 Управління тенантами

### TC-BT-4.1 Список тенантів
**Передумова:** `/admin/tenants`
**Кроки:**
1. Перевірити таблицю: назва, slug, план, кількість користувачів, статус
2. Перевірити пошук за назвою
**Очікуваний результат:** Таблиця з усіма тенантами, пошук працює
**Статус:** ⬜

### TC-BT-4.2 Створення тенанта
**Кроки:**
1. Натиснути "Створити тенант"
2. Заповнити: назва, slug (автогенерація), обрати план
3. Зберегти
**Очікуваний результат:** Тенант створено, slug унікальний, план призначено
**Статус:** ⬜

### TC-BT-4.3 Редагування тенанта
**Передумова:** Існуючий тенант
**Кроки:**
1. Натиснути "Редагувати" у рядку тенанта
2. Змінити назву та план
3. Зберегти
**Очікуваний результат:** Зміни збережено, дані оновлені в таблиці
**Статус:** ⬜

### TC-BT-4.4 Деактивація тенанта
**Кроки:**
1. Натиснути "Деактивувати" у рядку тенанта
2. Підтвердити в діалозі
**Очікуваний результат:** Статус змінено на "Неактивний", доступ для користувачів тенанта заблоковано
**Статус:** ⬜

## 23.5 Деталі тенанта — користувачі

### TC-BT-5.1 Список користувачів тенанта
**Передумова:** `/admin/tenants/{id}`
**Кроки:**
1. Перевірити таблицю користувачів: ім'я, email, роль, дата додавання
**Очікуваний результат:** Всі користувачі тенанта відображаються
**Статус:** ⬜

### TC-BT-5.2 Додавання користувача до тенанта
**Кроки:**
1. Натиснути "Додати користувача"
2. Ввести email та обрати роль (admin/manager/viewer)
3. Зберегти
**Очікуваний результат:** Користувач доданий, запрошення відправлено на email
**Статус:** ⬜

### TC-BT-5.3 Видалення користувача з тенанта
**Кроки:**
1. Натиснути "Видалити" біля користувача
2. Підтвердити в діалозі
**Очікуваний результат:** Користувач видалений з тенанта, втрачає доступ
**Статус:** ⬜

### TC-BT-5.4 Зміна ролі користувача
**Кроки:**
1. Обрати нову роль у dropdown біля користувача
2. Зберегти зміни
**Очікуваний результат:** Роль оновлена, права доступу змінені відповідно
**Статус:** ⬜

## 23.6 Кастомні домени

### TC-BT-6.1 Список кастомних доменів
**Передумова:** `/admin/domains`
**Кроки:**
1. Перевірити таблицю: домен, статус (активний/очікує верифікації/помилка), SSL, дата додавання
**Очікуваний результат:** Всі домени відображаються з актуальними статусами
**Статус:** ⬜

### TC-BT-6.2 Додавання кастомного домену
**Кроки:**
1. Натиснути "Додати домен"
2. Ввести доменне ім'я
3. Перевірити відображення DNS інструкцій: CNAME запис, значення
**Очікуваний результат:** Домен додано зі статусом "Очікує верифікації", DNS інструкції відображені
**Статус:** ⬜

### TC-BT-6.3 Верифікація домену — перевірка DNS
**Передумова:** Домен додано, DNS записи налаштовані
**Кроки:**
1. Натиснути "Перевірити DNS"
2. Дочекатися результату перевірки
**Очікуваний результат:** При коректних DNS — статус "Верифіковано", при некоректних — повідомлення про помилку
**Статус:** ⬜

### TC-BT-6.4 Статус SSL сертифікату
**Передумова:** Домен верифіковано
**Кроки:**
1. Перевірити статус SSL: "Випускається" → "Активний"
2. Перевірити дату закінчення сертифікату
**Очікуваний результат:** SSL активний, дата відображається
**Статус:** ⬜

### TC-BT-6.5 Помилки верифікації домену
**Передумова:** Домен додано, DNS НЕ налаштовані
**Кроки:**
1. Натиснути "Перевірити DNS"
2. Перевірити повідомлення про помилку
3. Перевірити підказку з правильними DNS записами
**Очікуваний результат:** Зрозуміле повідомлення про помилку з інструкціями
**Статус:** ⬜

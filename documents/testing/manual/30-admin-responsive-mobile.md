# 30 — Адмін-панель: мобільна адаптивність

> Інструмент: Chrome DevTools → Toggle Device Toolbar
> Пристрої: iPhone SE (375×667), iPhone 14 (390×844), iPad (768×1024)
> Браузер: Chrome 120+

---

## 1. САЙДБАР — ГАМБУРГЕР-МЕНЮ

### TC-AM-1.1 Сайдбар прихований за замовчуванням (375px)

**Передумова:** Браузер Chrome, DevTools → Toggle Device Toolbar → iPhone SE (375px). URL: `/admin`. Авторизований як admin.
**Кроки:**

1. Відкрити DevTools → встановити viewport width 375px
2. Перевірити що aside з класом `hidden lg:block` — прихований (display: none)
3. Перевірити що в хедері є кнопка з aria-label "Відкрити меню" (іконка Menu, 22px)
4. Перевірити що кнопка розташована в лівій частині хедера (перший елемент у flex)
5. Перевірити що основний контент займає повну ширину (375px мінус padding)
   **Очікуваний результат:** Сайдбар прихований (display: none). Кнопка "Відкрити меню" видима з іконкою Menu (22x22px). Контент-область = 375px - 24px padding (p-3) = 351px.
   **Статус:** ⬜

### TC-AM-1.2 Відкриття сайдбара як overlay (390px)

**Передумова:** Браузер Chrome, DevTools → iPhone 14 (390px). URL: `/admin`. Авторизований як admin.
**Кроки:**

1. Натиснути кнопку "Відкрити меню" (іконка ☰)
2. Перевірити що з'являється overlay: fixed inset-0, z-50
3. Перевірити затемнення фону: div з класом `bg-black/40` (40% opacity чорний)
4. Перевірити sidebar: `w-60` (240px), `bg-[var(--color-bg)]`, z-10 відносно overlay
5. Перевірити що заголовок "Clean Admin" видимий у верхній частині
6. Перевірити кнопку закриття: іконка Close (20px), розташована у верхньому правому куті sidebar header
7. Перевірити aria-атрибути: role="dialog", aria-modal="true", aria-label="Навігаційне меню"
   **Очікуваний результат:** Overlay: fixed, z-50, inset-0. Затемнення: bg-black/40. Sidebar: width=240px, z-10, bg-[var(--color-bg)]. Кнопка Close видима (20x20px). ARIA-атрибути присутні.
   **Статус:** ⬜

### TC-AM-1.3 Закриття сайдбара по кліку на overlay (390px)

**Передумова:** Браузер Chrome, DevTools → iPhone 14 (390px). URL: `/admin`. Sidebar overlay відкритий.
**Кроки:**

1. Натиснути (tap) на затемнену область справа від sidebar (на div з класом `bg-black/40`)
2. Перевірити що sidebar overlay зникає
3. Перевірити що основний контент знову доступний для взаємодії
4. Перевірити що кнопка "Відкрити меню" знову видима в хедері
   **Очікуваний результат:** Sidebar закривається по кліку на overlay. Контент доступний. Кнопка меню повертається.
   **Статус:** ⬜

### TC-AM-1.4 Закриття сайдбара по кнопці Close (375px)

**Передумова:** Браузер Chrome, DevTools → iPhone SE (375px). URL: `/admin`. Sidebar overlay відкритий.
**Кроки:**

1. Натиснути кнопку Close (×) в верхньому правому куті sidebar (aria-label "Закрити меню")
2. Перевірити що sidebar overlay зникає
3. Перевірити що контент знову доступний
   **Очікуваний результат:** Sidebar закривається по кнопці Close. Клас кнопки: `lg:hidden` — видима тільки на мобільних.
   **Статус:** ⬜

### TC-AM-1.5 Навігація через sidebar (375px)

**Передумова:** Браузер Chrome, DevTools → iPhone SE (375px). URL: `/admin`. Sidebar відкритий.
**Кроки:**

1. Перевірити що всі 5 секцій меню видимі з вертикальним скролом (overflow-y-auto на nav)
2. Прокрутити до секції "Налаштування"
3. Натиснути "Загальні" → перевірити перехід на `/admin/settings`
4. Перевірити що sidebar автоматично закрився після переходу (onClick setSidebarOpen(false))
5. Перевірити що пункт "Загальні" має активний стан (bg-primary, text-white)
   **Очікуваний результат:** Всі секції доступні через scroll. Після натискання sidebar закривається. Активний пункт виділений. Nav має overflow-y: auto.
   **Статус:** ⬜

### TC-AM-1.6 Сайдбар на планшеті (768px)

**Передумова:** Браузер Chrome, DevTools → iPad (768px). URL: `/admin`. Авторизований як admin.
**Кроки:**

1. Перевірити що sidebar прихований (breakpoint lg = 1024px, 768 < 1024)
2. Перевірити наявність кнопки гамбургер-меню
3. Відкрити sidebar → перевірити що це overlay (як на мобільних)
4. Перевірити що контент-область = 768px - padding
   **Очікуваний результат:** Sidebar прихований (lg:block не активний при 768px). Overlay-поведінка як на мобільних. Контент = 768px - 24px padding (p-3 sm:p-4) = 744px.
   **Статус:** ⬜

## 2. ХЕДЕР ТА BREADCRUMBS

### TC-AM-2.1 Хедер — мобільний layout (375px)

**Передумова:** Браузер Chrome, DevTools → iPhone SE (375px). URL: `/admin/orders`. Авторизований як admin.
**Кроки:**

1. Перевірити що кнопка гамбургер видима (lg:hidden, перший елемент)
2. Перевірити що breadcrumbs приховані (клас `sm:flex` = hidden на < 640px)
3. Перевірити що замість breadcrumbs відображається назва поточної сторінки: "Замовлення" (span, text-sm, font-semibold, sm:hidden)

5. Перевірити що іконка сповіщень (дзвінок) видима
6. Перевірити padding хедера: px-3 py-2.5 (12px x 10px)
   **Очікуваний результат:** Гамбургер видимий. Breadcrumbs приховані. Назва сторінки "Замовлення" відображається. Сповіщення видимі. Maintenance, search, hotkeys — приховані. Padding: 12px x 10px.
   **Статус:** ⬜

### TC-AM-2.2 Хедер — планшет (768px)

3. Перевірити що мобільна назва сторінки прихована (sm:hidden)

   **Статус:** ⬜

   **Очікуваний результат:** Контент вміщується в 320px без горизонтального скролу сторінки. Назви товарів обрізаються або переносяться. Кнопки дій доступні (touch target >= 44px). Пошукове поле = full-width. Layout не ламається.
   **Статус:** ⬜

### TC-AM-21.5 Touch targets менше 44px (Негативний)

**Передумова:** Браузер Chrome, DevTools → iPhone SE (375px). URL: `/admin/orders` або `/admin/products`. Авторизований як admin.
**Кроки:**

1. Через Inspect Element перевірити computed розміри інтерактивних елементів:
   - Чекбокси у таблиці
   - Іконки дій (редагувати, видалити, перегляд)
   - Посилання в рядках таблиці
   - Кнопки пагінатора
2. Зафіксувати елементи з touch target < 44x44px
3. Спробувати натиснути на маленькі елементи пальцем (або тач-симуляцією)
   **Очікуваний результат:** Всі інтерактивні елементи мають мінімальний touch target 44x44px (з урахуванням padding). Якщо є елементи < 44px — це баг, який потрібно виправити. Spacing між клікабельними елементами >= 8px. Випадкові натискання на сусідні елементи не відбуваються.
   **Статус:** ⬜

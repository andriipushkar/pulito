> Браузери: Chrome 120+, Firefox 120+, Safari 17+
> Інструмент: Chrome DevTools → Toggle Device Toolbar → Responsive
> Роздільні здатності: 1280×800, 1440×900, 1920×1080

## 1. САЙДБАР — РОЗГОРНУТИЙ СТАН

### TC-AD-1.1 Сайдбар — ширина та розташування (1280px)

**Передумова:** Браузер Chrome, відкрити DevTools (F12) → Toggle Device Toolbar → встановити ширину 1280px. URL: `/admin`. Авторизований як admin.
**Кроки:**

1. Відкрити DevTools → встановити viewport width 1280px
2. Перевірити сайдбар через Inspect Element: CSS-клас `w-60` (ширина 240px / 15rem)
3. Перевірити що сайдбар має `border-r border-[var(--color-border)]` та `bg-[var(--color-bg)]`
4. Перевірити що основний контент займає `flex-1` (1280 - 240 = 1040px)
5. Перевірити що заголовок "Clean Admin" видимий у верхній частині сайдбара
   **Очікуваний результат:** Сайдбар ширина = 240px (w-60), фіксований зліва. Контент-область = 1040px. Між сайдбаром та контентом — вертикальна border. Текст "Clean Admin" відображається як посилання на `/admin`.
   **Статус:** ⬜

### TC-AD-1.2 Сайдбар — згортання до іконок (1280px)

**Передумова:** Браузер Chrome, DevTools → viewport 1280px. URL: `/admin`. Авторизований як admin.
**Кроки:**

1. Натиснути кнопку згортання сайдбара (подвійна стрілка вліво, aria-label "Згорнути меню")
2. Перевірити через Inspect що сайдбар змінив клас на `w-16` (ширина 64px / 4rem)
3. Перевірити що назви пунктів меню приховані, залишились тільки іконки-emoji
4. Перевірити що заголовок "Clean Admin" прихований
5. Перевірити що при hover на іконку з'являється title-тултіп з назвою пункту
6. Перевірити що контент-область розширилась до 1280 - 64 = 1216px
7. Перевірити що між секціями відображається горизонтальний роздільник замість текстового заголовка секції
   **Очікуваний результат:** Сайдбар ширина = 64px (w-16). Іконки центровані. Title-тултіп з назвою з'являється при hover. Контент-область = 1216px. Стан зберігається у `localStorage` під ключем `admin-sidebar-collapsed`.
   **Статус:** ⬜

### TC-AD-1.3 Сайдбар — розгортання після згортання (1440px)

**Передумова:** Браузер Chrome, DevTools → viewport 1440px. URL: `/admin`. Сайдбар згорнутий.
**Кроки:**

1. Натиснути кнопку розгортання (подвійна стрілка вправо, aria-label "Розгорнути меню")
2. Перевірити що сайдбар повернувся до `w-60` (240px)
3. Перевірити що всі назви пунктів відображаються поруч з іконками
4. Перевірити що заголовки секцій ("Контент", "Канали", "Налаштування" тощо) відображаються як uppercase текст
5. Перевірити що в нижній частині відображається ім'я користувача, роль, посилання "На сайт" та "Вийти"
   **Очікуваний результат:** Сайдбар ширина = 240px. Текст пунктів меню відображається: іконка + label в один рядок. Заголовки секцій видимі (px-3 py-1, text-xs, font-semibold, uppercase). Футер сайдбара показує fullName, role, та два посилання.
   **Статус:** ⬜

### TC-AD-1.4 Сайдбар — навігація та active state (1920px)

**Передумова:** Браузер Chrome, DevTools → viewport 1920px. URL: `/admin/orders`. Авторизований як admin.
**Кроки:**

1. Перевірити що пункт "Замовлення" має клас `bg-[var(--color-primary)] text-white` (активний стан)
2. Перевірити що всі інші пункти мають `text-[var(--color-text)] hover:bg-[var(--color-bg-secondary)]`
3. Натиснути на "Товари" → перевірити перехід на `/admin/products`
4. Перевірити що тепер "Товари" має активний стан, а "Замовлення" — ні
5. Перевірити кількість навігаційних секцій: 5 секцій (без заголовку, Контент, Канали, Маркетплейси, Налаштування)
   **Очікуваний результат:** Активний пункт меню виділений primary-кольором з білим текстом. При переході активний стан переключається. Всі 5 секцій відображаються з відповідними заголовками. На 1920px контент-область = 1920 - 240 = 1680px.
   **Статус:** ⬜

## 2. ХЕДЕР ТА BREADCRUMBS

### TC-AD-2.1 Хедер — елементи та розташування (1280px)

**Передумова:** Браузер Chrome, DevTools → viewport 1280px. URL: `/admin/orders`. Авторизований як admin.
**Кроки:**

1. Перевірити що кнопка гамбургер-меню (☰) прихована (клас `lg:hidden`)
2. Перевірити breadcrumbs: "Dashboard > Замовлення" — видимі (клас `sm:flex`)
3. Перевірити наявність кнопки "Обслуговування" в хедері (клас `lg:flex`)
4. Перевірити наявність кнопки "Перейти до..." з хоткеєм Ctrl+K (клас `lg:flex`)
5. Перевірити іконку сповіщень (дзвінок) з бейджем кількості
6. Перевірити кнопку гарячих клавіш (kbd "/") — видима тільки на lg+ (клас `lg:block`)
   **Очікуваний результат:** Гамбургер прихований. Breadcrumbs видимі: "Dashboard" як посилання + ">" + "Замовлення" як текст. Кнопки maintenance, command palette, сповіщення, гарячі клавіші — всі видимі. Хедер padding = px-6 (24px) на lg+.

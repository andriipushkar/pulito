## 7.1 Список товарів (`/admin/products`)

### TC-AP-1.1 Пошук товарів з debounce
**Передумова:** `/admin/products`, список містить 20+ товарів
**Кроки:**
1. Клікнути на поле пошуку
2. Ввести `Порош` — зачекати 300мс
3. Переконатися що запит пішов після паузи (debounce), а не на кожну літеру
4. Очистити поле — список повернувся до повного
5. Ввести код товару (наприклад `ABC-123`) — перевірити пошук за кодом
**Очікуваний результат:** Пошук спрацьовує з debounce (~300мс), фільтрує за назвою та кодом, при очистці — повний список
**Статус:** ⬜

### TC-AP-1.2 Панель фільтрів — розкриття/згортання
**Передумова:** `/admin/products`
**Кроки:**
1. Натиснути кнопку "Фільтри" (toggle)
2. Панель фільтрів розгортається
3. Натиснути повторно — панель згортається
4. Розгорнути знову — стан фільтрів збережений
**Очікуваний результат:** Панель фільтрів toggle працює, стан зберігається при згортанні/розгортанні
**Статус:** ⬜

### TC-AP-1.3 Фільтр за категорією
**Передумова:** `/admin/products`, панель фільтрів відкрита
**Кроки:**
1. У випадаючому "Категорія" обрати конкретну категорію
2. Перевірити що список оновився — тільки товари з обраної категорії
3. Змінити на іншу категорію — список оновлюється
4. Обрати "Всі категорії" — повний список
**Очікуваний результат:** Фільтрація за категорією працює коректно, включаючи скидання
**Статус:** ⬜

### TC-AP-1.4 Фільтр за статусом (active/inactive)
**Передумова:** `/admin/products`, панель фільтрів відкрита
**Кроки:**
1. У фільтрі "Статус" обрати "Активний" — тільки активні товари
2. Обрати "Неактивний" — тільки неактивні товари
3. Скинути фільтр — всі товари
**Очікуваний результат:** Фільтр за статусом коректно показує активні/неактивні товари
**Статус:** ⬜

### TC-AP-1.5 Фільтр за залишком (out/low/in stock)
**Передумова:** `/admin/products`, панель фільтрів відкрита
**Кроки:**
1. У фільтрі "Залишок" обрати "Немає в наявності" — товари з quantity=0
2. Обрати "Мало" — товари з низьким залишком
3. Обрати "В наявності" — товари з достатнім залишком
4. Скинути — всі товари
**Очікуваний результат:** Фільтрація за рівнем залишку працює для всіх трьох варіантів
**Статус:** ⬜

### TC-AP-1.6 Сортування (11 варіантів)
**Передумова:** `/admin/products`, панель фільтрів відкрита
**Кроки:**
1. Перевірити всі 11 варіантів сортування у випадаючому "Сортування":
   - Назва (А-Я), Назва (Я-А)
   - Ціна (зростання), Ціна (спадання)
   - Залишок (зростання), Залишок (спадання)
   - Дата створення (нові), Дата створення (старі)
   - Код (А-Я), Код (Я-А)
   - Популярність
2. Для кожного варіанту переконатися що порядок товарів змінюється відповідно
**Очікуваний результат:** Всі 11 варіантів сортування працюють коректно
**Статус:** ⬜

### TC-AP-1.7 Комбінація фільтрів
**Передумова:** `/admin/products`, панель фільтрів відкрита
**Кроки:**
1. Обрати категорію + статус "Активний" + залишок "В наявності"
2. Додати сортування за ціною (зростання)
3. Ввести текст у пошук
4. Перевірити що всі фільтри застосовані одночасно
5. Скинути всі фільтри — повний список
**Очікуваний результат:** Фільтри комбінуються коректно, скидання повертає повний список
**Статус:** ⬜

### TC-AP-1.8 Bulk: чекбокс "Обрати всі"
**Передумова:** `/admin/products`, є товари в списку
**Кроки:**
1. Натиснути чекбокс "Обрати всі" в заголовку таблиці
2. Всі чекбокси рядків стають обраними
3. Натиснути повторно — всі знімаються
4. Обрати "Обрати всі", потім зняти один рядок — "Обрати всі" знімається (indeterminate)
**Очікуваний результат:** Чекбокс "Обрати всі" керує всіма рядками, підтримує стан indeterminate
**Статус:** ⬜

### TC-AP-1.9 Bulk: вибір окремих рядків
**Передумова:** `/admin/products`
**Кроки:**
1. Натиснути чекбокс на 3 різних рядках
2. Переконатися що тільки ці 3 обрані
3. Зняти один — залишилось 2
4. Перевірити що лічильник обраних оновлюється
**Очікуваний результат:** Індивідуальний вибір рядків працює, лічильник відображає кількість обраних
**Статус:** ⬜

### TC-AP-1.10 Bulk: Активувати / Деактивувати
**Передумова:** `/admin/products`, обрано 3 неактивних товари
**Кроки:**
1. У dropdown масових дій обрати "Активувати"
2. Натиснути "Виконати"
3. Підтвердити у модальному вікні
4. Перевірити що всі 3 товари стали активними
5. Повторити з "Деактивувати" — товари стали неактивними
**Очікуваний результат:** Масова активація/деактивація працює з підтвердженням
**Статус:** ⬜

### TC-AP-1.11 Bulk: Змінити категорію
**Передумова:** `/admin/products`, обрано 2 товари
**Кроки:**
1. У dropdown масових дій обрати "Змінити категорію"
2. З'являється picker категорій — обрати нову категорію
3. Натиснути "Виконати"
4. Підтвердити у модальному вікні
5. Перевірити що категорія змінилася для обох товарів
**Очікуваний результат:** Масова зміна категорії з picker та підтвердженням
**Статус:** ⬜

### TC-AP-1.12 Bulk: Експорт обраних
**Передумова:** `/admin/products`, обрано 5 товарів
**Кроки:**
1. У dropdown масових дій обрати "Експорт"
2. Натиснути "Виконати"
3. Перевірити що завантажується файл з даними тільки обраних товарів
**Очікуваний результат:** Експортується файл тільки з обраними товарами
**Статус:** ⬜

### TC-AP-1.13 Експорт всіх товарів
**Передумова:** `/admin/products`
**Кроки:**
1. Натиснути кнопку "Експорт всіх"
2. Дочекатися завантаження файлу
3. Відкрити файл — перевірити що містить всі товари з коректними даними
**Очікуваний результат:** Завантажується файл з усіма товарами
**Статус:** ⬜

### TC-AP-1.14 Клік на назву товару — перехід на деталі
**Передумова:** `/admin/products`
**Кроки:**
1. Клікнути на назву товару в таблиці
2. Перевірити перехід на `/admin/products/[id]`
3. Натиснути "Назад" — повернення до списку зі збереженими фільтрами
**Очікуваний результат:** Назва є посиланням на сторінку деталей, повернення зберігає стан фільтрів
**Статус:** ⬜

### TC-AP-1.15 Пагінація та вибір кількості на сторінку
**Передумова:** `/admin/products`, 50+ товарів
**Кроки:**
1. Перевірити наявність пагінації внизу списку
2. Перейти на сторінку 2 — список оновлюється
3. Перейти на останню сторінку
4. Змінити кількість на сторінку (наприклад 10 → 25 → 50)
5. Перевірити що кількість рядків відповідає обраному значенню
6. Перевірити що пагінація перераховується при зміні розміру сторінки
**Очікуваний результат:** Пагінація працює, page size selector змінює кількість рядків та перераховує сторінки
**Статус:** ⬜

## 7.2 Деталі товару (`/admin/products/[id]`)

### TC-AP-2.1 Галерея зображень — перегляд та видалення
**Передумова:** `/admin/products/[id]`, товар має 3+ зображення
**Кроки:**
1. Перевірити відображення галереї зображень
2. Навести курсор на зображення — з'являється кнопка видалення (hover)
3. Натиснути кнопку видалення
4. З'являється підтвердження — натиснути "Скасувати" — зображення залишилось
5. Натиснути видалення знову → підтвердити — зображення видалено
6. Перевірити що наступне зображення стало головним (якщо видалено перше)
**Очікуваний результат:** Кнопка видалення на hover, підтвердження перед видаленням, автоматична зміна головного зображення
**Статус:** ⬜

### TC-AP-2.2 Завантаження зображень (multiple)
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Натиснути кнопку "Додати зображення"
2. Обрати 3 файли одночасно (JPEG, PNG, до 5MB кожен)
3. Перевірити progress bar для кожного файлу
4. Дочекатися завершення — всі 3 з'явились у галереї
5. Перевірити що створено варіанти: full, medium, thumbnail, blur (WebP)
**Очікуваний результат:** Множинне завантаження з progress bar, автоматична обробка через sharp у 4 варіанти
**Статус:** ⬜

### TC-AP-2.3 Основна інформація — валідація обов'язкових полів
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Очистити поле "Назва" (required) — натиснути "Зберегти"
2. Помилка валідації: "Назва обов'язкова"
3. Очистити поле "Код" (required) — натиснути "Зберегти"
4. Помилка валідації: "Код обов'язковий"
5. Заповнити Назву та Код — зберегти успішно
**Очікуваний результат:** Обов'язкові поля Назва та Код валідуються, збереження блокується при порожніх значеннях
**Статус:** ⬜

### TC-AP-2.4 Основна інформація — Slug, Barcode, Category, Weight
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Змінити Slug вручну — зберегти — перевірити що URL товару оновився
2. Заповнити Barcode (штрихкод) — зберегти
3. Змінити Category ID через випадаючий список — зберегти — товар переїхав у нову категорію
4. Ввести Weight (вага) — зберегти
5. Ввести від'ємне значення у Weight — валідація не дозволяє
6. Ввести нечислове значення — валідація не дозволяє
**Очікуваний результат:** Всі поля зберігаються коректно, Weight валідується на додатне числове значення
**Статус:** ⬜

### TC-AP-2.5 Ціни — роздрібна (required) та стара ціна
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Очистити "Роздрібна ціна" (required) → зберегти — помилка валідації
2. Ввести від'ємне значення — помилка валідації
3. Ввести `120.50` — зберегти успішно
4. Заповнити "Стара ціна" значенням `150.00` — зберегти
5. Перевірити на фронті — стара ціна перекреслена, нова відображається
**Очікуваний результат:** Роздрібна ціна required, стара ціна формує перекреслення на фронті
**Статус:** ⬜

### TC-AP-2.6 Оптові ціни (Wholesale 1/2/3)
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Заповнити "Оптова 1": `100.00`
2. Заповнити "Оптова 2": `90.00`
3. Заповнити "Оптова 3": `80.00`
4. Зберегти
5. Увійти як оптовий користувач групи 1 — перевірити ціну 100.00
6. Увійти як оптовий групи 2 — ціна 90.00
7. Увійти як оптовий групи 3 — ціна 80.00
**Очікуваний результат:** Три рівні оптових цін зберігаються та відображаються відповідно до групи користувача
**Статус:** ⬜

### TC-AP-2.7 Залишок та чекбокси Active/Promo
**Передумова:** `/admin/products/[id]`
**Кроки:**
1. Очистити "Кількість" (required) → зберегти — помилка валідації
2. Ввести `50` — зберегти успішно
3. Зняти чекбокс "Активний" → зберегти — товар не відображається на фронті
4. Увімкнути "Активний" → зберегти — товар знову видимий
5. Увімкнути чекбокс "Акція" → зберегти — товар отримує мітку акції на фронті
6. Вимкнути "Акція" — мітка зникає
**Очікуваний результат:** Кількість required, чекбокс Active керує видимістю, чекбокс Promo додає мітку акції
**Статус:** ⬜

### TC-AP-2.8 WYSIWYG редактор опису
**Передумова:** `/admin/products/[id]`, секція "Опис"
**Кроки:**
1. Клікнути на WYSIWYG редактор
2. Ввести текст з форматуванням: жирний, курсив, список, заголовок
3. Вставити посилання
4. Зберегти
5. Перевірити на фронті — форматування збережено коректно
**Очікуваний результат:** WYSIWYG редактор зберігає форматування, на фронті HTML відображається коректно
**Статус:** ⬜

### TC-AP-2.9 Історія цін — розгортання та таблиця

5. Згорнути секцію — натиснути повторно — дані на місці
**Очікуваний результат:** Collapsible секція з таблицею історії цін (дата, стара/нова ціна) в хронологічному порядку
**Статус:** ⬜

## 7.3 Імпорт (`/admin/import`)

### TC-AP-3.1 Завантаження прайс-листа (XLSX)
**Передумова:** `/admin/import`, підготовлений XLSX файл з 5 товарами
**Кроки:**
1. У секції "Завантажити прайс-лист" натиснути кнопку або перетягнути файл
2. Обрати XLSX файл
3. Перевірити progress bar під час завантаження
4. Дочекатися завершення — файл прийнятий

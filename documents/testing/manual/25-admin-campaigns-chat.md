# 25 — Адмін: кампанії, чат

---

## 25.1 Список кампаній

### TC-CCH-1.1 Таблиця кампаній
**Передумова:** `/admin/campaigns`
**Кроки:**
1. Перевірити таблицю: назва, тип, статус, дати, знижка
2. Перевірити пагінацію
**Очікуваний результат:** Всі кампанії відображаються з актуальними статусами
**Статус:** ⬜

### TC-CCH-1.2 Фільтрація кампаній за статусом
**Кроки:**
1. Обрати фільтр "Активні"
2. Перевірити результати
3. Обрати "Заплановані", потім "Завершені"
**Очікуваний результат:** Відображаються тільки кампанії з обраним статусом
**Статус:** ⬜

### TC-CCH-1.3 Пошук кампаній
**Кроки:**
1. Ввести назву кампанії в поле пошуку
**Очікуваний результат:** Відображаються тільки відповідні кампанії
**Статус:** ⬜

---

## 25.2 Створення кампанії

### TC-CCH-2.1 Створення знижкової кампанії
**Передумова:** `/admin/campaigns/new`
**Кроки:**
1. Ввести назву кампанії
2. Обрати тип "Знижка"
3. Вказати умови: мінімальна сума, категорія товарів
4. Встановити значення знижки (% або фіксована)
5. Зберегти
**Очікуваний результат:** Кампанія створена з усіма параметрами
**Статус:** ⬜

### TC-CCH-2.2 Створення кампанії безкоштовної доставки
**Кроки:**
1. Обрати тип "Безкоштовна доставка"
2. Вказати мінімальну суму замовлення
3. Обрати дати дії
4. Зберегти
**Очікуваний результат:** Кампанія створена, безкоштовна доставка при досягненні мін. суми
**Статус:** ⬜

### TC-CCH-2.3 Створення бандл-кампанії
**Кроки:**
1. Обрати тип "Бандл"
2. Обрати товари що входять в набір
3. Встановити знижку на набір
4. Зберегти
**Очікуваний результат:** Бандл-кампанія створена, знижка застосовується при купівлі набору
**Статус:** ⬜

### TC-CCH-2.4 Купон — автогенерація коду
**Кроки:**
1. При створенні кампанії обрати "Купон"
2. Натиснути "Згенерувати код"
3. Перевірити що код унікальний
**Очікуваний результат:** Унікальний промокод згенеровано автоматично
**Статус:** ⬜

### TC-CCH-2.5 Купон — ручне введення коду
**Кроки:**
1. При створенні кампанії обрати "Купон"
2. Ввести код вручну
3. Перевірити валідацію унікальності
**Очікуваний результат:** Код прийнято якщо унікальний, помилка якщо вже існує
**Статус:** ⬜

## 25.3 Редагування та видалення кампаній

### TC-CCH-3.1 Редагування умов кампанії
**Передумова:** Існуюча кампанія
**Кроки:**
1. Відкрити кампанію на редагування
2. Змінити умови: мінімальна сума, знижка
3. Зберегти
**Очікуваний результат:** Зміни збережено, нові умови діють
**Статус:** ⬜

### TC-CCH-3.2 Пауза та відновлення кампанії
**Кроки:**
1. Натиснути "Призупинити" на активній кампанії
2. Перевірити що знижка більше не застосовується
3. Натиснути "Відновити"
4. Перевірити що знижка знову діє
**Очікуваний результат:** Кампанія коректно призупиняється та відновлюється
**Статус:** ⬜

### TC-CCH-3.3 Видалення кампанії
**Кроки:**
1. Натиснути "Видалити" біля кампанії
2. Підтвердити в діалозі
**Очікуваний результат:** Кампанія видалена
**Статус:** ⬜

### TC-CCH-3.4 Видалення активної кампанії — попередження
**Передумова:** Кампанія зі статусом "Активна"
**Кроки:**
1. Натиснути "Видалити"
2. Перевірити попередження що кампанія зараз активна
**Очікуваний результат:** Додаткове попередження перед видаленням активної кампанії
**Статус:** ⬜

## 25.4 Планування кампаній

### TC-CCH-4.1 Встановлення дат початку та завершення
**Кроки:**
1. При створенні кампанії обрати дату початку (майбутня дата)
2. Обрати дату завершення
3. Зберегти
**Очікуваний результат:** Кампанія зі статусом "Запланована"
**Статус:** ⬜

### TC-CCH-4.2 Автоматична активація
**Передумова:** Запланована кампанія з датою початку = сьогодні
**Кроки:**
1. Дочекатися або тригернути cron автоактивації
2. Перевірити статус кампанії
**Очікуваний результат:** Статус змінено на "Активна" автоматично
**Статус:** ⬜

### TC-CCH-4.3 Автоматична деактивація
**Передумова:** Активна кампанія з датою завершення = сьогодні
**Кроки:**
1. Дочекатися або тригернути cron
2. Перевірити статус кампанії
**Очікуваний результат:** Статус змінено на "Завершена" автоматично
**Статус:** ⬜

## 25.5 Чат — список кімнат

### TC-CCH-5.1 Список чат-кімнат
**Передумова:** `/admin/chat`
**Кроки:**
1. Перевірити список кімнат: імʼя клієнта, останнє повідомлення, час, статус
2. Перевірити бейдж непрочитаних повідомлень
**Очікуваний результат:** Всі кімнати відображаються, непрочитані виділені
**Статус:** ⬜

### TC-CCH-5.2 Фільтрація кімнат за статусом
**Кроки:**
1. Обрати фільтр "Відкриті"
2. Перевірити результати
3. Обрати "Закриті"
**Очікуваний результат:** Фільтрація працює коректно
**Статус:** ⬜

## 25.6 Чат — робота з повідомленнями

### TC-CCH-6.1 Історія повідомлень
**Передумова:** Відкрити чат-кімнату
**Кроки:**
1. Перевірити відображення всіх повідомлень: текст, час, автор
2. Перевірити скрол вгору для старих повідомлень
**Очікуваний результат:** Повна історія повідомлень, хронологічний порядок
**Статус:** ⬜

### TC-CCH-6.2 Відправлення повідомлення
**Кроки:**
1. Ввести текст повідомлення
2. Натиснути "Відправити" або Enter
**Очікуваний результат:** Повідомлення відправлено, зʼявилось у чаті
**Статус:** ⬜

### TC-CCH-6.3 Призначення менеджера
**Кроки:**
1. Обрати менеджера з dropdown
2. Зберегти
**Очікуваний результат:** Кімната привʼязана до менеджера, він отримує сповіщення
**Статус:** ⬜

### TC-CCH-6.4 Закриття та відкриття кімнати
**Кроки:**
1. Натиснути "Закрити кімнату"
2. Перевірити що статус "Закрита"
3. Натиснути "Відкрити знову"
**Очікуваний результат:** Статус змінюється, клієнт може знову писати після відкриття
**Статус:** ⬜

### TC-CCH-6.5 Сповіщення про нове повідомлення в реальному часі
**Кроки:**
1. Відкрити чат-кімнату в адмінці
2. Відправити повідомлення від клієнта (інший браузер)
3. Перевірити що повідомлення зʼявилось без перезавантаження
**Очікуваний результат:** Нове повідомлення зʼявляється миттєво, автоскрол вниз
**Статус:** ⬜

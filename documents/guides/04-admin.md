# Інструкція для адміністратора

Адміністратор має повний доступ до системи, включаючи всі функції менеджера (див. [03-manager.md](./03-manager.md)) та додаткові можливості системного адміністрування, описані в цьому посібнику.

---

## 1. Дашборд та аналітика

### 1.1. KPI (ключові показники ефективності)

Після входу в адмін-панель ви потрапляєте на дашборд з основними KPI. У верхній частині відображаються картки з показниками:

- **Виручка** — загальна сума оплачених замовлень за обраний період (сьогодні / тиждень / місяць).
- **Кількість замовлень** — загальна кількість замовлень за період.
- **Середній чек** — виручка, поділена на кількість замовлень.
- **Нові клієнти** — кількість нових реєстрацій за період.
- **Конверсія** — відсоток відвідувачів, що оформили замовлення.

Натисніть на будь-яку KPI-картку, щоб побачити деталізований графік за обраний період.

### 1.2. Модулі аналітики (18 модулів)

1. У бічному меню оберіть **Аналітика**.
2. Відкриється сторінка з доступними модулями аналітики. Нижче описано кожен.

#### ABC-аналіз

- Категоризація товарів за обсягом продажів:
  - **A** — топ-20% товарів, що дають 80% виручки.
  - **B** — наступні 30% товарів.
  - **C** — решта 50% товарів.
- Оберіть **Період** та натисніть **Сформувати**.
- Результат — таблиця з товарами, їх групою (A/B/C), кількістю продажів та сумою.

#### RFM-аналіз

- Сегментація клієнтів за трьома параметрами:
  - **Recency** — давність останньої покупки.
  - **Frequency** — частота покупок.
  - **Monetary** — загальна сума покупок.
- Кожен клієнт отримує оцінку від 1 до 5 по кожному параметру.
- Результат — матриця сегментів (наприклад, "Лояльні VIP", "Сплячі", "Під загрозою втрати").

#### LTV (Lifetime Value)

- Прогнозована загальна цінність клієнта за весь час:
  - Середній LTV по всіх клієнтах.
  - Розподіл клієнтів за рівнями LTV.
  - Графік динаміки LTV за місяцями.

#### Когортний аналіз

- Таблиця, що показує поведінку клієнтів, згрупованих за місяцем реєстрації:
  - Скільки з кожної когорти повернулися через 1, 2, 3... місяці.
  - Відсоток утримання (retention rate).
  - Колірна шкала від зеленого (високе утримання) до червоного (низьке).

#### Воронка продажів

- Візуалізація шляху покупця:
  1. Перегляд каталогу.
  2. Перегляд товару.
  3. Додавання до кошика.
  4. Початок оформлення.
  5. Завершення замовлення.
- Для кожного етапу — кількість користувачів та відсоток переходу на наступний.

#### Географія

- Карта або таблиця з розподілом замовлень за:
  - **Містами** — топ міст за кількістю замовлень та виручкою.
  - **Областями** — агрегація по областях.
- Кольорова карта з інтенсивністю за кількістю замовлень.

#### Канали

- Розподіл замовлень та виручки за маркетинговими каналами:
  - Прямий трафік.
  - Пошукові системи (Google, Bing).
  - Соціальні мережі (Facebook, Instagram, Telegram).
  - Реферальні посилання.
  - Email-розсилки.
- Графіки та таблиці з конверсією по кожному каналу.

#### Залишки

- Аналіз складських залишків:
  - Товари з нульовим залишком.
  - Товари з мінімальним залишком (менше порогу).
  - Обіговість складу — за скільки днів продається середній запас.
  - Таблиця з товарами, що потребують поповнення.

#### Ціни

- Аналіз ціноутворення:
  - Середня маржа по категоріях.
  - Товари з найвищою та найнижчою маржею.
  - Динаміка зміни цін за період.

#### Churn (відтік)

- Аналіз відтоку клієнтів:
  - Кількість клієнтів, що не купували останні 30/60/90 днів.
  - Відсоток відтоку за місяцями.
  - Список клієнтів "під загрозою" для проактивної роботи.

#### Інші модулі

- **Повернення** — статистика повернень та причини.
- **Знижки** — ефективність промокодів та купонів.
- **Час обробки** — середній час від створення до відправлення замовлення.
- **Продуктивність менеджерів** — кількість оброблених замовлень кожним менеджером.
- **Пошукові запити** — що шукають клієнти, запити без результатів.
- **Відгуки** — динаміка відгуків, середній рейтинг.
- **Сезонність** — порівняння продажів з минулорічними за цей самий період.
- **Реферали** — ефективність реферальної програми.

### 1.3. Алерти (сповіщення за порогами)

1. У розділі **Аналітика** перейдіть на вкладку **Алерти**.
2. Натисніть **Створити алерт**.
3. Заповніть форму:
   - **Назва** — зрозуміле найменування (наприклад, "Мало замовлень").
   - **Метрика** — оберіть показник для відстеження:
     - Кількість замовлень за день.
     - Виручка за день.
     - Залишок конкретного товару.
     - Кількість нових клієнтів.
   - **Умова** — менше / більше / дорівнює.
   - **Поріг** — числове значення.
   - **Канал сповіщення** — Email та/або Telegram.
4. Натисніть **Зберегти**.
5. Коли значення метрики досягне порогу, ви отримаєте сповіщення у вказаний канал.

#### Приклади алертів

| Назва | Метрика | Умова | Поріг |
|-------|---------|-------|-------|
| Мало замовлень | Замовлення за день | Менше | 5 |
| Великий чек | Сума замовлення | Більше | 50 000 грн |
| Товар закінчується | Залишок SKU-001 | Менше | 10 |

### 1.4. Звіти (13 типів)

1. У бічному меню оберіть **Звіти**.
2. Оберіть тип звіту:

| # | Звіт | Опис |
|---|------|------|
| 1 | Продажі | Деталізація по замовленнях, товарах, категоріях |
| 2 | Виручка | Динаміка виручки за днями/тижнями/місяцями |
| 3 | Товари | Рейтинг товарів за продажами |
| 4 | Клієнти | Статистика по клієнтах (нові, активні, LTV) |
| 5 | Залишки | Поточні залишки по складу |
| 6 | Замовлення | Список замовлень з усіма деталями |
| 7 | Повернення | Статистика повернень |
| 8 | Оплати | Звіт по оплатах (типи, суми, статуси) |
| 9 | Доставки | Статистика по способах доставки |
| 10 | Знижки | Ефективність промокодів та знижок |
| 11 | Менеджери | Продуктивність кожного менеджера |
| 12 | Маркетплейси | Продажі через підключені маркетплейси |
| 13 | Зведений | Загальний звіт з усіма ключовими показниками |

3. Вкажіть **Період** (від — до).
4. За необхідності додайте фільтри (категорія, менеджер, канал).
5. Натисніть **Сформувати**.
6. Звіт відобразиться на екрані (таблиця + графіки).
7. Для експорту натисніть одну з кнопок:
   - **PDF** — завантажити у форматі PDF.
   - **Excel** — завантажити у форматі .xlsx.
   - **CSV** — завантажити у форматі .csv.

## 2. Управління контентом

### 2.1. Банери

1. У бічному меню оберіть **Контент → Банери**.
2. Ви побачите список банерів у порядку відображення на сайті.

#### Створити банер

1. Натисніть **Додати банер**.
2. Заповніть форму:
   - **Назва** — для внутрішнього використання.
   - **Зображення** — завантажте зображення (рекомендований розмір відображається у підказці, наприклад 1200x400 px).
   - **Посилання** — URL, на який перейде користувач при натисканні на банер.
   - **Активний** — перемикач увімкнути/вимкнути відображення.
   - **Дата початку** та **Дата закінчення** — період показу (необов'язково).
3. Натисніть **Зберегти**.

#### Зміна порядку (drag-and-drop)

1. На сторінці списку банерів натисніть та утримуйте іконку **перетягування** (⠿) зліва від банера.
2. Перетягніть банер на потрібну позицію.
3. Відпустіть — новий порядок збережеться автоматично.

#### Видалити банер

1. Натисніть кнопку **Видалити** (іконка кошика) поряд з банером.
2. Підтвердіть видалення у діалоговому вікні.

### 2.2. Статичні сторінки CMS

1. У бічному меню оберіть **Контент → Сторінки**.
2. Ви побачите список статичних сторінок (Про нас, Умови доставки, Контакти тощо).
3. Натисніть на сторінку для редагування.
4. Використовуйте **WYSIWYG-редактор** для зміни вмісту:
   - Форматування тексту (жирний, курсив, заголовки).
   - Вставка зображень.
   - Вставка посилань.
   - Таблиці.
   - Списки.
5. Заповніть **SEO-поля**: Meta Title, Meta Description.
6. Натисніть **Зберегти**.

### 2.3. FAQ (питання та відповіді)

1. У бічному меню оберіть **Контент → FAQ**.
2. Ви побачите список питань та відповідей.

#### Додати FAQ

1. Натисніть **Додати питання**.
2. Введіть **Питання** (наприклад, "Як відстежити замовлення?").
3. Введіть **Відповідь** (використовуйте форматування для зручності).
4. Оберіть **Категорію** FAQ (Доставка, Оплата, Повернення тощо).
5. Вкажіть **Порядок** відображення.
6. Натисніть **Зберегти**.

#### Редагувати FAQ

1. Натисніть на питання у списку.
2. Внесіть зміни.
3. Натисніть **Зберегти**.

### 2.4. Блоки головної сторінки

1. У бічному меню оберіть **Контент → Головна сторінка**.
2. Ви побачите список блоків, що відображаються на головній:
   - **Банери** (слайдер).
   - **Категорії** (плитка).
   - **Акції** (товари зі знижками).
   - **Популярні товари**.
   - **Новинки**.
   - **Бренди**.
3. Для кожного блоку доступні дії:
   - **Увімкнути/вимкнути** — перемикач видимості.
   - **Змінити порядок** — перетягніть блок drag-and-drop.
   - **Налаштувати** — кількість товарів, заголовок блоку.
4. Натисніть **Зберегти** після внесення змін.

### 2.5. Публікації в соцмережі

1. У бічному меню оберіть **Контент → Соцмережі**.

#### Telegram-канал

1. Оберіть вкладку **Telegram**.
2. Натисніть **Нова публікація**.
3. Заповніть форму:
   - **Текст** — основний текст публікації (підтримує Markdown).
   - **Зображення** — додайте фото (необов'язково).
   - **Посилання на товар** — оберіть товар для автоматичної вставки посилання.
   - **Шаблон** — оберіть готовий шаблон (наприклад, "Новинка", "Акція", "Рекомендація").
4. Оберіть **Час публікації**:
   - **Зараз** — опублікувати негайно.
   - **Заплановано** — вкажіть дату та час.
5. Натисніть **Опублікувати** або **Запланувати**.

#### Instagram

1. Оберіть вкладку **Instagram**.
2. Аналогічний процес: текст, зображення, хештеги, час публікації.
3. Натисніть **Опублікувати** або **Запланувати**.

#### Шаблони

1. Перейдіть на вкладку **Шаблони**.
2. Натисніть **Створити шаблон**.
3. Вкажіть **Назву** та **Текст шаблону** з плейсхолдерами:
   - `{product_name}` — назва товару.
   - `{price}` — ціна.
   - `{old_price}` — стара ціна.
   - `{discount}` — відсоток знижки.
   - `{url}` — посилання на товар.
4. Натисніть **Зберегти**.

## 3. Налаштування

### 3.1. Загальні налаштування

1. У бічному меню оберіть **Налаштування → Загальні**.
2. Доступні поля:
   - **Назва магазину** — відображається у заголовку сайту та листах.
   - **Логотип** — завантажте зображення логотипу.
   - **Контактний email** — основна email-адреса.
   - **Контактний телефон** — відображається на сайті.
   - **Адреса** — фізична адреса магазину.
   - **Графік роботи** — дні та години.
   - **Соцмережі** — посилання на сторінки у Facebook, Instagram, Telegram, YouTube тощо.
3. Натисніть **Зберегти**.

### 3.2. Налаштування оплати

1. У бічному меню оберіть **Налаштування → Оплата**.
2. Ви побачите список платіжних систем.

#### LiqPay

1. Натисніть **LiqPay**.
2. Заповніть:
   - **Public Key** — публічний ключ з кабінету LiqPay.
   - **Private Key** — приватний ключ.
   - **Режим** — **Sandbox** (тестовий) або **Production** (бойовий).
3. Натисніть **Тестувати** — система виконає тестовий запит до API LiqPay та покаже результат.
4. Натисніть **Зберегти**.

#### Monobank (MonoPay)

1. Натисніть **Monobank**.
2. Заповніть:
   - **Token** — токен з кабінету Monobank Acquiring.
   - **Режим** — Sandbox або Production.
3. Натисніть **Тестувати** для перевірки з'єднання.
4. Натисніть **Зберегти**.

#### WayForPay

1. Натисніть **WayForPay**.
2. Заповніть:
   - **Merchant Account** — ідентифікатор мерчанта.
   - **Merchant Secret Key** — секретний ключ.
   - **Режим** — Sandbox або Production.
3. Натисніть **Тестувати**.
4. Натисніть **Зберегти**.

> **Увага:** Завжди тестуйте платіжну систему у режимі Sandbox перед переключенням на Production. Перевірте повний цикл: створення платежу → оплата → callback.

### 3.3. Налаштування доставки

1. У бічному меню оберіть **Налаштування → Доставка**.

#### Нова Пошта

1. Натисніть **Нова Пошта**.
2. Заповніть:
   - **API Key** — ключ з особистого кабінету Нової Пошти.
   - **Відправник** — дані відправника (ПІБ, телефон, адреса).
   - **Типове місто відправлення** — місто, з якого відправляєте.
   - **Типове відділення відправлення** — номер відділення.
3. Натисніть **Тестувати** — система зробить запит до API та покаже результат (наприклад, список міст).
4. Натисніть **Зберегти**.

#### Укрпошта

1. Натисніть **Укрпошта**.
2. Заповніть:
   - **API Bearer Token** — токен з кабінету Укрпошти.
   - **Counterparty UUID** — ідентифікатор контрагента.
   - **Дані відправника**.
3. Натисніть **Тестувати**.
4. Натисніть **Зберегти**.

### 3.4. Налаштування Email (SMTP)

1. У бічному меню оберіть **Налаштування → Email**.
2. Заповніть параметри SMTP-сервера:
   - **SMTP Host** — адреса сервера (наприклад, `smtp.gmail.com`).
   - **SMTP Port** — порт (587 для TLS, 465 для SSL).
   - **Username** — email-адреса або логін.
   - **Password** — пароль або App Password.
   - **Шифрування** — TLS або SSL.
   - **Ім'я відправника** — відображатиметься у полі "Від кого".
   - **Email відправника** — адреса у полі "Від кого".
3. Натисніть **Тестувати відправку**:
   - Введіть email-адресу для тестового листа.
   - Натисніть **Надіслати**.
   - Перевірте вхідні повідомлення. Якщо лист прийшов — налаштування коректні.
4. Натисніть **Зберегти**.

### 3.5. Telegram/Viber боти

1. У бічному меню оберіть **Налаштування → Месенджери**.

#### Telegram бот

1. Натисніть **Telegram**.
2. Заповніть:
   - **Bot Token** — токен бота, отриманий від @BotFather у Telegram.
   - **Webhook URL** — URL для отримання повідомлень (зазвичай заповнюється автоматично).
3. Натисніть **Встановити Webhook** — система зареєструє webhook у Telegram.
4. Налаштуйте **авто-відповіді**:
   - **Вітальне повідомлення** — текст, який отримає користувач при першому зверненні до бота.
   - **Відповідь на замовлення** — шаблон підтвердження замовлення.
   - **Відповідь при невідомій команді** — текст, якщо бот не розуміє запит.
5. Натисніть **Зберегти**.

#### Viber бот

1. Натисніть **Viber**.
2. Заповніть:
   - **Auth Token** — токен з Viber Admin Panel.
   - **Webhook URL**.
3. Натисніть **Встановити Webhook**.
4. Налаштуйте авто-відповіді аналогічно Telegram.
5. Натисніть **Зберегти**.

## 4. Ціноутворення

### 4.1. Персональні ціни

1. У бічному меню оберіть **Ціноутворення → Персональні ціни**.
2. Натисніть **Додати правило**.
3. Заповніть:
   - **Тип правила**:
     - **По користувачу** — ціна для конкретного клієнта.
     - **По категорії** — знижка на всі товари категорії для клієнта.
     - **По товару** — ціна на конкретний товар для клієнта.
   - **Клієнт** — оберіть клієнта зі списку або знайдіть за email.
   - **Товар** або **Категорія** (залежно від типу).
   - **Тип знижки** — Фіксована ціна / Відсоток знижки / Фіксована знижка.
   - **Значення** — числове значення (ціна або відсоток).
   - **Дата початку** та **Дата закінчення** (необов'язково).
4. Натисніть **Зберегти**.

### 4.2. Оптові правила

1. У бічному меню оберіть **Ціноутворення → Оптові ціни**.
2. Ви побачите три групи оптових цін:

| Група | Опис |
|-------|------|
| **Оптова 1** | Базовий оптовий рівень |
| **Оптова 2** | Середній рівень |
| **Оптова 3** | Максимальний рівень |

3. Натисніть на групу для налаштування.
4. Для кожної групи доступні:
   - **Загальна знижка** (%) — відсоток знижки від роздрібної ціни для всіх товарів.
   - **Знижка по категоріях** — різні відсотки для різних категорій.
   - **Індивідуальні ціни** — конкретна оптова ціна для окремих товарів.
5. Заповніть значення та натисніть **Зберегти**.

### 4.3. Купони

1. У бічному меню оберіть **Ціноутворення → Купони**.
2. Натисніть **Створити купон**.
3. Заповніть форму:
   - **Код купона** — унікальний код, який вводить клієнт (наприклад, `SALE20`).
   - **Тип знижки**:
     - **Відсоткова** — знижка у відсотках (наприклад, 20%).
     - **Фіксована** — знижка у гривнях (наприклад, 100 грн).
     - **Безкоштовна доставка**.
   - **Значення** — числове значення знижки.
   - **Мінімальна сума замовлення** — купон працює лише при замовленні від вказаної суми.
   - **Ліміт використань** — загальна кількість разів, скільки купон може бути використаний.
   - **Ліміт на клієнта** — скільки разів один клієнт може використати купон.
   - **Дата початку** та **Дата закінчення** — період дії.
   - **Категорії/товари** — обмеження на конкретні категорії або товари (необов'язково).
4. Натисніть **Зберегти**.

#### Перегляд статистики купона

1. У списку купонів натисніть на код купона.
2. Ви побачите:
   - Кількість використань.
   - Загальна сума знижок.
   - Список замовлень, де використано купон.

### 4.4. Програма лояльності

1. У бічному меню оберіть **Ціноутворення → Лояльність**.
2. Налаштуйте рівні лояльності:

#### Рівні

| Рівень | Мінімальна сума покупок | Множник нарахування |
|--------|------------------------|---------------------|
| Бронзовий | 0 грн (реєстрація) | 1x |
| Срібний | 5 000 грн | 1.5x |
| Золотий | 15 000 грн | 2x |

Для кожного рівня вкажіть:
- **Назва** рівня.
- **Мінімальна сума** покупок для досягнення.
- **Множник** нарахування балів (наприклад, 2x означає подвійне нарахування).
- **Відсоток нарахування** — базовий відсоток від суми покупки, що конвертується в бали.

#### Ручне нарахування балів

1. Перейдіть на вкладку **Нарахування**.
2. Натисніть **Нарахувати бали вручну**.
3. Оберіть **клієнта** зі списку.
4. Вкажіть **кількість балів**.
5. Вкажіть **причину** (подарунок, компенсація, бонус за промо тощо).
6. Натисніть **Нарахувати**.

## 5. SEO

### 5.1. SEO шаблони

1. У бічному меню оберіть **SEO → Шаблони**.
2. Налаштуйте шаблони автогенерації meta-тегів для товарів:

#### Meta Title шаблон

Приклад: `{product_name} — купити в {shop_name} | Ціна {price} грн`

Доступні змінні:
- `{product_name}` — назва товару.
- `{category_name}` — назва категорії.
- `{brand}` — бренд.
- `{price}` — ціна.
- `{shop_name}` — назва магазину.

#### Meta Description шаблон

Приклад: `{product_name} за ціною {price} грн. {short_description}. Доставка по Україні. Замовляйте в {shop_name}!`

3. Вкажіть шаблони та натисніть **Зберегти**.
4. Шаблони застосовуються автоматично до товарів, у яких не заповнені meta-теги вручну.

### 5.2. SEO аудит

1. У бічному меню оберіть **SEO → Аудит**.
2. Натисніть **Запустити аудит**.
3. Система просканує сайт та виявить проблеми:

#### Біті посилання (404)

- Список URL-адрес, що повертають помилку 404.
- Для кожного — сторінка, де знаходиться посилання.
- Дії: **Виправити** (змінити URL) або **Видалити** посилання.

#### Пропущені Meta

- Список товарів/сторінок без Meta Title.
- Список товарів/сторінок без Meta Description.
- Дії: **Заповнити вручну** або **Застосувати шаблон** для автоматичного заповнення.

#### Дублі

- Сторінки з однаковими Meta Title або Meta Description.
- Рекомендації щодо виправлення.

4. Після виправлення повторно запустіть аудит для перевірки.

## 6. Маркетплейси

1. У бічному меню оберіть **Маркетплейси**.
2. Ви побачите список підтримуваних маркетплейсів:

### 6.1. OLX

1. Натисніть **OLX → Підключити**.
2. Введіть:
   - **Client ID** та **Client Secret** — з кабінету розробника OLX.
   - **Refresh Token** — отримайте за інструкцією у документації OLX API.
3. Натисніть **Підключити**.
4. Після підключення налаштуйте:
   - **Категорії маппінг** — відповідність ваших категорій категоріям OLX.
   - **Товари для вивантаження** — оберіть, які товари публікувати.
   - **Оновлення цін** — автоматична синхронізація цін.
5. Натисніть **Синхронізувати** для початку вивантаження.

### 6.2. Rozetka

1. Натисніть **Rozetka → Підключити**.
2. Введіть:
   - **API Username** та **API Password** — дані з кабінету продавця Rozetka.
3. Налаштуйте маппінг категорій та товарів.
4. Натисніть **Синхронізувати**.

### 6.3. Prom.ua

1. Натисніть **Prom → Підключити**.
2. Введіть:
   - **Auth Token** — токен авторизації з налаштувань магазину на Prom.ua.
3. Налаштуйте маппінг.
4. Натисніть **Синхронізувати**.

### 6.4. Epicentr (Епіцентр)

1. Натисніть **Epicentr → Підключити**.
2. Введіть:
   - **API Key** — ключ з кабінету постачальника Епіцентру.
3. Налаштуйте маппінг категорій.
4. Натисніть **Синхронізувати**.

### 6.5. Синхронізація

Для всіх маркетплейсів доступні:

- **Автоматична синхронізація** — задайте інтервал (кожні 1/6/12/24 години).
- **Ручна синхронізація** — натисніть **Синхронізувати** для негайного оновлення.
- **Лог синхронізації** — перегляд історії синхронізацій, кількість оновлених товарів, помилки.
- **Статуси** — для кожного товару видно, чи опублікований він на маркетплейсі.

## 7. Система

### 7.1. Теми оформлення

1. У бічному меню оберіть **Система → Теми**.
2. Ви побачите список доступних тем оформлення сайту.
3. Для кожної теми доступний **Попередній перегляд**.
4. Натисніть **Активувати** для переключення на нову тему.
5. Зміни відобразяться на сайті негайно.

> **Рекомендація:** Перед активацією нової теми протестуйте її у попередньому перегляді, щоб переконатися, що все коректно відображається.

### 7.2. Режим обслуговування

1. У бічному меню оберіть **Система → Обслуговування**.
2. Увімкніть або вимкніть **Режим обслуговування**:
   - **Увімкнено** — відвідувачі бачать сторінку "Сайт на технічному обслуговуванні". Адміністратори можуть працювати як зазвичай.
   - **Вимкнено** — сайт працює у звичайному режимі.
3. Налаштуйте **текст повідомлення** для відвідувачів.
4. Натисніть **Зберегти**.

> **Використовуйте** режим обслуговування при масових оновленнях товарів, зміні теми або важливих налаштуваннях.

### 7.3. Аудит (журнал дій)

1. У бічному меню оберіть **Система → Аудит**.
2. Відкриється хронологічний журнал всіх дій у системі:

| Дата/час | Користувач | Дія | Деталі |
|----------|------------|-----|--------|
| 2026-03-20 14:30 | admin@shop.com | Зміна статусу замовлення | Замовлення #1234: Нове → Підтверджено |
| 2026-03-20 14:25 | manager@shop.com | Редагування товару | Товар SKU-001: ціна 200 → 250 грн |
| 2026-03-20 14:20 | admin@shop.com | Зміна налаштувань | Email SMTP оновлено |

3. Використовуйте **фільтри**:
   - **Користувач** — оберіть конкретного користувача.
   - **Тип дії** — замовлення, товари, налаштування, клієнти тощо.
   - **Дата** — діапазон дат.
4. Натисніть **Застосувати**.
5. Для експорту натисніть **Експорт** (Excel/CSV).

### 7.4. Webhook логи

1. У бічному меню оберіть **Система → Webhook логи**.
2. Ви побачите журнал вхідних та вихідних webhook-запитів:
   - **Дата/час** запиту.
   - **Напрямок** — вхідний (від платіжних систем, маркетплейсів) або вихідний.
   - **URL** — адреса запиту.
   - **Статус** — HTTP-код відповіді (200, 400, 500 тощо).
   - **Тіло запиту** — дані, що були надіслані (натисніть для перегляду).
   - **Тіло відповіді** — дані, що були отримані.
3. Використовуйте цей розділ для діагностики проблем з інтеграціями (платежі не підтверджуються, товари не синхронізуються тощо).

### 7.5. 2FA для адмінів

1. У бічному меню оберіть **Система → Безпека**.
2. У блоці **Двофакторна автентифікація для адміністраторів**:
   - **Обов'язково для адмінів** — перемикач, що вимагає 2FA для всіх акаунтів з роллю адміністратор.
   - **Обов'язково для менеджерів** — аналогічний перемикач для менеджерів.
3. Натисніть **Зберегти**.
4. Якщо 2FA стає обов'язковою, при наступному вході користувачі без налаштованої 2FA будуть перенаправлені на сторінку налаштування.

Процес налаштування 2FA для конкретного адміна:
1. Увійдіть в систему.
2. Перейдіть у **Налаштування профілю → Безпека**.
3. Натисніть **Увімкнути 2FA**.
4. Відскануйте QR-код додатком-автентифікатором.
5. Введіть 6-значний код для підтвердження.
6. Збережіть **резервні коди**.

### 7.6. Бекапи

1. У бічному меню оберіть **Система → Бекапи**.

#### Автоматичні бекапи

1. Налаштуйте розклад автоматичних бекапів:
   - **Частота** — щоденно / щотижня / щомісяця.
   - **Час** — о котрій годині виконувати (рекомендовано вночі).
   - **Зберігати останні** — кількість бекапів, що зберігаються (старіші видаляються автоматично).
2. Натисніть **Зберегти розклад**.

#### Ручне створення бекапу

1. Натисніть **Створити бекап зараз**.
2. Оберіть, що включити:
   - **База даних** — усі дані (замовлення, клієнти, товари, налаштування).
   - **Медіа-файли** — зображення товарів, банерів, завантажені файли.
   - **Все** — повний бекап.
3. Натисніть **Почати**.
4. Дочекайтеся завершення (прогрес-бар відображає поточний стан).
5. Після завершення бекап з'явиться у списку.

#### Список бекапів

- Для кожного бекапу відображається:
  - **Дата та час** створення.
  - **Тип** (автоматичний / ручний).
  - **Розмір** файлу.
  - **Вміст** (БД / Медіа / Повний).
- Доступні дії:
  - **Завантажити** — скачати бекап на ваш комп'ютер.
  - **Відновити** — відновити систему з цього бекапу (УВАГА: поточні дані будуть замінені).
  - **Видалити** — видалити бекап.

> **Увага:** Відновлення з бекапу замінить ВСІ поточні дані. Переконайтесь, що ви обрали правильний бекап. Рекомендовано створити поточний бекап перед відновленням.

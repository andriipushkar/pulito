- **Крони:** з 2 запланованих → 52 (прибирання, маркетплейс-синки, клієнтські розсилки з opt-out, SEO-автозаповнення). Усі живі, логуються.

## 📋 Звірити з бухгалтером (не податкова порада від мене)

- [ ] Правило визнання доходу для **маркетплейс-продажів** (валове vs нетто після комісії, дата виплати) — модуль показує валовий продаж.
- [ ] Дата доходу для **COD** (день виплати Нової Пошти, не доставки).
- [ ] **Інвалюта** — перерахунок у грн за курсом НБУ на дату надходження (не автоматизовано).
- [ ] **ЄСВ** — фіксований, поза модулем доходу.
- [ ] Заповни **дані ФОП** у `/admin/tax-report` (ІПН, ПІБ, група). _(Зараз: не задано)_

## ⏸️ Відкладено (за потреби — скажи)

- **XML для ДПС** — допишу, коли роздобудеш **XSD** конкретної форми (без неї валідний для завантаження файл не гарантувати; поки — ввід суми вручну в Кабінет/Дію).
- **Облік виплат маркетплейсів** (точний нетто-дохід) — окрема модель, якщо потрібна точність.
- **2FA** на фінансові розділи адмінки — посилення (не форсував, щоб не замкнути доступ).

## ⚙️ Інфраструктура (на замітку)

- Сервер на межі пам'яті (7.6GB; білд їсть ~5-6GB). Якщо деплої стануть критично довгими — розглянь +RAM або зменшення бандла.
- Деплой: **тільки через `./deploy.sh`** (атомний swap + auto-rollback). Не робити in-place build на живому.

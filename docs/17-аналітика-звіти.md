# 17. Аналітика та звіти

Pulito має вичерпну аналітичну підсистему — від real-time-дашборду й виявлення аномалій до RFM-сегментації, прогнозу попиту, прогнозу повторних покупок, прибутковості та податкової звітності ФОП. Цей розділ описує три блоки:

1. **Аналітика** (`/admin/analytics`, ~18 вкладок) — KPI, продажі, ABC, RFM, когорти, LTV, churn, воронка, канали, товарна аналітика, аномалії, прогноз попиту/покупок, аналітика пошуку, прибуток.
2. **Звіти** (`/admin/reports`) — 13 типів звітів, конструктор, експорт PDF/Excel/CSV, аналітичні алерти в Telegram/email.
3. **Податкова звітність** (`/admin/tax-report`) — ФОП, дохід касовим методом, CSV.

Окремо — **трекінг подій** (`ClientEvent`, клієнтський tracker, web-vitals).

Основні файли:

```
src/services/analytics.ts
src/services/analytics-reports.ts
src/services/dashboard.ts
src/services/churn.ts
src/services/anomaly.ts

src/services/search-intel.ts
src/services/customer-segments.ts
src/services/profit-report.ts
src/services/report-generator.ts
src/services/report-builder.ts
src/services/export.ts
src/services/analytics-pdf.ts
src/services/tax-report.ts
src/services/client-events.ts
src/lib/event-tracker.ts
src/components/admin/analytics/
```

> **Київський час.** Усі дати (аналітика, звіти, податки) — у `Europe/Kyiv` через хелпери `kyivMidnightUtc / kyivNextDayUtc / kyivDateIso / todayKyiv / daysAgoKyiv`. Будь-який фільтр діапазону дат має йти через них.

## Частина 1. Аналітика

### Дашборд

`src/services/dashboard.ts` — `getDashboardStats()` (усі метрики одним проходом):

- **Сьогодні:** замовлень, виручка, нові замовлення;
- **Користувачі:** всього, гуртовиків, нових за тиждень, гуртових запитів;
- **Товари:** активних, без залишку, з низьким запасом, без штрих-кодів;
- **Тренди:** top-5 товарів за 30 днів, останні 5 замовлень;
- **Часові ряди:** виручка по днях (7 днів), замовлення по годинах (сьогодні, Kyiv TZ).

### Базова аналітика

`src/services/analytics.ts`:

| Функція                                              | Опис                                                                                      |
| ---------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| `getConversionFunnel(days)`                          | агрегує `DailyFunnelStats`: перегляди → замовлення, конверсія кожного кроку + від початку |
| `getCohortAnalysis(months)`                          | когорти за місяцем реєстрації, % повторних покупок по місяцях утримання                   |
| `getABCAnalysis(days)`                               | ABC-класифікація за внеском у виручку: A (80%), B (15%), C (5%)                           |
| `getOrderProcessingTime(days, fromStatus, toStatus)` | час обробки між статусами (медіана, 95-перцентиль)                                        |

### Розширена аналітика

`src/services/analytics-reports.ts`:

| Функція                         | Опис                                                                                              |
| ------------------------------- | ------------------------------------------------------------------------------------------------- |
| `getStockAnalytics(days)`       | критичні запаси (<14 днів), dead stock (60+ днів без продажу), обороти                            |
| `getPriceAnalytics(days)`       | зміни цін, вплив промо (продажі до/після)                                                         |
| `getChannelAnalytics(days)`     | замовлення по джерелах (web/telegram_bot/viber_bot), UTM (source/medium/campaign), `ChannelStats` |
| `getGeographyAnalytics(days)`   | замовлення по містах (top-50), метод доставки, % виручки по регіонах, середній чек                |
| `getCustomerLTV(days)`          | top-50 клієнтів за LTV, розподіл по бакетах <500 / 500–2k / 2k–5k / 5k–20k / >20k ₴               |
| `getCustomerSegmentation(days)` | RFM-сегменти (10-сек preview)                                                                     |
| `getAllSegmentUserIds(days)`    | повні списки user-ID по RFM-сегментах (для кампаній, без лімітів)                                 |

**RFM-сегменти:** `champions`, `loyal`, `recent`, `promising`, `at_risk`, `sleeping`, `lost`, `new`.

### Churn / відтік

`src/services/churn.ts` — `getChurnRadar(opts)`: top-10 клієнтів з найвищим LTV, які не замовляли 30+ днів (з телефоном для переконтакту).

### Аномалії

`src/services/anomaly.ts` — `detectAnomalies()`: порівнює метрики «сьогодні» з 14-денним baseline; відхилення ±50% → severity `info/warning/danger`. Метрики: замовлення, виручка, rate скасування, конверсія.

### Прогноз попиту

`src/services/demand-forecast.ts` — `getDemandForecast(options)` (також сторінка `/admin/forecasting`): прогноз за 90 днів — середня денна реалізація → дні до out-of-stock. Urgency: `critical / soon / ok / over-stock / no-sales`.

### Прогноз покупок

`src/services/purchase-prediction.ts`:

| Функція                       | Опис                                                                          |
| ----------------------------- | ----------------------------------------------------------------------------- |
| `buildPredictions()`          | історія (2+ покупок) → середній інтервал → передбачена дата наступної покупки |
| `processReminders(daysAhead)` | in-app/push/email-нагадування про restock за 3 дні до predicted date          |
| `getUserPredictions(userId)`  | прогнози для окремого користувача                                             |

### Аналітика пошуку

`src/services/search-intel.ts` (сторінка `/admin/search-intel`):

| Функція                                 | Опис                                                                 |
| --------------------------------------- | -------------------------------------------------------------------- |
| `getTopZeroResultSearches(limit)`       | top-30 запитів без результатів (втрачені продажі)                    |
| `getTopAllSearches(limit)`              | top-30 усіх запитів                                                  |
| `generateSearchInsights(entries, opts)` | AI-інсайти (Claude/Gemini): які товари додати / синоніми / категорії |

### Прибуток

`src/services/profit-report.ts` — `getProfitReport(opts)` (сторінка `/admin/analytics/profit`):

- **Прибуток = виручка − COGS** за період (Kyiv-дати);
- собівартість = `supplierCostAtSale ?? product.cost`;
- розбивка по товарах / категоріях / постачальниках;
- `unknownCostUnits` — товари без собівартості.

```ts
{
  totals: { revenue, cogs, profit, margin, units, orders, unknownCostUnits },
  topProducts,   // 20 найприбутковіших
  lossProducts,  // негативний профіт (дешевше за собівартість)
  byCategory, bySupplier
}
```

### Вкладки аналітики

`src/components/admin/analytics/` — `/admin/analytics` рендерить ~18 вкладок через `next/dynamic` (Recharts lazy-load):

| Вкладка                            | Що показує                                       |
| ---------------------------------- | ------------------------------------------------ |
| Performance / KPI                  | замовлень, виручка, середній чек, конверсія      |
| ConversionFunnel                   | графік воронки                                   |
| CohortAnalysis                     | утримання когорт по місяцях                      |
| ABCAnalysis                        | таблиця товарів за A/B/C                         |
| StockAnalytics                     | критичні запаси, dead stock, обороти             |
| PriceAnalytics                     | зміни цін, вплив промо                           |
| ChannelAnalytics                   | джерела, UTM, конверсія по каналах               |
| GeographyAnalytics                 | замовлення по містах, метод доставки             |
| CustomerLTV                        | top-50 клієнтів, розподіл LTV                    |
| CustomerSegmentation / RFMAnalysis | RFM-сегменти, матриця Recency/Frequency/Monetary |
| ChurnPrediction                    | на ризику відтоку (top-10)                       |
| AlertsConfig                       | налаштування алертів (метрика + threshold)       |

Базові вкладки: dashboard, sales, products, clients, orders. Угорі — вибір діапазону днів і експорт PDF для вкладки чи summary.

`src/services/report-generator.ts` (сторінка `/admin/reports`):

| Ключ                    | Назва                                                     |
| ----------------------- | --------------------------------------------------------- |
| `sales_summary`         | Звіт про продажі                                          |
| `products_stock`        | Залишки товарів                                           |
| `orders_by_status`      | Замовлення за статусом                                    |
| `clients_activity`      | Активність клієнтів                                       |
| `wholesale_report`      | Гуртові продажі                                           |
| `delivery_report`       | Звіт по доставках (ТТН, місто, адреса, вартість)          |
| `financial_report`      | Фінансовий звіт (виручка, знижки, доставка, чистий дохід) |
| `returns_cancellations` | Повернення та скасування                                  |

### Конструктор звітів

`src/services/report-builder.ts` (сторінка `/admin/reports/builder`):

```ts
type Dimension = 'status' | 'clientType' | 'deliveryMethod' | 'paymentMethod' | 'monthYear';
type Metric = 'orderCount' | 'totalRevenue' | 'avgCheck';
```

Користувач обирає entity (orders/products/users), вимір і метрики + колонки.

### Експорт даних

`src/services/export.ts`:

| Функція                  | Опис                                                                       |
| ------------------------ | -------------------------------------------------------------------------- |
| `exportOrders(params)`   | фільтри: статус, тип клієнта, дати, метод доставки/оплати, менеджер, пошук |
| `exportClients(params)`  | роль, статус гуртовика, група, заблокований, дати                          |
| `exportProducts(params)` | категорія, активні, налаштування колонок                                   |

XLSX автоматично підганяє ширину колон (макс 50 символів). CSV-експорт має захист від formula-injection.

### Бібліотеки експорту

| Бібліотека                                   | Використання                               |
| -------------------------------------------- | ------------------------------------------ |
| **PDFKit** (`src/services/analytics-pdf.ts`) | PDF-звіти (ландшафт), брендування, таблиці |
| **ExcelJS** (`src/services/export.ts`)       | XLSX/CSV з форматуванням і ширинами колон  |
| **Recharts**                                 | графіки в компонентах (lazy-load)          |
| **Redis**                                    | кеш згенерованих звітів (~5 хв TTL)        |

Підписки на звіти (subscriptions) — email за розкладом (daily/weekly/monthly).

### Аналітичні алерти

`src/services/jobs/check-analytics-alerts.ts` — `checkAnalyticsAlerts()` (щодня): перевіряє налаштовані `AnalyticsAlert` проти метрик дня (Kyiv TZ), надсилає **Telegram / email** при перевищенні threshold.

Метрики: `daily_revenue`, `daily_orders`, `avg_check`, `stock_zero`, `new_users`, `cancelled_orders`. Структура алерту: `{ metric, condition: 'above'|'below', threshold }`.

## Частина 3. Податкова звітність (ФОП)

`src/services/tax-report.ts` (сторінка `/admin/tax-report`):

| Функція                  | Опис                                                                                                                                             |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `getTaxReport(input)`    | дохід ФОП за період касовим методом (за `Payment.paidAt`), розділяє прямі канали vs маркетплейси, рахує податок, порівнює з річним лімітом групи |
| `taxReportToCsv(report)` | експорт CSV (укр. Excel, роздільник `;`)                                                                                                         |

**Касовий метод:** дохід рахується за датою отримання платежу, не за датою замовлення. Повернення — за `RefundRequest.refundedAt`.

**Розділення доходу:**

- **direct** (сайт / telegram / viber) — точна каса;
- **marketplace** (Rozetka / Prom / OLX / Epicentr) — показано **валовий** продаж (реальні виплати/комісії не трекуються);
- **COD** (накладений платіж) — інформаційний прапор (дата = коли позначено оплаченим).

**Параметри:**

```ts
{
  year, quarter?,        // період
  group: 1 | 2 | 3,      // ФОП-група (3 → ratePercent; 1/2 → fixedTax)
  ratePercent?,          // ставка для групи 3
  fixedTax?              // фіксований податок для груп 1/2
}
```

Налаштування групи/ставки зберігаються в `SiteSetting` (`TaxSystem`, без окремої міграції). Попередження власнику: перевищення річного ліміту групи → ризик втрати єдиного податку; XML-вивантаження відкладено до отримання XSD.

## Трекінг подій

### Серверний запис

`src/services/client-events.ts` (модель `ClientEvent`):

| Функція                           | Опис                                 |
| --------------------------------- | ------------------------------------ |
| `recordClientEvent(input)`        | запис однієї події з валідацією типу |
| `recordClientEventsBatch(events)` | пакетний запис                       |

Дозволені типи (`ALLOWED_EVENT_TYPES`): `page_view`, `product_view`, `category_view`, `search`, `add_to_cart`, `remove_from_cart`, `cart_view`, `checkout_started`, `checkout_step`, `order_completed`, `wishlist_add`, `wishlist_remove`, `instagram_post_published/likes/comments`, `email_open`.

### Клієнтський трекер

`src/lib/event-tracker.ts` (`'use client'`):

| Функція             | Опис                                                                                   |
| ------------------- | -------------------------------------------------------------------------------------- |
| `trackEvent(input)` | додає в чергу (max 50), надсилає batch кожні 5 с або на вихід (`navigator.sendBeacon`) |
| `flushTracker()`    | примусова відправка черги                                                              |

Session ID живе в `sessionStorage` (переживає перезавантаження в межах сесії).

### Web Vitals

Метрики продуктивності (LCP/CLS/INP тощо) збираються на клієнті й живлять моніторинг швидкості сторінок.

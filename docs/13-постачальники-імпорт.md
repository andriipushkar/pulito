# 13. Постачальники та імпорт товарів

Цей розділ описує дві пов'язані підсистеми Pulito:

1. **Імпорт товарів** — разове/планове наповнення каталогу з файлів та фідів (`src/services/import.ts`, адмінка `/admin/import`).
2. **Модуль постачальників** (`SupplierChannel`) — авто-синхронізація **цін і залишків** із фідів постачальників за двома моделями виконання: **консигнація** (товар у нас на складі) і **дропшип** (відвантажує постачальник). Модуль повністю реалізовано й розгорнуто (2026-06-13/14).

## Два режими каналу постачальника (`syncMode`)

`SupplierChannel.syncMode` (enum `SupplierSyncMode`) розділяє два **окремі кодові шляхи** синхронізації. Єдина точка входу — `runSupplierSync()` у `src/services/suppliers/dispatch.ts`, яка маршрутизує за режимом:

| Режим                   | Шлях                                                     | Що робить                                                                                                                                                                         |
| ----------------------- | -------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `catalog_import` (деф.) | `syncSupplierChannel()` → `importProducts` (`import.ts`) | Легасі-каталог-імпортер: тягне фід, **створює/оновлює** товари по `code`/`barcode`/`name`, `priceRetail = ціна з фіду` напряму (без націнки/cost/прив'язки)                       |
| `price_stock`           | `syncSupplierConsignment()` (`suppliers/sync.ts`)        | Консигнація/дропшип: фід керує **закупівельною ціною** (`Product.cost`) → роздрібна = `cost + markup`, і **залишком** лише **вже прив'язаних** товарів. Ніколи не створює товарів |

> Спільні примітиви парсингу (кодування, ціна, кількість) винесені в `src/utils/feed-parse.ts`, тож обидва шляхи не можуть мовчки розійтися в обробці.

## Модуль постачальників (`SupplierChannel`)

Модель `SupplierChannel` (`prisma/schema/integration.prisma`). Адмінка — секція `SupplierChannelsSection` на сторінці `/admin/import` + сторінки `/admin/supplier-channels/[id]/{import,reconciliation,products,logs}`. Сервіси — `src/services/suppliers/`.

### Налаштування каналу

| Поле                                        | Призначення                                                                                        |
| ------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `feedUrl`, `format`, `auth`, `scheduleCron` | Джерело фіду, формат, авторизація, розклад                                                         |
| `syncMode`                                  | `catalog_import` / `price_stock`                                                                   |
| `fulfillment`                               | `own_stock` (консигнація) / `dropship`                                                             |
| `markupType`, `markupValue`                 | Націнка: `percent` (cost + cost·value%) або `fixed` (cost + value ₴)                               |
| `minPrice`                                  | Мінімальна роздрібна ціна (поріг — ціна ніколи нижче)                                              |
| `stockPolicy`                               | `hide` (ховати при 0) / `backorder` («під замовлення» — продаж при 0)                              |
| `feedCurrencyRate`                          | Курс конвертації ціни фіду в ₴ (деф. 1; cost = `purchasePrice × rate`)                             |
| `reserveAware`                              | Віднімати наші відкриті (невідвантажені) резерви від кількості з фіду                              |
| `zeroMissing`                               | Обнуляти прив'язані товари, відсутні у фіді (лише якщо збіглося ≥50% — захист від обрізаного фіду) |
| `notifyTelegramChatId`, `notifyEmail`       | Канали сповіщення дропшип-постачальника                                                            |
| `isActive`, `lastSyncAt`                    | Активність, час останньої синхронізації                                                            |

**Дві моделі виконання** використовують той самий рушій синхронізації; різниця — лише **після замовлення**:

- **own_stock (консигнація):** товар фізично у нас, ми відвантажуємо.
- **dropship:** відвантажує постачальник; при замовленні йому йде сповіщення (`notifyDropshipSuppliers` → Telegram `notifySupplierDropshipOrder` + email-fallback).

### Прив'язка товарів (first-import)

Консигнаційний sync **ніколи не створює** товарів — прив'язка є свідомим, **вручну підтвердженим** кроком (`suppliers/first-import.ts`):

- `previewSupplierFeed(channelId)` — класифікує рядки фіду: **linked** / **suggested** (збіг по `code==sku` або `barcode`) / **unmatched**. Read-only.
- `linkSupplierProducts(channelId, links)` — проставляє `Product.supplierId` + `supplierSku` (ключ збігу `@@unique([supplierId, supplierSku])`). Приймає `productId` АБО `productCode`. Захист від дубль-SKU й прив'язки до іншого каналу.
- `unlinkSupplierProducts()` — відв'язка (`POST .../[id]/unlink`).
- UI: сторінка `/admin/supplier-channels/[id]/import` (таблиця збігів, `ProductPicker` з debounce-пошуком на unmatched-рядках).

### Рушій синхронізації (`suppliers/sync.ts`)

`syncSupplierConsignment(channelId, {dryRun})`:

1. **Advisory-lock** на канал (`pg_try_advisory_lock`, namespace `SSYN`) — серіалізує синхронізації одного каналу (cron vs «синхронізувати зараз» vs два cron-тіки). Інші канали синхронізуються паралельно. dry-run lock не бере.
2. Створює `SupplierSyncLog` (status `running`) **до** fetch — щоб помилка фіду все одно лишила слід.
3. `fetchSupplierFeedBuffer()` → `parseSupplierFeed()`.
4. Бере **прив'язані** товари (`supplierId = channel.id`, `supplierSku != null`), мапа по SKU.
5. Для кожного рядка фіду:
   - **Залишок:** `available=false` → 0. При `reserveAware` віднімаються відкриті резерви (groupBy `orderItem` по нетермінальних замовленнях).
   - **Ціна:** лише якщо фід дав придатну `purchasePrice` (null → **тільки залишок**, ціна/cost не чіпаються — bad-row не опублікує товар за 0 ₴). `newCost = round2(purchasePrice × rate)`, `newRetail = computeRetailPrice(cost, markupType, markupValue, minPrice)` (override товару перекриває націнку каналу). На зміну роздрібної — запис `PriceHistory`.
   - Записує лише за наявності змін; `version++`.
6. Записи **flush'аться чанками по 200** в окремих транзакціях (атомарно по чанку — фід на 10k не зірве interactive-transaction timeout).
7. `zeroMissing` (опц.): обнуляє прив'язані товари, відсутні у фіді, але **лише якщо** збіглося ≥50% (захист від обрізаного завантаження).
8. **Fail-safe:** будь-яка помилка fetch/parse → `SupplierSyncLog` = `failed`, **нічого не записується** (залишки/ціни лишаються як були), помилка пробрасується. Транзиторний збій постачальника не може обнулити каталог.

`computeRetailPrice` (`suppliers/markup.ts`) рахує через `@/utils/money` (копійко-точно): percent/fixed + floor по `minPrice`.

### Резерв та політика наявності

- **Резерв при замовленні:** `createOrder` декрементує `product.quantity` з gte-guard (резервує і дропшип-товари). Snapshot `OrderItem.supplierCostAtSale` пишеться для звірки.
- **Backorder («під замовлення»):** `Product.allowBackorder` (зі `stockPolicy='backorder'`). При 0 товар лишається продаваним: кошик пропускає stock-guard, вітрина показує «Під замовлення». Не застосовується для обраного варіанта (варіанти — реальний залишок).
- **reserveAware:** коли фід постачальника НЕ зменшується на наші замовлення — sync пише `quantity = supplierQty − openReserved`, щоб не переекспонувати продане.

### Виплати постачальникам та звірка

- **Reconciliation** (`suppliers/reconciliation.ts`): `getSupplierReconciliation({from, to, supplierId?})` — по кожному постачальнику: units sold / revenue / cost (`Σ supplierCostAtSale × qty`) / margin / paid / **balance (cost − paid)**. Київські дати, виключає скасовані. Роут `supplier-channels/reconciliation`, сторінка `/admin/supplier-channels/[id]/reconciliation` з пресетами дат.
- **Виплати** (`SupplierPayout`): `createSupplierPayout(supplierId, amount, periodFrom/To, note)`. Форма «Записати виплату» на сторінці звірки; колонки paid/balance.

### Cron-синхронізація

Cron `src/app/api/v1/cron/sync-supplier-channels/route.ts` (у crontab `0 */2 * * *`). Per-channel due-calc через `CronExpressionParser` (between `lastSyncAt` і `now` за `scheduleCron`) гейтить реальні запуски. Той самий dispatcher (`runSupplierSync`), що й кнопка «синхронізувати». На збій каналу — Telegram `notifyManagerSupplierFailed` (покриває ризик мовчазного stale-фіду). Лог синхронізацій — `/admin/supplier-channels/[id]/logs`.

## Імпорт товарів (`/admin/import`)

Сервіс `src/services/import.ts` (`importProducts`, `ImportResult`). Створює/оновлює товари з файлу або фіду, матчить по `code`/`barcode`/`name`.

### Підтримувані формати

| Формат                | Парсер                       | Нотатки                                                                                                                                        |
| --------------------- | ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **YML**               | `fast-xml-parser` (XXE-safe) | `vendorCode`→sku, `purchase_price`/`price`, `quantity_in_stock`/`stock_quantity`                                                               |
| **CSV**               | sniff `;` vs `,`             | header-map UA/RU/EN → sku/price/quantity/name/barcode                                                                                          |
| **XLSX**              | ExcelJS (dynamic import)     | `COLUMN_MAP` (укр→ключ)                                                                                                                        |
| **1С-ZIP CommerceML** | `adm-zip` (PK magic) + XML   | ZIP зі split `import.xml` + `offers.xml` — merge каталогу між файлами; `КоммерческаяИнформация` → Каталог Товар + ПакетПредложений Предложення |
| **Google Sheets**     | —                            | `normalizeFeedUrl()` переписує `/edit`-лінк → `/export?format=csv` (зберігає gid); адмін обирає формат `csv`                                   |

### Надійність імпорту (hardening 2026-06-14)

Спільні примітиви в `src/utils/feed-parse.ts`:

- **Кодування (`decodeFeedBuffer`):** чесно визначає charset — BOM (UTF-8/UTF-16), XML-prolog `encoding="..."`, інакше UTF-8 з fallback на **windows-1251** при появі replacement-символів. Українські 1С/Excel-CSV часто cp1251 — без цього кирилиця в заголовках/SKU перетворюється на mojibake й мовчки ламає матчинг.
- **Ціна (`parseFeedPrice`):** locale-aware (UA `1 299,90`, US `1,299.90`, EU `1.299,90`, grouped `1,234,567.89`) **без класичного 1000× багу** (одинокий 3-значний хвіст — це тисячі, не десяткові). `≤0`/`NaN`/негатив → null (нульова закупівельна = mis-mapped фід, не «безкоштовний товар»).
- **Кількість (`parseFeedQuantity`):** невід'ємне ціле; прибирає пробіли (`1 000`→1000); неприпустиме → 0.

Інші гарантії:

- **SSRF-захист (`isAllowedUrl`):** блокує internal/private URL (IPv4 + IPv6: loopback, private, CGNAT, link-local, unique-local, IPv4-mapped); лише http/https; `localhost`/`.local`/`.internal` заборонені.
- **Стрім + ліміти:** таймаут 60 с, кеп 50 МБ (OOM-захист на великих фідах).
- **Транзакції + lock:** синхронізація під advisory-lock, атомарний rollback.
- **Дедуплікація** рядків фіду.
- **Зображення:** завантаження з таймаутом 10 с, кеп 5 МБ, дозволені `image/jpeg|png|webp`.
- **CommerceML тип ціни:** обирається правильний тип ціни з кількох.

`importProducts` повертає `ImportResult`: `totalRows`, `created`, `updated`, `skipped`, `imagesImported/Failed`, `variantsCreated/Updated`, `errors[]`, `durationMs`. Лог — `ImportLog`.

## Чим консигнаційний sync відрізняється від каталог-імпорту

| Аспект             | `catalog_import`           | `price_stock` (консигнація)                 |
| ------------------ | -------------------------- | ------------------------------------------- |
| Створення товарів  | Так (по code/barcode/name) | **Ніколи** (лише прив'язані)                |
| Роздрібна ціна     | = ціна з фіду напряму      | = `cost + markup` (з override/min/currency) |
| `Product.cost`     | не пише                    | пише закупівельну ціну                      |
| Прив'язка          | по полях товару            | по `supplierSku` (manual first-import)      |
| Виплати/звірка     | немає                      | `SupplierPayout` + reconciliation           |
| Дропшип-сповіщення | немає                      | так (`notifyDropshipSuppliers`)             |

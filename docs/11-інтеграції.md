# 11. Зовнішні інтеграції

Цей розділ — повний довідник усіх зовнішніх інтеграцій платформи **Pulito**: платіжні системи, служби доставки, боти, соцмережі, ERP, аналітика, email, push, сховище, моніторинг, AI та сервіси Google. Платформа інтегрує понад **20 зовнішніх API**.

## Як це працює: два способи налаштування

Pulito має дві моделі зберігання облікових даних для інтеграцій:

1. **Адмінка із шифруванням (DB-first)** — credentials лежать у таблиці `SiteSetting` у **зашифрованому вигляді** (AES-256-GCM через `@/lib/encryption`). Це джерело правди. Власник магазину змінює ключі без редеплою. Цим шляхом ідуть **платіжні системи, боти, соцмережі, маркетплейси, частина доставки**.
2. **ENV (змінні оточення)** — для частини інтеграцій (трекінг GA4/CAPI, push, сховище, моніторинг, Google OAuth) ключі читаються з `process.env`. Для деяких сервісів ENV лишився **fallback'ом**: спершу читається БД, якщо там порожньо — ENV.

> 🔐 **Важливо:** платіжні ключі (LiqPay, Monobank, WayForPay) живуть **лише** в `/admin → Налаштування платежів` із шифруванням. ENV-fallback для платежів **видалено** свідомо (див. розділ 05).

**Загальний патерн (`channel-config.ts`):** канальні credentials (`telegram`, `facebook`, `instagram`, `tiktok`, `olx`, `rozetka`, `prom`, `epicentrk`) зберігаються під ключем `channel_<тип>`. Чутливі поля (`apiKey`, `apiSecret`, `apiToken`, `accessToken`, `refreshToken`, `clientSecret`, `password`, `botToken`, `pageAccessToken`, `appSecret`) шифруються. При збереженні робиться **merge**: порожнє чутливе поле **не перезаписує** збереженого значення (захист від випадкового стирання токенів). У відповідях API значення повертаються в masked-вигляді (`•••...xxxx`).

## Платежі

Платіжні провайдери налаштовуються в `/admin/payment-settings` (UI), зберігаються через `PUT /api/v1/admin/payment-settings` із шифруванням чутливих полів. Credentials завантажуються через `integration-credentials.ts` (`getLiqPayCreds`, `getMonobankCreds`, `getWayForPayCreds`). **ENV-fallback відсутній.**

Спільна логіка ініціації та обробки — у `src/services/payment.ts` (`initiatePayment`, `handlePaymentCallback`, `refundPayment`). Провайдери — у `src/services/payment-providers/`. Вебхуки — у `src/app/api/webhooks/{liqpay,monobank,wayforpay}/`.

### LiqPay

| Параметр     | Значення                                                                                                                                    |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| Файли        | `payment-providers/liqpay.ts`, `api/webhooks/liqpay/route.ts`                                                                               |
| Credentials  | `payment_liqpay_public_key` (plaintext), `payment_liqpay_private_key` (encrypted), `payment_liqpay_sandbox`, `payment_liqpay_paypart_count` |
| Завантаження | `getLiqPayCreds()` (DB-only)                                                                                                                |
| Сума         | у гривнях (без копійок)                                                                                                                     |
| Підпис       | SHA1(privateKey + base64data + privateKey)                                                                                                  |
| Endpoint     | `https://www.liqpay.ua/api/3/checkout`                                                                                                      |

**Можливості:**

- `createPayment()` — стандартна оплата (base64-payload, version 3, action `pay`).
- `createPaypartPayment()` — оплата частинами (paytypes `paypart`, `instalment_count` 2–24).
- `createApplePayPayment()` / `createGooglePayPayment()` — Apple Pay / Google Pay (paytypes `apay`/`gpay`).
- `refundPayment()` — повернення (action `refund`); статус `reversed`→refunded, `refund_wait`→processing.

**Вебхук:** `formData` з полями `data` (base64) + `signature`. Верифікація `verifyCallback()` через `safeEqual()` (constant-time). Статуси: `success`/`sandbox`→оплачено, `failure`/`error`/`reversed`→невдача, решта→в обробці. Ідемпотентність через Redis `SET NX` на `webhook:liqpay:{transactionId}` (TTL 24 год). Rate-limit 100 вебхуків/хв на IP.

**Покрокове налаштування:**

1. Зареєструйтесь на **https://www.liqpay.ua/uk** → **Зареєструватись** (потрібна картка ПриватБанку, підтвердження email).
2. Увійдіть у бізнес-кабінет **https://www.liqpay.ua/uk/adminbusiness** → **API** → **Ключі API**.
3. Скопіюйте **Public Key** і **Private Key** (для тестів — вкладка «Тестовий режим»).
4. У кабінеті **API** → **Налаштування** вкажіть **Server-Server URL**: `https://ваш-домен/api/webhooks/liqpay`.
5. Вставте ключі в **`/admin/payment-settings`** (Public Key — plaintext, Private Key — шифрується). Sandbox вмикається тут же тоглом «LiqPay Sandbox-режим» (перед запуском обов'язково ВИМКНУТИ). Оплата частинами — тогл «ПриватБанк — Оплата частинами» + кількість місяців 2–24.

> Ключі вводяться лише в адмінці з шифруванням, ENV-fallback для платежів видалено.

### Monobank

| Параметр     | Значення                                                                           |
| ------------ | ---------------------------------------------------------------------------------- |
| Файли        | `payment-providers/monobank.ts`, `api/webhooks/monobank/route.ts`                  |
| Credentials  | `payment_monobank_token` (encrypted)                                               |
| Завантаження | `getMonobankCreds()` (DB-only)                                                     |
| Сума         | у **копійках** (×100 на відправку, ÷100 на приймання)                              |
| Підпис       | ECDSA SHA256 публічним ключем Mono (кешується 24 год, авто-інвалідація на ротації) |
| Endpoint     | `https://api.monobank.ua/api/merchant/invoice/create`                              |

**Можливості:**

- `createPayment()` — `invoice/create`, header `X-Token`, ccy 980 (UAH), reference `order_{id}`.
- `refundPayment()` — `invoice/cancel`, унікальний `extRef` на кожну спробу (дозволяє часткові повернення).

**Вебхук:** JSON body + header `X-Sign`. Публічний ключ тягнеться з `/api/merchant/pubkey`, кешується 24 год; при невдалій верифікації кеш скидається й робиться один retry (обробка ротації ключа). Вебхуки старші 10 хв відхиляються. Ідемпотентність `webhook:monobank:{invoiceId}`.

**Покрокове налаштування:**

1. Підключіть інтернет-еквайринг через **Monobank для бізнесу**: **https://www.monobank.ua/business** (тестовий токен можна отримати через підтримку Monobank).
2. Після підключення отримаєте **X-Token** еквайрингу.
3. Вставте токен у **`/admin/payment-settings`** (поле Monobank token — шифрується).
4. Webhook URL не вводиться вручну — додаток сам передає `https://ваш-домен/api/webhooks/monobank` при створенні інвойсу.

### WayForPay

| Параметр     | Значення                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------- |
| Файли        | `payment-providers/wayforpay.ts`, `api/webhooks/wayforpay/route.ts`                          |
| Credentials  | `payment_wayforpay_merchant_account` (plaintext), `payment_wayforpay_secret_key` (encrypted) |
| Завантаження | `getWayForPayCreds()` (DB-only)                                                              |
| Сума         | у гривнях                                                                                    |
| Підпис       | HMAC-MD5 по semicolon-joined масиву полів                                                    |
| Endpoint     | `https://api.wayforpay.com/api`                                                              |

**Можливості:** `createPayment()` (`CREATE_INVOICE`, orderReference з timestamp для унікальності, фільтр платіжних систем card/privat24/googlePay/applePay), `refundPayment()` (`REFUND`). Підтримка Apple/Google Pay.

**Вебхук:** JSON з `merchantSignature`; верифікація `verifyCallback()` через `safeEqual()`. Статуси: `Approved`→success; `Declined`/`Refunded`/`Voided`/`Expired`→failure. Відповідь магазину підписана (`createCallbackResponse`, status `accept`/`refuse`).

**Покрокове налаштування:**

1. Зареєструйте мерчанта на **https://wayforpay.com/** → **Зареєструватись** (підтвердження email + верифікація).
2. У **Особистий кабінет** → **Налаштування магазину** скопіюйте **Merchant Account** (ідентифікатор) і **Secret Key**.
3. Там же, у **Налаштування магазину** → **URL для відповідей**, вкажіть `https://ваш-домен/api/webhooks/wayforpay`.
4. Вставте Merchant Account (plaintext) і Secret Key (шифрується) у **`/admin/payment-settings`**.
5. Apple/Google Pay через WayForPay: відкрийте заявку на Apple Pay merchant у кабінеті (потребує домен), завантажте файл валідації за шляхом `/.well-known/apple-developer-merchantid-domain-association`, потім увімкніть тогли в `/admin/payment-settings` → секція «Apple Pay та Google Pay».

### Спільні механізми платежів

- **Валідація суми** в `handlePaymentCallback()`: різниця >0.01 → `PAYMENT_AMOUNT_MISMATCH` → failure; виявлення «копійки×100» (`PAYMENT_AMOUNT_LIKELY_KOPECKS_CONFUSION`).
- **Захист від пониження статусу:** якщо вже `paid` — наступні колбеки ігноруються.
- **Повернення** під PostgreSQL advisory-lock (namespace `0x52454655`) проти подвійного refund + пропорційне списання бонусів лояльності.
- **Side-effects (non-blocking):** `notifyDropshipForOrder()` (сповіщення дропшип-постачальника) та `trackPurchase()` (CAPI + GA4).
- **Тест credentials:** `POST /api/v1/admin/payment-settings/test` (≤5 спроб/користувача).

### Власні вебхуки магазину (WebhookSubscription)

Окрема система вихідних вебхуків для зовнішніх систем власника. Файл `src/services/webhook-dispatcher.ts`, модель `WebhookSubscription`.

- Події: `order.created`, `order.status_changed`, `payment.received`, `stock.low`.
- `dispatchWebhook(event, payload)` — POST JSON на URL підписника з header `X-Webhook-Signature: hmacSha256(body, secret)`, таймаут 5 с. Доставки логуються у `WebhookDelivery` (fire-and-forget).
- Секрет генерується (32 hex-байти), шифрується at-rest, plaintext показується один раз.
- Адмінка `/admin/webhooks`: API `GET/POST /api/v1/admin/webhooks`, `PATCH/DELETE /api/v1/admin/webhooks/{id}`, логи доставок `/deliveries` + retry `/deliveries/{id}/retry`.

## Доставка

Дві служби через офіційні API. Ключі — в `/admin/delivery-settings` (зашифровані), завантаження `getNovaPoshtaCreds()`/`getUkrposhtaCreds()` з **ENV-fallback** (на відміну від платежів). Вибір вартості — у `delivery-cost.ts`.

### Нова Пошта

Файл `src/services/nova-poshta.ts` — **повна інтеграція** (понад 25 методів). Credentials: ENV `NOVA_POSHTA_API_KEY` або DB `delivery_nova_poshta_api_key` (DB перекриває).

| Група                    | Функції                                                                                                              |

| ТТН                      | `createInternetDocument`, `getDocumentList`, `deleteInternetDocument`                                                |
| Вартість/дата            | `estimateDeliveryCost` (`getDocumentPrice`), `getDeliveryDate` (`getDocumentDeliveryDate`)                           |
| Трекінг                  | `trackParcel` (`TrackingDocument.getStatusDocuments`)                                                                |

| Друк                     | `buildPrintUrl`, `fetchPrintPdf` (`document`/`marking`/`marking100x100`)                                             |
| Контрагенти              | `getCounterparties`, `getCounterpartyContactPersons`, `getCounterpartyAddresses`                                     |

Налаштування відправника в БД: `delivery_nova_poshta_sender_ref`, `_sender_city_ref` (деф. Київ), `_sender_warehouse_ref`, `_sender_phone`.

**Покрокове налаштування:**

1. Увійдіть в особистий кабінет на **https://novaposhta.ua/** → **Налаштування** → **Безпека** → **API ключі** → **Створити ключ**.
2. Скопіюйте згенерований API-ключ (API v2.0, докладніше — https://developers.novaposhta.ua/).
3. Вставте ключ у **`/admin/delivery-settings`** (поле НП; шифрується). Альтернатива — ENV `NOVA_POSHTA_API_KEY` (БД перекриває ENV).
4. Там же заповніть дані відправника: місто, відділення/поштомат, контактна особа, телефон — без них не створюється ТТН.

### Укрпошта

Файл `src/services/ukrposhta.ts` — eCom API. **Двотокенна авторизація:** `Authorization: Bearer {bearerToken}` завжди в header + `?token={counterpartyToken}` у query для всіх ендпоінтів **окрім `/addresses`**. Credentials: ENV `UKRPOSHTA_BEARER_TOKEN`/`UKRPOSHTA_COUNTERPARTY_TOKEN` або DB `delivery_ukrposhta_bearer_token`/`_counterparty_token`.

| Група          | Функції                                                                                                                                             |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Відправлення   | `createShipment` (address→client→shipment), `createShipmentGroup`, `addShipmentToGroup`, `cancelShipment`                                           |
| Адреси/клієнти | `createAddress`, `createClient`                                                                                                                     |
| Вартість       | `estimateDeliveryCost` — `POST /domestic/delivery-price` (чиста калькуляція, **без** `?token`, без побічних ефектів; null якщо немає обох postcode) |
| Трекінг        | `trackParcel`                                                                                                                                       |
| Друк           | `getShipmentForm` (A4), `getShipmentLabel` (100×100), `getGroupForm103` (реєстр)                                                                    |
| Довідник       | `searchCities`, `getPostOfficesByCityId`, `getPostOfficesByPostcode`                                                                                |

> ⚠️ `estimateDeliveryCost` НЕ використовує `/shipments` — це CREATE-ендпоінт, ризик створення реальних відправлень.

**Покрокове налаштування:**

1. Зареєструйтесь як бізнес-клієнт: **https://www.ukrposhta.ua/ua** → **Для бізнесу**.
2. Отримайте **Bearer Token** (і Counterparty Token) через підтримку або API-портал **https://dev.ukrposhta.ua/**.
3. Вставте `bearer token` і `counterparty token` у **`/admin/delivery-settings`** (шифруються). Альтернатива — ENV `UKRPOSHTA_BEARER_TOKEN` / `UKRPOSHTA_COUNTERPARTY_TOKEN` (БД перекриває ENV).

> Токени потрібні лише для трекінгу, друку лейблів і створення відправлень. Autocomplete міст на чекауті використовує публічний address-classifier без авторизації.

### Палетна доставка

Файл `src/services/pallet-delivery.ts`. Конфіг — JSON у `pallet_delivery_config`. `calculatePalletDeliveryCost(weightKg, region)` = `basePrice + pricePerKg×вага`, помножене на регіональний `multiplier`; безкоштовно від `freeDeliveryThreshold`. `validatePalletOrder()` перевіряє `minWeightKg`/`maxWeightKg` (деф. 100–5000 кг).

### Самовивіз

Тип `pickup` у `delivery-cost.ts` — завжди вартість 0. Налаштування: `delivery_pickup_enabled`, `_address`, `_hours`, `_phone` (з `checkout-config.ts`).

### Резолвер вартості (`delivery-cost.ts`)

`calculateDeliveryCost(method, itemsTotal)`: `pickup`/`pallet`→0; якщо `itemsTotal ≥ delivery_free_shipping_threshold` → 0; інакше фіксована вартість `delivery_{nova_poshta,ukrposhta}_fixed_cost` (порожньо = 0 = «авто через API»).

## Боти

### Telegram

Файли `src/services/telegram.ts`, `bot-auto-reply.ts`, `bot-welcome.ts`. Credentials: `getTelegramCreds()` — DB key `channel_telegram` (JSON, `botToken` encrypted) з ENV-fallback (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHANNEL_ID`, `TELEGRAM_MANAGER_CHAT_ID`). `getTelegramCreds()` **не перевіряє** прапорець `enabled` — токени працюють для сповіщень навіть коли маркетинговий канал вимкнено.

**Сповіщення менеджеру:** `notifyManagerNewOrder` (нове замовлення з кнопками), `notifyManagerLowStock`, `notifyManagerMarketplaceMessages`, `notifyManagerOversoldAlert`, `notifyManagerMarketplaceAlert`, `notifyManagerHealthAlert`, `notifyManagerFeedback`, `notifyManagerSupplierFailed`. Дропшип: `notifySupplierDropshipOrder`.

**Сповіщення клієнту:** `sendClientNotification`, `notifyClientStatusChange` (зміна статусу замовлення з emoji).

**Клієнтський бот @z1pulitotradebot:** меню (🛒 каталог, 🔥 акції, 🆕 новинки, ⭐ популярні, 📦 мої замовлення, ✍️ відгук, 📞 контакти, ⚙️ налаштування), пагінація по 5, `/barcode <EAN>` пошук по штрихкоду, інлайн-пошук, гуртові ціни для `wholesaler`. Прив'язка акаунту через `/link` (одноразовий токен 10 хв → `user.telegramChatId`). HTML-escaping усіх полів від користувача, flood-control (`retry_after`).

**Авто-відповіді (`bot-auto-reply.ts`):** таблиця `BotAutoReply`, тригери `keyword`/`exact`/`regex`, пріоритет. **Welcome A/B (`bot-welcome.ts`):** `BotWelcomeMessage`, випадковий варіант, лічильники impressions/conversions.

**Покрокове налаштування:**

1. У Telegram знайдіть **@BotFather** → команда `/newbot` → введіть назву й юзернейм (має закінчуватись на `bot`). BotFather видасть **токен** вигляду `7123456789:AAH...`.
2. (Опційно) через @BotFather: `/setdescription`, `/setabouttext`, `/setuserpic`, `/setcommands` (start/help/status).
3. **Chat ID менеджера:** напишіть боту **@userinfobot** — він покаже ваш `Id`; або надішліть повідомлення своєму боту й відкрийте `https://api.telegram.org/bot<ТОКЕН>/getUpdates`, знайдіть `"chat":{"id":...}`.
4. **Channel ID:** створіть канал, додайте бота адміністратором з правом відправки, напишіть у канал, потім у `getUpdates` знайдіть `"chat":{"id":-100...}` (починається з `-100`).
5. Вставте токен у **`/admin/integrations`** → Telegram (ключ `channel_telegram`, `botToken` шифрується). Альтернатива — ENV `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHANNEL_ID`, `TELEGRAM_MANAGER_CHAT_ID` (БД-first з ENV-fallback).
6. **Webhook** для клієнтського бота: `curl -X POST "https://api.telegram.org/bot<ТОКЕН>/setWebhook" -H "Content-Type: application/json" -d '{"url":"https://ваш-домен/api/webhooks/telegram"}'`. Перевірка — `getWebhookInfo`.

> **Viber видалено** як API-канал (публікації/бот/сповіщення). Залишено лише контактне посилання `social_viber`. Не плутати з колишнім `channel_viber`.

## Соцмережі

Налаштовуються в `/admin/integrations`, зберігаються через `channel-config.ts` (DB-first з ENV-fallback). Версія Meta Graph API централізована в `src/services/meta-graph.ts` — **v21.0**.

### Facebook

Файл `facebook.ts`. `getFacebookCreds()` → DB `channel_facebook` (`pageId`, `pageAccessToken` encrypted), ENV-fallback `FACEBOOK_PAGE_ACCESS_TOKEN`/`FACEBOOK_PAGE_ID`. Можливості: `publishTextPost`, `publishPhotoPost`, `publishMultiPhotoPost` (альбом). Експоненційний retry (до 3, обробка 429).

### Instagram

Файл `instagram.ts`. `getInstagramCreds()` → DB `channel_instagram` (`accessToken`, `businessAccountId`, `tokenExpiresAt`), ENV-fallback `INSTAGRAM_ACCESS_TOKEN`/`INSTAGRAM_BUSINESS_ACCOUNT_ID`. Авто-refresh токена (`saveRefreshedInstagramToken`).

**Можливості:** `publishImagePost`, `publishCarouselPost` (2–10 фото), `publishReelsPost` (відео з полінгом статусу), `postFirstComment`, `getAccountInsights` / `getMediaInsights` (impressions/reach/engagement). **Квота:** 25 публікацій/24 год (rolling window у Redis, `instagram-quota.ts`).

**Покрокове налаштування:**

1. Зайдіть на **https://developers.facebook.com/** → станьте розробником → **My Apps** → **Create App** → тип **Business**.
2. На панелі додатку **Add Products** → **Instagram Basic Display** → **Set Up**; у **Basic Display** заповніть Valid OAuth Redirect URI `https://ваш-домен/api/v1/auth/instagram/callback` (+ deauthorize/delete URL).
3. **Roles** → **Instagram Testers** → додайте свій IG-username; прийміть запрошення в Instagram → **Settings** → **Apps and Websites** → **Tester Invites**.
4. Отримайте короткоживучий токен через `https://api.instagram.com/oauth/authorize?client_id=...&redirect_uri=...&scope=user_profile,user_media&response_type=code`, обміняйте `code` на токен (`POST /oauth/access_token`), потім на довгоживучий 60-дн (`GET graph.instagram.com/access_token?grant_type=ig_exchange_token`).
5. **Business Account ID:** `curl "https://graph.instagram.com/me?fields=id,username&access_token=<ТОКЕН>"` → візьміть `id`.
6. Вставте accessToken + businessAccountId у **`/admin/integrations`** → Instagram (`channel_instagram`, шифрується). Альтернатива — ENV `INSTAGRAM_ACCESS_TOKEN` / `INSTAGRAM_BUSINESS_ACCOUNT_ID`. Токен авто-оновлюється кроном (`instagram-token-refresh`).

### TikTok

Файл `tiktok.ts`. `getChannelConfig('tiktok')` → DB `channel_tiktok` (`accessToken`, `openId`), без ENV-fallback. `publishPhotoPost()` через `POST /v2/post/publish/content/init/` (тип PHOTO, DIRECT_POST, до 35 фото). Потребує верифікації домену (`PULL_FROM_URL`) у dev-порталі TikTok. Повертає асинхронний `publishId`.

## ERP — 1С / BAS (CommerceML)

Файл `src/services/integration-1c.ts`. Двостороння синхронізація з 1С/BAS.

| Функція                | Призначення                                                                                                               |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `importProductsFrom1C` | Імпорт товарів (upsert по `code`; категорії auto-create; фізичні розміри/cost оновлюються лише якщо 1С прислала значення) |
| `exportOrdersTo1C`     | Експорт замовлень (фільтри статус/дати, ≤1000)                                                                            |
| `updateStockFrom1C`    | Оновлення залишків по `code`                                                                                              |
| `updatePricesFrom1C`   | Оновлення цін (retail + 3 рівні гурту)                                                                                    |

Формат обміну — CommerceML (`КоммерческаяИнформация`). Кеш інвалідується після кожної операції.

**Покрокове налаштування:**

1. Згенеруйте токен авторизації: `openssl rand -hex 32`.
2. Додайте в `.env`: `ONEC_API_TOKEN=<токен>` і `ONEC_SYNC_ENABLED=true`, перезапустіть додаток.
3. У 1С/BAS створіть HTTP-з'єднання: сервер `ваш-домен`, порт `443`, захищене з'єднання — Так.
4. Налаштуйте обробку обміну з ендпоінтами (всі — `Authorization: Bearer <ONEC_API_TOKEN>`):
   - товари — `POST /api/v1/integration/1c/products`
   - залишки — `POST /api/v1/integration/1c/stock`
   - ціни — `POST /api/v1/integration/1c/prices`
   - замовлення — `GET /api/v1/integration/1c/orders`
5. Маппінг ключів: `Код`→`sku`/`code` (унікальний, по ньому upsert), `Наименование`→`name`, `Цена`→`price`, `Остаток`→`stock`, `Штрихкод`→`barcode`, `Группа`→категорія.
6. Розклад обміну: товари/ціни — раз на добу, залишки — кожні 15–30 хв, замовлення — кожні 5–10 хв; відправляйте батчами 100–500 товарів.

## Аналітика та трекінг

### Серверний трекінг (`server-tracking.ts`)

Серверні події в обхід блокувальників реклами. `trackPurchase()` / `trackAddToCart()` шлють подію паралельно в **Facebook CAPI + GA4** (non-blocking).

**Facebook Conversion API (CAPI):** ENV `FACEBOOK_PIXEL_ID`, `FACEBOOK_CAPI_TOKEN`. Endpoint `POST graph.facebook.com/v21.0/{pixelId}/events`. **Дедуплікація** через `event_id = "{eventName}-{orderId}"` (узгоджено з `code` замовлення в усіх трьох викликах trackPurchase) — браузерний Pixel шле той самий `eventID`, тож подія рахується раз. Email/телефон хешуються SHA256 (телефон нормалізується до E.164: `0XXXXXXXXX`→`380XXXXXXXXX`). Передаються `client_ip_address`, `client_user_agent`, `external_id`.

**GA4 Measurement Protocol:** ENV `GA4_MEASUREMENT_ID`, `GA4_API_SECRET`. Endpoint `POST google-analytics.com/mp/collect`. Параметри `value`, `currency` (деф. UAH), `transaction_id`, `items[]`.

### Клієнтський трекінг

- **Facebook Pixel** (`src/lib/fbpixel.ts`): `fbqTrack(event, params, opts)` — обгортка над `window.fbq`. `eventID` (`Purchase-{orderNumber}`) для дедуплікації з сервером.
- **GA4 gtag** (`src/lib/gtag.ts`): вантажиться лише якщо є `NEXT_PUBLIC_GA4_ID`. `gtagEvent.{viewItem,addToCart,removeFromCart,beginCheckout,purchase,search,selectItem}`; валюта UAH.

## Email — SMTP (Nodemailer)

Файли `src/services/email.ts`, `smtp-config.ts`. Конфіг DB-first: `smtp_host`, `smtp_port`, `smtp_user`, `smtp_pass` (encrypted), `smtp_from`, `smtp_from_name`, `smtp_secure`. ENV-fallback `SMTP_HOST` (деф. smtp.gmail.com), `SMTP_PORT` (587), `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`.

**Можливості:** `sendEmail()` (HTML+plain-text fallback, retry 3× з backoff), транзакційні листи (підтвердження замовлення, верифікація 24 год, скидання пароля 1 год), кампанії, email-sequences, abandoned-cart. Tracking-pixel (`{{notificationId}}`), `List-Unsubscribe` (One-Click), Reply-To, вкладення. Транспортер кешується 60 с (`invalidateSmtpConfigCache` скидає).

**Покрокове налаштування (Gmail App Password):**

1. Увімкніть двоетапну перевірку: **https://myaccount.google.com/security** → **Двоетапна перевірка**.
2. Створіть App Password: **https://myaccount.google.com/apppasswords** → ім'я застосунку `Pulito` → **Створити** → скопіюйте 16-символьний пароль (показується один раз).
3. Заповніть SMTP в **`/admin → Налаштування → Email`** (DB-first, `smtp_pass` шифрується) або в `.env`: `SMTP_HOST=smtp.gmail.com`, `SMTP_PORT=587`, `SMTP_USER=ваш@gmail.com`, `SMTP_PASS=<app password без пробілів>`, `SMTP_FROM=...`.

> Альтернативи: SendGrid (`SMTP_USER=apikey`, `SMTP_PASS=` API-ключ, host `smtp.sendgrid.net`) або Mailgun (`smtp.mailgun.org`). Для доставляємості налаштуйте SPF/DKIM/DMARC і `SMTP_FROM` на власному домені.

## Push — Web Push (VAPID)

Файл `src/services/push.ts`. Ключі ENV `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`; контакт `VAPID_EMAIL` (деф. mailto:noreply@pulito.trade, override `settings.vapid_email`). Підписки в `PushSubscription` (userId, endpoint, p256dh, auth).

**Можливості:** `subscribePush`, `sendPushNotification` (одному), `sendPushToAll`, `sendPushToAdmins`, `getVapidPublicKey`. Батчі по 50, авто-очистка протермінованих підписок (404/410).

**Покрокове налаштування:**

1. Згенеруйте пару ключів у корені проєкту: `npx web-push generate-vapid-keys` (виведе Public Key і Private Key).
2. Додайте в `.env`: `VAPID_PUBLIC_KEY=...`, `VAPID_PRIVATE_KEY=...`, `VAPID_EMAIL=mailto:admin@ваш-домен` (префікс `mailto:` обов'язковий).
3. На prod вкажіть **ті самі** ключі, що й на dev, інакше наявні підписки інвалідуються. Push працює лише по HTTPS (або localhost).

## Пошук — Typesense

Повнотекстовий пошук (typo-tolerance, фасети, українська мова). Локально стартує з `docker compose up -d` на `http://localhost:8108`. Конфіг — ENV.

**Покрокове налаштування:**

1. **Свій сервер:** встановіть пакет з `https://dl.typesense.org/releases/` (`dpkg -i typesense-server-*.deb` → `systemctl enable --now typesense-server`); згенеруйте API-ключ `openssl rand -hex 32`.
2. **Або Typesense Cloud:** зареєструйтесь на **https://cloud.typesense.org/** → створіть кластер (є безкоштовний план) → візьміть хост, порт і API-ключ з дашборду.
3. Додайте в `.env`: `TYPESENSE_HOST=...`, `TYPESENSE_PORT=8108` (cloud — 443), `TYPESENSE_API_KEY=...`, `TYPESENSE_PROTOCOL=http` (cloud — `https`).
4. Перша індексація: `POST /api/v1/cron/reindex-products` (Bearer cron-secret) або **`/admin → Налаштування → Пошук → Переіндексувати`**. Далі товари індексуються автоматично при створенні/редагуванні/зміні ціни.

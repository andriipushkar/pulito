# 02. Технології (стек)

Цей документ описує **повний технологічний стек** Pulito з поясненням, **чому** обрана кожна технологія і **де** вона використовується. Усі версії звірені з реальним `package.json` репозиторію.

> Загальний контекст системи — у [01-огляд-проєкту.md](01-огляд-проєкту.md). Як ці технології взаємодіють у шарах — у [03-архітектура.md](03-архітектура.md).

## Кеш, черги, фонові задачі

| Технологія      | Версія | Чому                                                                                                                               | Де використовується                               |
| --------------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| **Redis**       | 7      | Універсальний in-memory store: кеш, сесії/blacklist токенів, rate-limit (sliding window), pub/sub для event-bus, бекенд для BullMQ | `src/lib/redis.ts`, event-bus, rate-limit         |
| **ioredis**     | 5.9.x  | Redis-клієнт для Node                                                                                                              | `src/lib/redis.ts`, `src/lib/queue/connection.ts` |
| **BullMQ**      | 5.69.x | Надійні фонові черги з retry/backoff (email, push, PDF, marketplace-sync)                                                          | `src/lib/queue/` (queues, workers, connection)    |
| **cron-parser** | 5.5.x  | Парсинг cron-виразів для планувальника задач                                                                                       | Cron-роути та планування                          |

**Черги BullMQ** (з `src/lib/queue/queues.ts`): `email`, `push`, `pdf`, `marketplace-sync` — кожна зі своїми `attempts`/`backoff`/`removeOnComplete`. Cron-задачі (~60 роутів під `/api/v1/cron/*`) запускаються зовнішнім планувальником через захищені CRON_SECRET-ендпоінти. Детальніше — у [03-архітектура.md](03-архітектура.md) та [19-деплой-операції.md](19-деплой-операції.md).

| ------------------ | ----------------- | --------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **Typesense**      | 27 (client 3.0.x) | Швидкий fulltext-пошук із typo-tolerance, фасетами, синонімами; легший за Elasticsearch | Каталог, autocomplete; конфіг `TYPESENSE_*` env |

## Аутентифікація та безпека

| Технологія               | Версія | Чому                                                                 | Де використовується                              |
| ------------------------ | ------ | -------------------------------------------------------------------- | ------------------------------------------------ |
| **jsonwebtoken**         | 9.0.x  | JWT access (15m) + refresh (30d) токени; RS256 (асиметрично) / HS256 | `src/services/auth.ts`, `src/middleware/auth.ts` |
| **bcryptjs**             | 3.0.x  | Хешування паролів                                                    | Реєстрація/логін                                 |
| **Google OAuth 2.0**     | —      | Соціальний вхід; `email_verified` перевірка                          | `GOOGLE_CLIENT_ID/SECRET`, auth-роути            |
| **2FA (TOTP)**           | —      | Двофакторка з backup-кодами; секрети шифруються                      | `src/app/(admin)/admin/setup-2fa`, кабінет       |
| **isomorphic-dompurify** | 2.36.x | Захист від XSS у user-generated контенті (відгуки, опис)             | Рендер HTML                                      |
| **encryption.ts**        | —      | Шифрування чутливих даних (платіжні ключі, 2FA-секрети) у БД         | `src/lib/encryption.ts`                          |
| **cookie**               | 1.1.x  | Парсинг/серіалізація httpOnly cookies                                | Auth, сесії                                      |

**Захист на рівні застосунку:** rate limiting (Redis sliding window, глобальний + per-route у `src/middleware/api-rate-limit.ts` + `rate-limit-config.ts`), CSRF (Origin + X-Requested-With), 12 security headers (CSP+nonce, HSTS, COEP, COOP) у `src/proxy.ts`, idempotency (X-Idempotency-Key для замовлень), cookie consent (3 категорії), GDPR. Детально — у [15-безпека.md](15-безпека.md).

> CSP `script-src` мусить тримати `'unsafe-eval'` — деякі бандл-залежності використовують eval; прибирати лише після усунення eval-залежностей.

## CI/CD та інфраструктура

| Інструмент                  | Чому                                                             | Де                   |
| --------------------------- | ---------------------------------------------------------------- | -------------------- |
| **GitHub Actions**          | Pipeline: lint → test → build → e2e → deploy                     | `.github/workflows/` |
| **Docker / Docker Compose** | Локальні залежності (PostgreSQL + PgBouncer + Redis + Typesense) | `docker-compose.yml` |
| **PM2**                     | Process-менеджер на проді (`pm2 restart pulito`)                 | Прод-сервер          |
| **Nginx**                   | Reverse proxy / TLS-термінація                                   | Прод-сервер          |
| **@next/bundle-analyzer**   | Аналіз розміру бандлу (`npm run analyze`)                        | Dev                  |

> Деплой через атомарний `./deploy.sh` (білд у `.next-staging`, atomic swap, auto-rollback), а не in-place білд. Білд капований heap 3072 + cpus:2, swap 8GB; ніколи не запускати 2 білди одночасно (OOM). Детально — у [19-деплой-операції.md](19-деплой-операції.md).

# 10. Адмін-панель

Адмін-панель — центр керування всім магазином Pulito: товарами, замовленнями, складом, клієнтами, маркетингом, контентом, маркетплейсами, соцмережами та платформними налаштуваннями. Це окрема група маршрутів Next.js:

```
src/app/(admin)/admin/
```

Сторінки — переважно клієнтські (`'use client'`), що працюють через `apiClient` (`/api/v1/admin/*`). Бізнес-логіка живе в `src/services/*.ts`, спільні UI-компоненти — у `src/components/admin/` (і `src/components/admin/analytics/` для аналітики).

Адмінка нараховує ~75 сторінок, згрупованих у 11 секцій сайдбару. Єдине джерело правди про меню, breadcrumbs і Command Palette — масив `ADMIN_PAGES` у `src/app/(admin)/_lib/admin-pages.ts`.

## Доступ і ролі

Тип ролі — `UserRole`. У контексті адмінки релевантні:

| Роль                                      | Опис                                                                         | Доступ                                                                                                                                                                         |
| ----------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `OWNER` / root-admin (`role === 'admin'`) | Власник платформи. У коді layout — `isPlatformAdmin = user.role === 'admin'` | Повний доступ, включно з секцією **Платформа** (Біллінг, Тенанти, Системний стан, Журнал дій, Feature flags, Прогнозування, Аналітика пошуку)                                  |
| `ADMIN`                                   | Адміністратор магазину                                                       | Усе, крім `platformOnly`-сторінок                                                                                                                                              |
| `MANAGER`                                 | Менеджер                                                                     | Доступ до адмінки; частина чутливих дій обмежена (наприклад, у персональних цінах менеджер бачить лише закріплених за ним клієнтів). Менеджери звільнені від обов'язкового 2FA |

Гейт доступу — у `src/app/(admin)/layout.tsx`: якщо `!user || (user.role !== 'admin' && user.role !== 'manager')`, рендериться екран входу замість контенту. Сторінки з прапором `platformOnly: true` приховані від звичайних admin/manager у сайдбарі (`getNavSections(isPlatformAdmin)`), але технічно досяжні за URL — фінальний захист на цих сторінках/маршрутах API.

## Каркас інтерфейсу (layout)

`src/app/(admin)/layout.tsx` + `src/components/admin/AdminSidebar.tsx` дають спільну оболонку для всіх сторінок.

### Сайдбар

- **Секції, що згортаються** — пункти згруповані за `ADMIN_SECTIONS` (Огляд, Замовлення, Каталог, …). Активна секція автоматично розгортається при зміні маршруту; стан розгорнутих секцій зберігається в `localStorage` (`admin-sidebar-expanded-sections`).
- **Пошук по меню** — поле пошуку фільтрує пункти по всіх (видимих ролі) сторінках.
- **Закріплення (pin)** — будь-яку сторінку можна закріпити (`★`); закріплені виносяться у верхню групу та прибираються зі своєї секції, щоб не дублюватись. Стан — `localStorage` (`admin-sidebar-pinned`).
- **Згортання сайдбару** — кнопка перемикає вузький/широкий режим (`w-16` ⟷ `w-60`); стан у `localStorage` (`admin-sidebar-collapsed`), синхронізований між вкладками через `storage`-подію.
- **Живі бейджі** — пункти можуть показувати лічильники (`badgeKey`): нові замовлення (`newOrders`), новий зворотний зв'язок (`newFeedback`), очікувані гуртові заявки (`pendingWholesale`).
- **Мобільний режим** — сайдбар як overlay-діалог.

### Топбар

- **Breadcrumbs** будуються з `pathname` через `PATH_LABELS`; числові сегменти показуються як `#id`.
- **Перемикач режиму обслуговування** (див. нижче).
- **Command Palette** (`Ctrl+K`) — кнопка диспатчить подію клавіатури.
- **Ask AI** — швидке посилання на `/admin/ask`.
- **Help-панель**, **дзвіночок сповіщень** (live-стрім нових замовлень/повідомлень через `useAdminNotifications`, групування, «прочитати всі»), **довідка гарячих клавіш** (`/`).

### Command Palette

`src/components/admin/CommandPalette.tsx` — швидка навігація (`Ctrl+K`):

- Пошук по всіх сторінках меню.
- Секції результатів: **Закріплені** (з `admin-sidebar-pinned`), **Дії** (швидкі дії-ярлики: створення, збережені фільтри), **Нещодавні** (`admin-command-palette-recent`), решта.
- Швидкі дії (`QUICK_ACTION_SPECS`) — наприклад, переходи зі вже застосованими фільтрами.

### Інші елементи оболонки

- `SessionTimeoutBanner` — попередження про закінчення сесії.
- `MarketplaceHealthAlert` — банер, якщо впав health маркетплейсів.
- `AdminErrorBoundary` — перехоплення помилок рендеру.
- PWA-маніфест підмінюється на `/admin/manifest.webmanifest`, щоб встановлений з адмінки застосунок відкривався одразу на `/admin`.

## Безпека: 2FA, IP-обмеження, режим обслуговування

### Двофакторна автентифікація (2FA)

Layout вимагає 2FA для адмінів (роль `admin`, не `manager`) на критичних маршрутах. Список `TWO_FA_REQUIRED_PREFIXES`:

```
/admin/payment-settings, /admin/smtp-settings, /admin/settings, /admin/users,
/admin/audit-log, /admin/feature-flags, /admin/tenants, /admin/billing, /admin/domains,
/admin/integrations, /admin/webhooks, /admin/channel-settings, /admin/channels,
/admin/marketplaces, /admin/bot-settings
```

- Якщо адмін без `twoFactorEnabled` відкриває такий маршрут — **жорсткий блок** з екраном «увімкніть 2FA» (редирект-екран на `/admin/setup-2fa`).
- На решті сторінок показується **необов'язковий банер** з можливістю відкласти («нагадати завтра» / «сховати на місяць» — через `localStorage` `admin-2fa-snooze-until`).
- Налаштування 2FA — на `/admin/setup-2fa` (TOTP RFC 6238, секрет AES-256-GCM, QR `otpauth://`, 10 резервних кодів). Сервіс — `src/services/totp.ts`.

### IP-обмеження доступу

У `src/proxy.ts` діє whitelist IP для адмінки (UI і API). Джерело — DB-налаштування `admin_allowed_ips` (з `/admin/settings → Система`) з фолбеком на `ADMIN_ALLOWED_IPS`. Якщо список заданий, доступ до `/admin` та `/api/v1/admin` дозволений лише з цих IP (локальні `127.0.0.1`/`::1` завжди дозволені для cron/дебагу). Інакше — `403` (HTML для UI, JSON для API).

### Режим обслуговування (maintenance)

- Перемикається з топбару адмінки (`PUT /api/v1/admin/maintenance`). Стан — DB-налаштування `maintenance_mode` з фолбеком на env `MAINTENANCE_MODE`.
- `src/proxy.ts` редиректить відвідувачів на `/maintenance`, окрім дозволених шляхів (`/maintenance`, `/admin`, `/api/v1/admin`, `/api/v1/auth`).
- Поширення стану обмежене TTL кешу налаштувань (≤60 с).

## Секція: Огляд

### Dashboard (`/admin`)

Головна сторінка адмінки (`src/app/(admin)/admin/page.tsx`). Дані — `GET /api/v1/admin/dashboard/stats`. Особливість: **набір віджетів повністю налаштовується** — порядок змінюється drag-and-drop (`@dnd-kit`), окремі віджети ховаються; layout зберігається на сервері (`/api/v1/admin/dashboard/settings`).

Порядок віджетів за замовчуванням (`DEFAULT_WIDGET_ORDER`):

| Віджет                                      | Що показує                                                                                                                                          |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Аномалії** (`AnomaliesWidget`)            | Відхилення сьогоднішніх метрик від 14-денного базису (`src/services/anomaly.ts`)                                                                    |
| **AI-резюме** (`AIDashboardSummary`)        | 3–5 речень про день, згенеровані Claude/Gemini (`ai-dashboard.ts`)                                                                                  |
| **Статистика** (`stats`)                    | KPI-картки: замовлення/виручка за період (перемикач сьогодні/тиждень/місяць), нові замовлення, гуртові заявки. Sparkline-тренди, порівняння з учора |
| **Радар відтоку** (`ChurnRadarWidget`)      | Топ-клієнти з високим LTV, які не замовляли 30+ днів (`churn.ts`)                                                                                   |
| **Якість товарів** (`ProductQualityWidget`) | Товари з проблемами (без фото/alt/опису)                                                                                                            |
| **Рекомендації**                            | Що потребує уваги + статус бекапів (свіжий/застарілий/відсутній)                                                                                    |
| **Виручка за тиждень**                      | Стовпчики по днях, клік → замовлення дня                                                                                                            |
| **Сьогодні по годинах**                     | Погодинні замовлення/виручка з піком                                                                                                                |
| **Останні замовлення**                      | Список з переходом у деталі                                                                                                                         |
| **Користувачі / Товари**                    | Міні-метрики (всього, гуртові, нові; активні, низький залишок, без штрихкоду…)                                                                      |
| **Топ-товари**                              | Найбільш продавані за період                                                                                                                        |

Додатково:

- **Швидкі дії** — data-driven: спершу те, що потребує дії (нові замовлення, out-of-stock, гуртові заявки), потім статичні ярлики (додати товар, імпорт, кампанія).
- **Алерти** — кольорові банери для критичного.
- **Onboarding-чеклист** — лише для свіжого магазину без замовлень (товари → платежі → доставка → SMTP).
- **Auto-refresh** — налаштований інтервал (30/60/300 с); live-toast про нові замовлення (подія `admin:new-order`).
- **Activity feed** (`ActivityFeed`) — live-стрім подій унизу.

### Аналітика (`/admin/analytics`)

`src/app/(admin)/admin/analytics/page.tsx` — 18 вкладок. Період (7/30/90 днів) і експорт спільні. Експорт: **PDF** (`/api/v1/admin/analytics/export-pdf`, `src/services/analytics-pdf.ts`) і **CSV** (для sales/products/orders, з захистом від CSV-інʼєкцій — нейтралізація `=+-@`). Окреме посилання «💰 Прибуток» → `/admin/analytics/profit` (виручка − собівартість по товарах/категоріях/постачальниках).

Вкладки, що тягнуть `/api/v1/admin/analytics?type=…` (`src/services/analytics.ts`):

| Вкладка        | Зріз даних                                                                                                    |

| **Dashboard**  | KPI (виручка, замовлення, середній чек, нові юзери, в обробці, low-stock) + інсайти + прогноз (avg/7/30 днів) |

### Звіти (`/admin/reports`)

Бібліотека готових звітів (`analytics-reports.ts`, `report-generator.ts`): зведення продажів, залишки складу, замовлення за статусами, активність клієнтів, гуртові замовлення, доставка, фінансовий, повернення/скасування, гуртові групи, топ-товари, активність менеджерів, канали залучення, зведення дня. Можливості: вибір звіту + параметри (дати, фільтри), **email-підписка** (щоденно/щотижня/щомісяця через cron) з керуванням підписками, перехід у конструктор.

### Конструктор звітів (`/admin/reports/builder`)

Ad-hoc-звіти без SQL (`report-builder.ts`): вибір виміру (статус / тип клієнта / спосіб доставки / спосіб оплати / місяць-рік), метрик (кількість замовлень / виручка / середній чек), діапазону дат. Результат — таблиця з агрегаціями + експорт CSV.

### Податкова звітність (`/admin/tax-report`)

Розрахунок податку ФОП на спрощеній системі (`tax-report.ts`, касовий метод). Налаштування: ІПН, ПІБ, група (1/2/3), ставка/фіксований збір, поріг доходу. Звіт по рік/квартал: брутто, повернення/скасування, нетто, накладені платежі (COD), розбивка прямі продажі / маркетплейс. Розрахунок податку, попередження про наближення до ліміту групи, експорт CSV. Маркетплейс-дохід рахується як gross. Не замінює офіційний розрахунок бухгалтера.

## Секція: Замовлення

### Замовлення (`/admin/orders`, `/admin/orders/[id]`)

Головний модуль обробки замовлень (`src/services/order.ts`). Список:

- **Фільтри**: статус, тип клієнта (роздріб/гурт), спосіб і статус оплати, метод доставки, діапазон дат; стан у URL.
- **Пошук**: номер замовлення, ім'я, телефон (debounce 300 мс). **Сортування**: дата/сума/статус/номер.
- **Статистичні картки** (нові, в обробці, неоплачені, за сьогодні, виручка) — клікабельні як фільтри.
- **Панель уваги**: без ТТН 24 год, неоплачені 24 год, у обробці 3+ дні.
- **Масові дії**: зміна статусу, створення ТТН Нова Пошта, друк ТТН (PDF), додавання в реєстр (scan sheet), експорт XLSX.
- **Швидкі операції**: `OrderQuickEditDrawer` (швидке редагування/статус), збережені фільтри (`SavedViews`), гарячі клавіші (`j/k/e/s`), поллінг нових замовлень (30 с) з аудіосигналом.

Деталі замовлення (`/admin/orders/[id]`):

- **Керування статусом** з валідацією дозволених переходів і коментарем; відновлення залишків при скасуванні.
- **ТТН Нова Пошта** (`src/services/nova-poshta.ts`): ручне введення (валідація 14 цифр), автоматичне створення через `CreateTTNForm`, друк/скасування/відстеження (`TTNActions`).
- **PDF-документи** (`src/services/pdf.ts`): рахунок-фактура, накладна доставки.
- **Редагування позицій** (`OrderItemsEditor`) — лише в статусах Нове/В обробці/Підтверджено; видалення повертає залишок.
- **Таймлайн** (`OrderTimeline`), **LTV-бейдж клієнта**, маржа по товарах, внутрішні нотатки менеджера (авто-збереження), призначення менеджера, відправка фото товару клієнту, **Pack Mode** для складу, алерт про відсутність товару.

### Pick & Pack (`/admin/pack`)

Інтерфейс комплектування для складу. Завантажує замовлення `confirmed`/`paid` (поллінг 60 с). Сканування штрихкоду/SKU (перевірка штрихкоду → SKU → ID), перевірка залишків (з можливістю override), групування товарів за `locationCode`, навігація між замовленнями, відмітка скомплектованого, `Mark Packed` → `Ship` (з підтвердженням), друк, опційно камера (`CameraBarcodeScanner`). Великий шрифт, автофокус на полі сканування, аудіосигнал.

### Реєстри НП (`/admin/scan-sheets`)

Реєстри (scan sheets) Нова Пошта — групи ТТН для одного забору кур'єром. Перегляд реєстрів (номер, дата, кількість ТТН, статус друку), оновлення, видалення. Наповнюються через масову дію «Додати в реєстр» зі сторінки замовлень; інтеграція через `insertDocumentsToScanSheet`.

### Палетна доставка (`/admin/pallet-delivery`)

Налаштування палетної доставки (`pallet-delivery.ts`, `pallet.ts`): увімкнення, мін/макс вага, базова ціна за палету, ціна за кг понад базу, поріг безкоштовної доставки, орієнтовні дні, **регіональні коефіцієнти** (множники по регіонах України). Вартість = (basePrice + (вага − minWeight) × pricePerKg) × множник_регіону. Список активних палетних замовлень (`PalletListSection`).

### Підписки (`/admin/subscriptions`)

Повторні замовлення (`subscription.ts`). Фільтр за статусом (активні/призупинені/скасовані), дані (юзер, частота — weekly/biweekly/monthly, дата наступної доставки, кількість позицій), аналітична панель (`SubscriptionAnalyticsBar`). Cron `process-subscriptions` (без авто-списання) формує замовлення, `subscription-reminders` (9:00) нагадує.

## Секція: Каталог

### Товари (`/admin/products`, `/admin/products/new`, `/admin/products/[id]`)

Найбільший модуль каталогу (`src/services/product.ts`). Список:

- **CRUD**: створення (`/new`), редагування (drawer + повна сторінка), видалення з підтвердженням, **дублювання** (новий код).
- **Фільтри**: пошук (назва/код/штрихкод, debounce 300 мс), категорія, бренд, статус (активні/неактивні), склад (`stock=out`/`low`), `missingBarcode`. Сортування за датою/назвою/ціною/кількістю/продажами/порядком. Збережені фільтри.
- **Inline-редагування** в таблиці: ціна, кількість, `sortOrder` — з **оптимістичним блокуванням версій** (виявлення конфлікту при паралельному редагуванні, `409` → перезавантаження).
- **Масові дії** (`ProductQuickEditDrawer` / bulk): активувати/деактивувати, видалити, змінити категорію/бренд, **змінити ціну** (відсоток / +сума / фіксована / округлення; цілі: рітейл і 3 рівні гурту; прев'ю + ідемпотентні ключі), експорт XLSX, друк етикеток зі штрихкодами.
- Меню «…»: експорт XLSX за фільтрами, повний експорт, **переіндексація Typesense**. Віджет залишків (`InventoryStatsWidget`).

Детальна сторінка товару:

- **Основне**: назва, код (унікальний), slug (автоген), штрихкод, категорія, бренд.
- **Ціни**: рітейл, ціна «було», 3 рівні гурту (wholesale/2/3), собівартість (для маржі).
- **Залишок і габарити**: кількість, прихований залишок, вага (г), розміри (мм) для розрахунку доставки.
- **Статус/промо**: активний, промо + дати промо.
- **Зображення** (`ProductImagesManager` + `src/services/image.ts`, `watermark.ts`, `image-quality.ts`): drag-and-drop завантаження/сортування, формати на вхід JPG/PNG/WebP (до 5 МБ, мін 300×300, доповнення до 800×800), **обробка** — водяний знак (за замовч. `pulito.trade`), опційне видалення фону, генерація розмірів (800/400/150/20 px) і конвертація у **WebP + AVIF**, alt-текст за шаблоном. Аудит якості: noImages/missingMain/tooSmall/missingAlt/tooFew(<3).
- **SEO**: title (≤70), description (≤160) з валідацією; IndexNow-пінг при активації.
- **Опис**: короткий, повний (WYSIWYG), специфікації (`SpecsEditor` — key/value, JSON або «Key: Value»).
- **Багатомовність (EN)**: назва, описи, специфікації, SEO.
- **Варіанти** (`VariantsSection`): SKU, штрихкод, ціни, кількість, параметри (вибір на вітрині — фаза 2).
- **Бейджі** (`ProductBadgesSection`): promo/new_arrival/hit/eco + кастомні (текст/колір/пріоритет).
- **Залишок по складах** (`WarehouseStockSection`) + резерв, **підказка поповнення** (`RestockSuggestionHint`), **постачальники** (`SupplierChannelsSection`).

### Категорії (`/admin/categories`)

Ієрархія категорій (`category.ts`, кеш `LONG`). CRUD з drag-and-drop сортуванням, батьківські/дочірні, видимість, slug, опис, SEO + EN, іконка та обкладинка, **об'єднання (merge)** категорій, AI-генерація опису/SEO. ### Торгові марки (`/admin/brands`)

Бренди (`brand.ts`). CRUD + пошук, пагінація, логотип, вебсайт, країна, видимість, sortOrder, SEO + EN, лічильник активних товарів, версіювання конфліктів. Видалення блокується за наявності товарів.

### Комплекти (`/admin/bundles`)

### Публікації (`/admin/publications`)

Автопостинг (`publication.ts`). Створення: шаблон (новинка/промо/новини), вибір товарів, заголовок/текст, зображення/відео (карусель), водяний знак, канали (Telegram/Instagram/Facebook/TikTok/OLX/Rozetka/Prom/Epicentr/Site), хештеги, планування (швидкі «за 1 год», «завтра 9:00»), **AI-генерація** тексту, окремий текст на канал. Список з статусами (draft/scheduled/published/partial/error) + retry помилкових каналів. **Календарний вигляд** з drag-and-drop переносом. Масова публікація кількох товарів.

### Шаблони публікацій (`/admin/publication-templates`)

Шаблони з підстановкою (`publication-template.ts`): назва, канали, заголовок/текст/хештеги/перший коментар; змінні `{name}`/`{code}`/`{category}`/`{price}`; прев'ю; **bulk-генерація SEO** для всіх товарів за шаблоном.

### Чат із клієнтами (`/admin/chat`)

Чати підтримки (`chat.ts`): список кімнат, фільтр за статусом (open/assigned/resolved/closed), прив'язка користувач/агент, кількість повідомлень, пагінація, перехід у переписку.

### Google Business (`/admin/google-business`)

Профіль Google Business (`google-business.ts`): назва/адреса/телефон/сайт, рейтинг і відгуки, оновлення з кешем (5–10 хв, `force=1`), посилання на редагування/відгуки. Потребує `google_maps_api_key` + `google_business_place_id`.

### Модерація (`/admin/moderation`)

Модерація відгуків і Telegram-повідомлень (`review.ts`, `review-ai.ts`). Вкладка «Правила»: тип (stop_words/link_block/flood_limit), дія (delete/warn/ban), JSON-конфіг, активність. Вкладка «Логи»: історія дій з позначкою false positive.

## Секція: Налаштування

### Загальні (`/admin/settings`)

Глобальні налаштування сайту (`settings.ts`), розбиті на розділи: Загальні (назва/телефон/email/адреса/години/поріг безкоштовної доставки), Реквізити (юр-назва, ЄДРПОУ, ІПН, IBAN, банк, юр-адреса — з валідацією), Соцмережі (контактні посилання), Дисплей (`hide_all_quantity`), Аналітика (GA/Pixel), Доставка, AI, Лояльність, **Система** (IP-whitelist `admin_allowed_ips`, режим обслуговування). Збереження лише змінених полів.

### Платіжні системи (`/admin/payment-settings`)

Платіжні провайдери (`payment.ts`, `payment-providers/`): LiqPay (public/private key, sandbox), Monobank (токен), WayForPay (merchant/пароль/ключі). Для кожного: toggle, тест з'єднання, конфіг webhook, посилання на кабінет/доку, інструкція. Ключі лише тут (зашифровані), без env-фолбеку — це адмін-only.

### Служби доставки (`/admin/delivery-settings`)

Перевізники (`delivery-cost.ts`, `nova-poshta.ts`, `ukrposhta.ts`): Нова Пошта (API-ключ, відправник/місто/склад UUID, телефон, контакт), Укрпошта, фіксована вартість. Для кожного: toggle, тест, webhook, інструкція.

### Email/SMTP (`/admin/smtp-settings`)

SMTP (`smtp-config.ts`): хост/порт/користувач/пароль, SSL/TLS, тестовий лист, збереження.

### Головна сторінка (`/admin/homepage`)

Конструктор головної (`homepage.ts`): блоки (hero, категорії, тренди, новинки, бренди, відгуки, FAQ, SEO-текст), увімкнення/вимкнення, drag-and-drop порядок, WYSIWYG SEO-текст, оптимістичне блокування, попередження про незбережені зміни. Єдиний `DEFAULT_BLOCKS`; головна — `no-store`.

### Теми (`/admin/themes`)

Теми оформлення (`theme.ts`): набір CSS-змінних (`--color-primary`, `--color-bg`, `--color-text`, `--color-in-stock`…), готові теми з прев'ю, редагування кольорів, активація, видалення кастомних.

### SEO-шаблони (`/admin/seo-templates`)

Шаблони мета-тегів (`seo-template.ts`) для product/category: titleTemplate (≤70), descriptionTemplate (≤160), altTemplate; змінні `{name}`/`{code}`/`{category}`/`{price}`; прев'ю; bulk-генерація SEO.

### SEO-аудит (`/admin/seo-audit`)

Сканер SEO-проблем: сирітські редиректи та ланцюги, відсутні/короткі title/description, дублі заголовків, проблеми зображень (немає/немає основного/без alt), тонкий контент (<300 симв.), некоректні slug, биті посилання (404/500), canonical. Самоперевірки через `127.0.0.1`.

### 404 лог (`/admin/not-found-log`)

Лог 404: шлях, referrer, user-agent, кількість, перша/остання поява, сортування за частотою/часом, видалення окремих/усіх. Основа для створення редиректів.

### Домени (`/admin/domains`)

Керування доменами (`domain.ts`): додавання, верифікація через DNS TXT (токен), статус, видалення.

## Секція: Платформа (SaaS)

Доступ — лише root-admin/OWNER (`platformOnly`). Усі вимагають 2FA.

### Біллінг (`/admin/billing`)

Біллінг тенантів (`billing.ts`): план і статус (trial 14 днів / active / past_due), зміна плану з пропорційним кредитом, історія інвойсів (`INV-00001`, атомарний лічильник), ліміти (товари/замовлення), записи використання. Валюта UAH, цикл — місяць.

### Тенанти (`/admin/tenants`)

CRUD тенантів (`tenant.ts`): назва/slug/domain/логотип/колір/план, isActive, безпечне видалення (відмова за наявності користувачів, force — окремо), додавання/видалення користувачів з роллю (member/admin/owner), фільтри (план/активність/пошук), кеш із інвалідацією.

### Системний стан (`/admin/health`)

Знімок здоров'я (`marketplace-health.ts`, `slow-queries.ts`, `deployment-history.ts`, `sentry-issues.ts`): топ-issues Sentry (рівень, кількість, користувачі), повільні запити (`pg_stat_statements`, опційно), останні деплої (git log), health маркетплейсів з uptime-бакетами.

### Журнал дій (`/admin/audit-log`)

Аудит критичних дій (`audit.ts`): фільтри (дата Kyiv, користувач, тип дії, сутність), IP, розгортання деталей (`AuditDiff` — before/after JSON), колір-код дій. Логування ніколи не валить основну операцію (fail-soft у stderr), але важливі дії пишуться в тій самій транзакції.

### Feature flags (`/admin/feature-flags`)

Прапори фіч (`feature-flag.ts`): ключ/опис/enabled, % розгортання (стабільний хеш `userId % 100`), цільові ролі та userId, видалення; Redis-кеш TTL 60 с.

### Прогнозування (`/admin/forecasting`)

Рекомендації замовлення (`demand-forecast.ts`, `purchase-prediction.ts`): 90-денне середнє → дні до вичерпання → рекомендована кількість (Lead Time + Buffer, налаштовуються). Фільтри (рухомі товари, urgency critical/soon/ok/over-stock/no-sales). Запас із `WarehouseStock` (− резерв), fallback `Product.quantity`.

### Аналітика пошуку (`/admin/search-intel`)

Інтелект пошуку (`search-intel.ts`): топ-30 запитів без результатів + топ-30 усіх; **AI-рекомендації** (Claude/Gemini з fallback на правила) — які товари додати, які синоніми. Санітизація prompt перед LLM.

## Окремі сторінки

### Запитати AI (`/admin/ask`)

Natural-language роутер (`nl-router.ts`, `ai-dashboard.ts`) — у топбарі (не в сайдбарі). Перетворює запит українською («замовлення за тиждень», «товари без фото», «клієнти з Києва», номер телефону) у відфільтрований `/admin/...` URL через regex-правила (без LLM, для швидкості). Захист: ліміт 500 символів, санітизація control-chars, заборона `//` (open-redirect), дати у Kyiv TZ.

### Налаштування 2FA (`/admin/setup-2fa`)

TOTP-2FA (`totp.ts`): генерування секрету (20 байт, base32), QR `otpauth://`, верифікація 6-значного коду перед збереженням (AES-256-GCM), 10 резервних кодів (одноразові hex SHA-256), вимкнення з підтвердженням коду, історія логінів (IP/девайс/браузер/ОС). RFC 6238, крок 30 с, ±1 крок на clock-skew.

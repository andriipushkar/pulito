# 20. Тестування

Цей розділ описує тестовий стек платформи **Pulito**: unit/integration (Vitest), E2E (Playwright), навантажувальні тести (k6), accessibility (axe) і visual regression. Усі команди беруться з `package.json` та реальних конфігів проєкту.

Тестова база велика: **~696 файлів, ~6306 тестів** у Vitest (повністю зелена станом на 2026-06-02), плюс понад 70 E2E-специфікацій Playwright.

> ⚠️ **Vitest запускайте ТІЛЬКИ з директорії `pulito/`** (корінь проєкту). Запуск з іншого CWD ламає резолв аліаса `@` та setup-файлів.

---

## 1. Unit / Integration тести (Vitest)

Конфіг — `vitest.config.ts`:

- `environment: 'node'`
- `setupFiles: ['src/test/setup.ts']`
- **exclude:** `node_modules`, `e2e`, `.next`, `documents/testing/integration/**`
- **coverage:** провайдер `v8`, репортери `text` + `lcov`; include `src/**/*.ts(x)`; з покриття виключені UI-сторінки (`app/(admin)`, `app/(shop)`, `page.tsx`/`layout.tsx`), middleware, native-залежні сервіси (`image.ts`, `nova-poshta.ts`, PDF-генерація), cron-jobs (тестуються інтеграційно), OG-генерація (Edge).

**Команди:**

| Скрипт                  | Команда                 | Призначення                                                      |
| ----------------------- | ----------------------- | ---------------------------------------------------------------- |
| `npm test`              | `vitest run`            | One-shot прогін усіх unit/integration тестів.                    |
| `npm run test:watch`    | `vitest`                | Watch-режим (перезапуск при зміні файлів).                       |
| `npm run test:coverage` | `vitest run --coverage` | З покриттям (v8); генерує `coverage/` + `coverage-summary.json`. |

```bash
cd /home/pulitotrade/pulito       # обов'язково з кореня!
npm test
npm run test:watch
npm run test:coverage
```

Файли тестів лежать поряд із кодом: `src/**/*.test.ts` / `*.test.tsx`.

> **Підводний камінь з моками:** `vi.mock` не перехоплює `await import()` усередині SUT — використовуйте статичні імпорти. `clearAllMocks` не чистить чергу `Once`-моків.

## 2. Інтеграційні тести (окремий конфіг + Docker)

| Скрипт                     | Команда                                                                                                                                                            |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `npm run test:integration` | `docker compose -f docker-compose.test.yml up -d --wait && vitest run --config tests/integration/vitest.config.ts; docker compose -f docker-compose.test.yml down` |

Підіймає окрему тестову інфраструктуру (`docker-compose.test.yml`), ганяє інтеграційні тести з власним конфігом (`tests/integration/vitest.config.ts`), потім гасить контейнери. Ці тести виключені з основного `vitest.config.ts`, тому не плутаються з unit-набором.

## 3. E2E тести (Playwright)

Конфіг — `playwright.config.ts`:

- `testDir: './e2e'`, `fullyParallel: true`
- **workers:** 1 (CI) / 2 (локально) — dev-сервер не тягне 4 паралельні адмін-сторінки з важкими Prisma-запитами.
- **retries:** 2 (CI) / 1 (локально).
- **projects:** `chromium` (Desktop Chrome) + `mobile-chrome` (Pixel 5).
- **globalSetup:** `./tests/e2e/global-setup.ts`.
- **storageState:** `./tests/e2e/.auth/storage.json` (зберігає авторизовану сесію).
- **webServer (локально):** піднімає `npm run dev` з `DISABLE_RATE_LIMIT=1` та `APP_URL=http://localhost:3000` (http-cookies на localhost); `reuseExistingServer: true`. У CI сервер запускають окремо (`npm start`).
- `baseURL` береться з `PLAYWRIGHT_BASE_URL` (деф. `localhost:3000`).

**Команди:**

| Скрипт                | Команда                | Призначення                     |
| --------------------- | ---------------------- | ------------------------------- |
| `npm run test:e2e`    | `playwright test`      | Усі E2E-сценарії.               |
| `npm run test:e2e:ui` | `playwright test --ui` | Інтерактивний UI-режим (debug). |

```bash
npx playwright install --with-deps chromium   # перший раз
npm run test:e2e
npm run test:e2e -- --project=chromium         # тільки desktop
PLAYWRIGHT_BASE_URL=http://127.0.0.1:3000 npm run test:e2e   # проти запущеного сервера
```

## 4. Accessibility (axe)

| Скрипт              | Команда                                           |
| ------------------- | ------------------------------------------------- |
| `npm run test:a11y` | `playwright test tests/e2e/accessibility.spec.ts` |

Використовує `@axe-core/playwright` — прогоняє axe-аудит на ключових сторінках, ловить порушення WCAG (контраст, ARIA, alt-тексти, фокус).

```bash
npm run test:a11y
```

## 5. Visual regression

| Скрипт                           | Команда                                               | Призначення                                                 |
| -------------------------------- | ----------------------------------------------------- | ----------------------------------------------------------- |
| `npm run test:e2e:visual`        | `playwright test tests/e2e/visual-regression.spec.ts` | Порівняння скриншотів із базовими снапшотами.               |
| `npm run test:e2e:visual:update` | `... --update-snapshots`                              | Перегенерувати еталонні снапшоти (після навмисних UI-змін). |

Базові скриншоти лежать у `tests/e2e/visual-regression.spec.ts-snapshots/`. Падіння = візуальна різниця → переглянути diff, і якщо зміна навмисна — оновити снапшоти.

```bash
npm run test:e2e:visual
npm run test:e2e:visual:update   # після свідомих змін дизайну
```

## 6. Навантажувальні тести (k6)

Сценарії — у `tests/load/` (`smoke.js`, `load.js`, `stress.js`, `spike.js`). Детальніше — `tests/load/README.md`.

| Скрипт                     | Команда                       | Профіль                                  |
| -------------------------- | ----------------------------- | ---------------------------------------- |
| `npm run test:load:smoke`  | `k6 run tests/load/smoke.js`  | 1 VU, 30 с — швидка перевірка живучості. |
| `npm run test:load`        | `k6 run tests/load/load.js`   | Нормальне навантаження.                  |
| `npm run test:load:stress` | `k6 run tests/load/stress.js` | Пікове навантаження.                     |
| `npm run test:load:spike`  | `k6 run tests/load/spike.js`  | Раптовий сплеск трафіку.                 |

**Smoke** перевіряє: `/`, `/catalog`, `/api/v1/health`, `/sitemap.xml`. **Thresholds:** `http_req_duration p95 < 1500ms`, `http_req_failed rate < 1%`.

```bash
# Встановлення k6 (Ubuntu) — див. tests/load/README.md
sudo apt-get install k6

npm run test:load:smoke
k6 run -e BASE_URL=https://pulito.trade tests/load/load.js   # проти прода
```

**Орієнтири якості** (з README):

| Метрика                   | Добре    | Прийнятно | Погано    |
| ------------------------- | -------- | --------- | --------- |
| `http_req_duration` (p95) | < 500 ms | < 1500 ms | > 1500 ms |
| `http_req_failed`         | 0 %      | < 1 %     | > 1 %     |

## 7. CI/CD пайплайн (GitHub Actions)

Файл `.github/workflows/ci.yml` (тригери: push у `main`/`develop`, PR у `main`). Node 24, кеш `node_modules`/`.next/cache`/Playwright. Джоби:

1. **`lint-typecheck`** — `npx prisma generate`, `tsc --noEmit`, `npm run lint`.
2. **`security-scan`** — `npm audit` (prod high / all critical) + grep-сканер захардкоджених секретів (AWS/OpenAI/GitHub/GitLab токени, паролі).
3. **`unit-tests`** — `npm run test:coverage`, аплоад `coverage/` + summary-таблиця в `GITHUB_STEP_SUMMARY`.
4. **`build`** — (needs lint+unit) `npm run build`, аплоад `.next` артефакту.
5. **`docker-build`** — збірка Docker-образу через Buildx (gha-кеш), верифікація.
6. **`e2e-tests`** — (needs build) сервіси Postgres 16 + Redis 7, `prisma migrate deploy`, `npm start` + `wait-on`, `npm run test:e2e -- --project=chromium`; при падінні — аплоад report + traces.
7. **`deploy-staging`** — (на `develop`, push) rsync на staging-сервер.

Інші воркфлоу: `deploy.yml` (деплой), `backup.yml`, `cron-payments-tracking.yml`.

## 8. Рекомендований локальний прогін перед PR

```bash
cd /home/pulitotrade/pulito

npm run typecheck      # tsc --noEmit
npm run lint           # eslint --fix
npm test               # vitest run (~6300 тестів)
npm run test:e2e -- --project=chromium   # E2E (потрібен dev-сервер / webServer піднімається сам)
```

`deploy.sh` додатково форсить `tsc --noEmit` як Gate 0 і `admin-smoke` після свапу — тож мінімальний «зелений» набір для впевненого деплою = typecheck + vitest + admin-smoke.

# 05. Конфігурація — змінні оточення (ENV)

Цей розділ — повний довідник усіх змінних оточення платформи **Pulito**. За основу взято файл `.env.example`, описаний рядок за рядком. Локальний файл називається `.env` і лежить у корені проєкту (`/home/pulitotrade/pulito/.env` на сервері).

## Як це працює: ENV vs Адмінка

Багато налаштувань платформи **перенесено в адмінпанель** (`/admin → Налаштування`) і зберігаються в БД. Для таких параметрів **ENV є лише fallback** (початкове / резервне значення). Це зроблено, щоб власник магазину міг змінювати конфіги без редеплою.

**Особливо важливо:**

- 🔐 **Платіжні ключі** (LiqPay, Monobank, WayForPay) та **боти/канали** (Telegram, Instagram, Facebook) налаштовуються **через адмінку із шифруванням**, а не через ENV. ENV-значення лишилися як історичний fallback, але джерело правди — зашифровані записи в БД (`/admin → Налаштування платежів`, `/admin → Інтеграції`).
- ⚙️ Параметри, помічені нижче значком **«Адмінка»**, читаються DB-first; ENV застосовується лише якщо в БД порожньо.

**Правило застосування змін у ENV (production):**

- **Server-only змінні** (`TELEGRAM_*`, `NOVA_POSHTA_API_KEY`, `SMTP_*`, платіжні ключі) → достатньо `pm2 restart pulito`.
- **`NEXT_PUBLIC_*`** та структурні зміни → потрібен **rebuild** (`./deploy.sh` або `npm run build && pm2 restart pulito`), бо вони «запікаються» в бандл.

**Генерація секретів:** `openssl rand -hex 32`.

## 4. JWT / автентифікація

| Змінна                 | Обов'язкова       | Приклад           | Опис                                                                                       |
| ---------------------- | ----------------- | ----------------- | ------------------------------------------------------------------------------------------ |
| `JWT_SECRET`           | **Так**           | (32+ hex)         | Ключ підпису JWT. Згенеруйте: `openssl rand -hex 32`. **Ніколи не використовуйте дефолт.** |
| `JWT_ACCESS_TTL`       | Ні (деф. `15m`)   | `15m`             | Час життя access-токена.                                                                   |
| `JWT_REFRESH_TTL`      | Ні (деф. `30d`)   | `30d`             | Час життя refresh-токена.                                                                  |
| `JWT_ALGORITHM`        | Ні (деф. `HS256`) | `HS256` / `RS256` | Алгоритм підпису.                                                                          |
| `JWT_PRIVATE_KEY_PATH` | Лише для RS256    | `./private.pem`   | Шлях до приватного ключа RS256.                                                            |
| `JWT_PUBLIC_KEY_PATH`  | Лише для RS256    | `./public.pem`    | Шлях до публічного ключа RS256.                                                            |

> Для RS256 згенеруйте пару:
>
> ```bash
> openssl genrsa -out private.pem 2048
> openssl rsa -in private.pem -pubout -out public.pem
> ```

## Чеклист перед запуском (production)

1. `JWT_SECRET`, `APP_SECRET`, `TYPESENSE_API_KEY` — згенеровані заново (`openssl rand -hex 32`).
2. `DATABASE_URL` — на PgBouncer (6432), `REDIS_URL` — на робочий Redis.
3. `APP_URL` = `https://pulito.trade`, `NODE_ENV=production`, `TYPESENSE_PROTOCOL=https`.
4. 🔐 Платіжні ключі (LiqPay/Mono/WayForPay) внесені в **адмінку**, не лише ENV.
5. 🔐 Telegram-бот + НП-ключ внесені (адмінка / ENV відповідно).
6. SMTP заповнений (інакше не йдуть листи).
7. `NEXT_PUBLIC_*` (GA4, Pixel) задані **до** збірки.

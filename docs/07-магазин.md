# 07. Магазин (вітрина / storefront)

Цей розділ описує **публічну вітрину** платформи Pulito — усе, що бачить покупець: головну сторінку, каталог, картку товару, кошик, оформлення замовлення та супутні розділи. Технічно вітрина — це група маршрутів `(shop)` у Next.js 16 App Router:

```
src/app/[locale]/(shop)/
```

Усі сторінки локалізовані через сегмент `[locale]` і використовують `next-intl`. Базові Server-компоненти роблять fetch даних на сервері (SEO, швидкий перший рендер), інтерактивні частини винесені в клієнтські компоненти під `src/components/` (`home`, `catalog`, `product`, `cart`, `checkout`).

> Інтернаціоналізація, PWA та логіка кошика для гостя/користувача описані наприкінці розділу, бо вони наскрізні для всієї вітрини.

## Головна сторінка

Файл: `src/app/[locale]/(shop)/page.tsx`. Рендериться **динамічно** (`no-store`), бо набір і порядок блоків керується з адмінки. Блоки приходять з `getHomepageBlocks()` (`src/services/homepage.ts`), а їхній дефолт визначено єдиним `DEFAULT_BLOCKS` (читання формату `{ version, blocks }`).

### Типи динамічних блоків

| Тип блоку          | Компонент             | Що показує                                                                                                       |
| ------------------ | --------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `banner_slider`    | `BannerSlider.tsx`    | Embla-карусель банерів (desktop/mobile зображення, CTA, декоративні SVG-фони).                                   |
| `categories`       | `CategoryGrid.tsx`    | Сітка карток категорій.                                                                                          |
| `promo_products`   | `ProductCarousel.tsx` | Карусель акційних товарів.                                                                                       |
| `new_products`     | `ProductCarousel.tsx` | Новинки → посилання `/catalog?sort=newest`.                                                                      |
| `popular_products` | `ProductCarousel.tsx` | Популярне → `/catalog?sort=popular`.                                                                             |
| `brands`           | `BrandStrip.tsx`      | Стрічка логотипів брендів.                                                                                       |
| `seo_text`         | (інлайн)              | Санітизований HTML-блок з адмінки.                                                                               |
| `local_lviv`       | `LocalAdvantage.tsx`  | Блок переваг для Львова (адреса, поріг безкоштовної доставки). **Вимкнено власником** — не плутати з контактами. |

Окремо клієнтський `RecentlyViewedSection.tsx` показує нещодавно переглянуті товари (з `localStorage`).

Компоненти головної: `src/components/home/`.

## Каталог

Файл: `src/app/[locale]/(shop)/catalog/page.tsx`. Це основна точка входу для пошуку та фільтрації товарів.

### Пошук (Typesense + PostgreSQL fallback)

- Основний рушій — **Typesense** (`src/services/typesense.ts`), колекція `products` з полями: `name`, `code`, `slug`, `categoryName/Slug`, `brandName/Slug`, `priceRetail`, `quantity`, `isActive`, `isPromo`, `ordersCount` (default sorting), `imagePath`. Налаштовані синоніми та tolerance до помилок (зокрема українські варіанти).
- Індексація **fire-and-forget** при create/update товару (не блокує запит, помилки ковтаються). Реіндекс — крон `/api/v1/cron/reindex-products`.
- **Fallback на PostgreSQL full-text** (`src/services/product.ts → fullTextSearchProductIds`): `tsvector` + `ts_rank_cd` + trigram-similarity. Рейтинг: точний збіг коду → ранг tsvector → нечіткий (similarity > 0.2).

> Важливо для експлуатації: ніколи не запускати `prisma db push` на проді — це ламає GENERATED-колонку `search_vector` і пошук «вмирає» по всьому сайту. Деталі в [06](06-база-даних.md).

### Фільтри, сортування, пагінація

Компоненти: `src/components/catalog/`.

| Компонент               | Роль                                                                                                                                                                   |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `FilterSidebar.tsx`     | Десктоп-панель: дерево категорій, чекбокси брендів, ціновий слайдер (`DualRangeSlider`, 0–10000, крок 50), чекбокси «акція» та «в наявності». Стан зберігається в URL. |
| `MobileFilterSheet.tsx` | Модальна панель фільтрів для мобільних.                                                                                                                                |
| `CatalogToolbar.tsx`    | Сортування + перемикач мобільних фільтрів.                                                                                                                             |
| `FilterChips.tsx`       | Активні фільтри як чипи з кнопками скидання.                                                                                                                           |
| `CatalogClient.tsx`     | Розкладка (sticky sidebar + сітка).                                                                                                                                    |

**Query-параметри:** `search`, `category` (slug), `brand` (через кому), `price_min`, `price_max`, `promo`, `in_stock`, `sort` (`popular`/`price_asc`/`price_desc`/`name_asc`/`newest`), `page`. Пагінація — 20 товарів на сторінку.

## Картка товару

Файл: `src/app/[locale]/(shop)/product/[slug]/page.tsx`. Компоненти: `src/components/product/` (40+ файлів).

### Галерея та медіа

- `ImageGallery.tsx` — основна карусель з мініатюрами, zoom/lightbox.
- `FloatingBuyBar.tsx` — липка панель «Купити» при скролі.
- `PriceHistoryChart.tsx` — динаміка ціни в часі.

### Варіанти та купівля

`ProductInfo.tsx` — вибір варіантів (мультивимір: розмір/колір/тощо як pill-кнопки за ключем), лічильник кількості, «Додати в кошик», перемикач списку бажань. Перший варіант у наявності пре-вибирається.

### Рекомендації та соцдокази

| Компонент                                                 | Що робить                                                                  |
| --------------------------------------------------------- | -------------------------------------------------------------------------- |
| `BoughtTogetherSection.tsx`                               | «З цим купують» — до 6 товарів з `/api/v1/products/{id}/recommendations`.  |
| `ProductCarousel.tsx`                                     | Перевикористовувана горизонтальна карусель (акценти `promo`/`new`/`hits`). |
| `RecentlyViewedSection.tsx` / `RecentlyViewedTracker.tsx` | Нещодавно переглянуті (localStorage).                                      |
| `ShareButtons.tsx`                                        | Поділитися (Telegram, Facebook, Email, копіювати).                         |
| `SocialProof.tsx`                                         | Бейджі кількості відгуків і рейтингу.                                      |

### Відгуки

`ReviewSection.tsx` — пагінований список з 5-зірковим рейтингом, бейджами «підтверджена покупка», відповідями адміна, лічильником «корисно», сортуванням (newest/helpful/rating). Завантаження фото: `ReviewImageUpload.tsx`, галерея: `ReviewImageGallery.tsx`.

### Наявність і back-in-stock

`SubscribeButton.tsx` / `BackInStockButton.tsx` — підписка на сповіщення про повернення товару в наявність (`/api/v1/back-in-stock`, email).

### SEO

`ProductJsonLd.tsx` (offers + агрегований рейтинг), `ProductFaqJsonLd.tsx`, `ProductOgProperty.tsx` (`og:type=product`, ціна/валюта). Деталі — [«SEO та маркетинг»](16-seo-маркетинг.md).

## Комплекти / бандли

Файли: `src/app/[locale]/(shop)/bundles/page.tsx` (список) та `bundles/[slug]/page.tsx` (деталі). Дані — `src/services/bundle.ts`, компоненти — `src/components/bundle/`.

Бандл — це набір товарів із **реальною знижкою на checkout** (знижка бандла застосовується до суми позицій ще до купона). Покупець бачить вигоду відносно купівлі товарів поодинці.

## Кошик

Файл: `src/app/[locale]/(shop)/cart/page.tsx`. Компоненти: `src/components/cart/`.

| Компонент                 | Роль                                                                                                                              |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `CartItemRow.tsx`         | Рядок позиції (десктоп): фото, назва, код, варіант, кількість, ціна, видалення.                                                   |
| `SwipeableCartItem.tsx`   | Мобільний swipe-to-delete.                                                                                                        |
| `CartSummary.tsx`         | Підсумок: проміжна сума, прогрес-бар безкоштовної доставки, поле купона, перемикач списання бонусів, кнопка переходу на checkout. |
| `CartRecommendations.tsx` | Рекомендації під кошиком (client-only, `ssr: false`).                                                                             |

**Ключові властивості:**

- **Real-time перерахунок** при зміні кількості/видаленні позиції.
- Валідація купона з кешем (`localStorage` + перевірка `appliedTotal`).
- Прогрес-бар і поріг безкоштовної доставки.
- Усвідомлення гуртової групи користувача (роздріб vs гурт).
- Опція знижки за підписку (`SUBSCRIPTION_DISCOUNT_PERCENT`).

## Оформлення замовлення (Checkout)

Файл: `src/app/[locale]/(shop)/checkout/page.tsx`. Компоненти: `src/components/checkout/`. Процес — **4 кроки** з клієнтським станом, підтримує **гостьове** оформлення.

### Кроки

1. **Контакти** (`StepContacts.tsx`) — ім'я, email, телефон (передзаповнюються для залогінених).
2. **Доставка** (`StepDelivery.tsx`):
   - `nova_poshta` → `NovaPoshtaPicker.tsx` (автокомпліт міста/відділення),
   - `ukrposhta` → `UkrposhtaPicker.tsx`,
   - `pickup` — самовивіз,
   - `pallet` → `PalletDeliveryForm.tsx` (палетна доставка для гурту, див. [09](09-b2b-гурт.md)),
   - ручний режим (довільний текст). Оцінка вартості — `DeliveryCostEstimate.tsx`.
3. **Оплата** (`StepPayment.tsx`) — картка, Apple/Google Pay, банківський переказ, накладений платіж, швидка оплата гаманцем (`WalletQuickPay.tsx`).
4. **Підтвердження** (`StepConfirmation.tsx`) — огляд і фінальне відправлення.

Допоміжні: `CheckoutSteps.tsx` (індикатор), `FreeShippingBar.tsx`, `StockWarningBanner.tsx`, `SubscriptionToggle.tsx` (частота повторного замовлення), `TrustBadges.tsx`, `FrequentlyBought.tsx`, `OrderSuccess.tsx`.

### Логіка ціноутворення на checkout

- **Купони** — `/api/v1/coupons/validate`, знижка % або фіксована, перевіряється проти суми **після** бандл-знижок (інтегровано в `createOrder`: validate + apply + атомарне redeem + компенсація).
- **Бонуси лояльності** — списання балів (1 бал = 1 ₴).
- **Volume pricing** — гуртові знижки за обсягом (див. [09](09-b2b-гурт.md)).
- **Idempotency key** — безпека від подвійного сабміту; замовлення створюється через `/api/v1/orders`.

Логіка замовлень детально — [09](09-b2b-гурт.md) та [09. Кошик і замовлення] основного індексу.

## Порівняння товарів

Файл: `src/app/[locale]/(shop)/comparison/page.tsx`, компонент `src/components/product/ComparisonTable.tsx`.

- ID товарів зберігаються в **localStorage** (хук `useComparison()`), не в URL.
- Жорсткий ліміт **до 4 товарів** (`ids.slice(0, 4)`).
- Таблиця-матриця: фото, назви, коди, ціни (роздріб/гурт), усі рядки специфікацій, кнопки «в кошик» і «прибрати».
- Дані вантажаться на вимогу: `/api/v1/products/by-ids?ids=...`.

## Інтернаціоналізація (i18n)

- Налаштування: `src/i18n/routing.ts`, рушій — `next-intl`. Локаль у сегменті `[locale]`.
- Повні словники присутні для **uk / en / pl / ro** у `src/messages/{uk,en,pl,ro}.json`.
- Наразі в роутингу активна лише **`uk`** (`locales: ['uk']`, `defaultLocale: 'uk'`); інфраструктура для en/pl/ro збережена для повторного увімкнення.
- Server-частини використовують `getTranslations()`, клієнтські — `useTranslations()`.

## PWA (offline + інсталяція)

- **Маніфест:** `src/app/manifest.ts` (Next Metadata API) + `public/manifest.json` (display `standalone`, theme `#2563eb`, шорткати: Каталог/Кошик/Замовлення/Порівняння, іконки 192/512 maskable, скріншоти).
- **Service Worker:** `public/sw.js`:
  - precache: `/`, `/offline`, `/catalog`, `/cart`, `/comparison`;
  - API read-only GET (`/api/v1/products*`, `/api/v1/categories*`) — network-first з TTL ~5 хв;
  - статика (`/_next/static/*`, `/images/*`, `/fonts/*`) — cache-first;
  - auth/admin/webhooks — без кешу;
  - динамічні сторінки — кеш до 50 записів (FIFO), версіонування `clean-shop-{SW_VERSION}`, очистка застарілого кешу на `activate`.
- Офлайн-сторінка: `src/app/offline/`. Push-сповіщення у SW наразі **не реалізовані**.

---

## Кошик: гість vs користувач

- **Гість** — кошик у клієнтському сторі (контекст `CartContext` + `localStorage`); ціни роздрібні.
- **Залогінений** — додатково враховується роль (`wholesaler`) та гуртова група для коректних цін; купони/бонуси/персональні ціни застосовуються на checkout.
- При вході гостьовий кошик зберігається на клієнті й переноситься в сесію.

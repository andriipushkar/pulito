# 08. Особистий кабінет

Особистий кабінет — це розділ `account` всередині вітрини, доступний залогіненим користувачам. Базовий шлях:

```
src/app/[locale]/(shop)/account/
```

Спільні UI-компоненти кабінету (заголовки, картки, дашборди) лежать у `src/components/account/`. Кабінет адаптується під роль користувача: частина вкладок (`finance`, `status`, `notes`, `manager`, `pricelist`, повтор замовлення, часті товари) видима лише гуртовим клієнтам (`role === 'wholesaler'`) або адмінам. Гуртова частина детально описана в [09](09-b2b-гурт.md).

Більшість вкладок — клієнтські сторінки, що працюють через `apiClient` (`/api/v1/me/*`) і контексти `AuthContext` / `CartContext`.

## Дашборд

Файл: `account/page.tsx`. Привітання за часом доби + 4 KPI-картки: кількість замовлень, бали лояльності + рівень, загальні витрати + поточна знижка, підсумок кошика. Нижче — прогрес-бар до наступного рівня лояльності, банер апгрейду до гурту (для роздробу), 3 останні замовлення з кнопкою «Повторити» (гурт), сітка часто замовлюваних товарів (гурт) і блок нагадувань про повторну закупівлю (`RestockReminders`).

## Замовлення

Файли: `account/orders/page.tsx` (список) і `account/orders/[id]/page.tsx` (деталі).

### Список

Фільтр за статусом (`new_order`, `processing`, `confirmed`, `paid`, `shipped`, `completed`, `cancelled`, `returned`), картки з номером, датою, кількістю позицій, методами доставки/оплати, сумою та кольоровим бейджем статусу. Пагінація — 10 на сторінку.

### Деталі замовлення

- Шапка: номер, дата, сума, кольоровий статус.
- Кнопки дій: **друк / PDF-рахунок** (`PrintableOrder`), **«Оплатити зараз»** (LiqPay / Monobank / WayForPay — лише для неоплачених онлайн-замовлень), **«Скасувати»** (лише для `new_order`/`processing`).
- Два блоки інформації: контакт + адреса доставки (з ТТН-посиланням НП, якщо є) + коментар; метод і статус оплати + підсумки (знижка, доставка, проміжна сума).
- Таблиця позицій: фото, назва, код, ціна на момент замовлення, кількість, сума.

### Timeline статусів

Якщо є історія, показується вертикальний timeline переходів `старий → новий` з датою та опціональним коментарем.

### Reorder (повторне замовлення)

Кнопка «Повторити замовлення»: тягне актуальні дані товарів через `/api/v1/products/by-ids?ids=...`, додає в кошик **лише наявні** позиції в потрібній кількості, рахує пропущені (toast), редіректить у кошик. Доступно гуртовим клієнтам.

### Повернення (return-request)

Запит на повернення/повернення коштів формується по замовленню; обробка net/full-refund реалізована на боці API (деталі — [09. Кошик і замовлення] основного індексу та [10. Оплата]).

## Профіль і налаштування

Файл: `account/settings/page.tsx`.

- **Профіль:** email (read-only), ПІБ, телефон (з форматуванням), збереження.
- **Зміна пароля:** складність 8–128 символів, мінімум по одній великій/малій літері, цифрі та спецсимволу; підпис «Встановити» або «Змінити» залежно від наявності пароля.
- **Google-акаунт:** прив'язка/відв'язка (відв'язати можна лише за наявності пароля; OAuth-лінк → `/api/v1/auth/google`).
- **Експорт даних (GDPR ст. 20):** JSON з профілем, адресами, замовленнями, списками бажань; файл `my-data-YYYY-MM-DD.json`.
- **Видалення акаунта (незворотне):** діалог з підтвердженням введенням слова «ВИДАЛИТИ» + поточний пароль; `DELETE /api/v1/me/account`; персональні дані анонімізуються, історія замовлень лишається знеособленою; вихід і редірект на головну.

---

## Адреси

Файл: `account/addresses/page.tsx`. Множинні адреси доставки. Модалка: ярлик, місто (обов'язкове), вулиця, будинок, квартира, індекс, «зробити основною». Картки з бейджем «За замовчуванням», дії: зробити основною / редагувати / видалити. API: `GET/POST/PUT/DELETE /api/v1/me/addresses[/{id}]`.

---

## Безпека та 2FA

Файл: `account/security/page.tsx`.

### Двофакторна автентифікація (TOTP)

| Крок  | Дія                                                                                                                      |
| ----- | ------------------------------------------------------------------------------------------------------------------------ |
| 1     | «Увімкнути 2FA» → `POST /api/v1/auth/2fa/setup` повертає `{ secret, otpauthUrl, qrDataUrl }`.                            |
| 2     | Сканування QR у Google Authenticator / Authy.                                                                            |
| 3     | Введення 6-значного коду → `POST /api/v1/auth/2fa/verify` повертає `twoFactorEnabled` + **8 backup-кодів** (одноразові). |
| Вимк. | `POST /api/v1/auth/2fa/disable` з валідним кодом.                                                                        |

### Історія входів

Кнопка «Завантажити» → `GET /api/v1/me/login-history`: пристрій (емодзі), браузер + ОС, IP, дата/час.

## Лояльність

Файл: `account/loyalty/page.tsx`, компоненти `LoyaltyDashboard.tsx`, `ChallengeCard.tsx`.

- **4 рівні:** Bronze → Silver → Gold → Platinum. Кожен має знижку % і множник балів (×1/×1.5/×2/×3) та поріг витрат.
- Статус-картка: рівень, баланс балів, знижка, множник, прогрес-бар до наступного рівня (витрачено vs поріг, скільки лишилось).
- **Челенджі** (за наявності): картки з ціллю, поточним прогресом, нагородою (кількість замовлень / сума / реферали).
- **Стріки** — у дашборді лояльності.
- Стрічка останніх транзакцій балів (нарахування/списання/згорання) з дельтою та датою.
- API: `GET /api/v1/me/loyalty`, `GET /api/v1/me/loyalty/challenges`.

---

## Реферальна програма

Файл: `account/referral/page.tsx`.

- 3-кроковий онбординг: поділись посиланням → друг реєструється → бонус після його першого замовлення.
- Реферальне посилання (read-only + копіювання) і окремий код.
- Статистика (3 картки): запрошено, конвертовано (+ % конверсії), сумарний бонус (₴).
- API: `GET /api/v1/me/referral`.

---

## Списки бажань (множинні)

Файл: `account/wishlist/page.tsx`, компонент `WishlistShareButton.tsx`. На відміну від швидкого wishlist у вітрині (localStorage), тут — **кілька іменованих списків** на сервері.

- Створення/перейменування/видалення списків; дефолтний створюється автоматично.
- Дії над списком: поділитися (генерує посилання), «додати все в кошик» (лише наявне), перейменувати, видалити.
- Сітка товарів: фото, назва, ціна, оверлей «немає в наявності», кнопки «прибрати» і «в кошик».
- Очистка порожніх списків одним кліком.
- API: `GET/POST /api/v1/me/wishlists`, `PUT/DELETE /api/v1/me/wishlists/{id}`, `DELETE /api/v1/me/wishlists` (порожні), `DELETE /api/v1/me/wishlists/{id}/items/{productId}`.

---

## Швидке замовлення (bulk Excel/CSV)

Файл: `account/quick-order/page.tsx`. Для гурту/адміна.

- Введення: текстове поле формату `КОД  КІЛЬКІСТЬ` на рядок **або** завантаження файлу: `.txt`, `.csv`, `.xlsx/.xls` (бібліотека `read-excel-file`, колонка A=код, B=кількість).
- `POST /api/v1/quick-order` розпізнає товари; таблиця результатів зі статусами: OK / не знайдено / недостатньо на складі (показує доступну кількість).
- Підсумок (знайдено / всього, сума) і кнопка «Додати все в кошик» (лише `found`). Ефективна ціна = гуртова за наявності, інакше роздрібна.

---

## Підписки (Subscribe & Save)

Файл: `account/subscriptions/page.tsx`, компонент `SubscriptionCard.tsx`.

- Бейджі: всього / активні / на паузі.
- Картка підписки: частота (тижнева/двотижнева/місячна/раз на 2 міс.), дата наступної доставки, список позицій, сума за доставку, статус.
- Дії: пауза / відновлення / скасування — `PATCH` та `DELETE /api/v1/me/subscriptions/{id}`.
- Створюється з картки товару у вітрині.

> Крони `process-subscriptions` (\*/30, без авто-списання) і `subscription-reminders` (9:00) обробляють підписки на бекенді.

## Прогнози покупок (Restock reminders)

Компонент: `src/components/account/RestockReminders.tsx` (на дашборді). Прогнозує дату наступної купівлі за історією замовлень. Картки: фото, назва, прогнозована дата, кнопка «Замовити знову». Дані: `predictedNextDate`, `avgIntervalDays`, `confidence`. Не рендериться, якщо прогнозів немає. API: `GET /api/v1/me/predictions`.

---

## Прайс-листи PDF

Файл: `account/pricelist/page.tsx`. Гурт бачить персоналізований гуртовий PDF (за тарифною групою), роздріб — роздрібний. Завантаження: `GET /api/v1/pricelist?type=retail|wholesale` (Bearer, refresh при 401), метадані: `GET /api/v1/pricelist/meta` (кількість позицій, дата оновлення). Для роздробу — банер-CTA на заявку про гурт. Генерація PDF — `src/services/pricelist.ts` (див. [09](09-b2b-гурт.md)).

---

## Сповіщення

Файл: `account/notifications/page.tsx` + `account/notifications/preferences/page.tsx`.

- Бейдж непрочитаних, фільтри (всі / замовлення / акції / інше), «прочитати всі».
- Список з іконками за типом (`order_status`, `promo`, `price_change`, `back_in_stock`, `review_request`, `welcome`/`winback`/`system_notification`), клік позначає прочитаним і веде на пов'язану сторінку. Завантаження по 20 (load more).
- Налаштування (`preferences`) — канали email/push за типом сповіщення.
- API: `GET /api/v1/me/notifications?page&limit&filter`, `PUT /api/v1/me/notifications/{id}/read`, `PUT /api/v1/me/notifications`.

---

## Приватність та GDPR

Файл: `account/privacy/page.tsx`. Окрема кнопка експорту всіх даних у JSON (`GET /api/v1/me/gdpr-export`, файл `pulito-data-export-{timestamp}.json`). Повне видалення також доступне в `settings` (self-delete з підтвердженням) або за запитом на `privacy@pulito.trade`.
